E5+ — TP de synthèse : intégrer la caisse

Intermédiaire 40 min Piste 4 · Unreal
PrérequisE4 — Import, E5 — Material Editor et L2c — Bake & export.
🎯Objectif du TP

Boucler le pipeline dans Unreal : importer la caisse et sa texture ORM, reconstruire le Material (avec le split ORM), éclairer en Lumen et comparer au rendu de référence ref_high.png. Tu vérifies concrètement le packing ORM vu en L2c.

📦Fichiers de l'exercice

La caisse — importe le mesh, et utilise les maps que tu as exportées en L2c (preset Unreal Packed : Base Color, Normal, ORM).

  • caisse_low.fbx
  • .glb
  • UV
  • README

Cible visuelle : ref_high.png. ORM : Occlusion (R), Roughness (G), Metallic (B).

🔧 Démonstration pas à pas

  1. Importer. Dans le Content Drawer, Import ▸ caisse_low.fbx (coche Import Normals) et tes textures (Base Color, Normal, ORM).
  2. Régler les textures. Ouvre la Base Color : laisse sRGB coché. Ouvre l'ORM : décoche sRGB (données linéaires). La Normal doit avoir Compression = Normalmap (Unreal le détecte souvent au nom _N).
  3. Créer le Material. Clic droit ▸ Material, double-clique pour l'ouvrir. Ajoute trois Texture Sample : Base Color → Base Color ; Normal → Normal.
  4. Splitter l'ORM. Pose le Texture Sample de l'ORM (sampler type Linear Color ou Masks). Relie ses sorties : R → Ambient Occlusion, G → Roughness, B → Metallic. C'est le cœur du TP.
  5. Appliquer. Save, puis assigne le Material au static mesh de la caisse dans la scène.
  6. Éclairer (Lumen). Une DirectionalLight + une SkyLight (Lumen actif : Project Settings ▸ Rendering ▸ GI = Lumen, voir E6). Oriente le soleil pour révéler la Normal.
  7. Comparer. Place ref_high.png à côté de la Viewport : matières, usure, reflets. Ajuste si besoin.

🖐Essaie maintenant~1 min

Dans le Material, débranche momentanément le G (Roughness) de l'ORM et mets une constante 0.1 : toute la caisse devient brillante comme un miroir. Rebranche : la rugosité peinte revient. Tu vois ce que le canal Roughness pilote réellement.

⚠️ Pièges & erreurs courantes

Symptôme Roughness et métal ont l'air faux, couleurs bizarres.
Cause ORM importée en sRGB (c'est une donnée linéaire).
Correction Décoche sRGB sur l'ORM ; sampler type Linear Color/Masks dans le Material.
Symptôme J'ai branché l'ORM directement sur Base Color.
Cause ORM confondue avec une texture couleur.
Correction L'ORM se splitte : R→AO, G→Roughness, B→Metallic. La couleur vient de la Base Color.
Symptôme L'éclairage semble plat, la Normal ne se voit pas.
Cause Texture Normal non réglée en Normalmap.
Correction Compression Settings ▸ Normalmap ; branche-la sur l'entrée Normal.
Symptôme Scène trop sombre malgré la lumière.
Cause Pas de SkyLight / ambiance : les ombres deviennent noires.
Correction Ajoute une SkyLight (et vérifie que Lumen est actif) pour la lumière indirecte.

🔧Exercice pratique

Intègre la caisse texturée dans Unreal et rapproche-toi de ref_high.png.

  1. Importe le mesh et tes maps L2c (Base Color sRGB, ORM linéaire, Normal en Normalmap).
  2. Construis le Material avec le split ORM (R→AO, G→Roughness, B→Metallic).
  3. Éclaire en Lumen (Directional + SkyLight).
  4. Compare à la référence et ajuste.
Critères de réussite
  • L'ORM est importée en linéaire et correctement splittée (R/G/B → AO/Roughness/Metallic).
  • La Base Color est en sRGB, la Normal en Normalmap.
  • Bois et métal sont aux bons endroits, avec l'usure ; le relief se lit.
  • La scène est éclairée en Lumen (Directional + SkyLight).
  • Le rendu Unreal ressemble à ref_high.png.

Vérifie ta compréhension

Clique sur la réponse que tu penses correcte : la bonne réponse et une explication s'affichent aussitôt.

📌À retenir
  • L'ORM se splitte dans le Material : R→AO, G→Roughness, B→Metallic.
  • Espaces couleur : Base Color en sRGB, ORM en linéaire, Normal en Normalmap.
  • Éclaire en Lumen avec Directional + SkyLight pour une ambiance correcte.
  • La référence valide le résultat : compare matières, usure et reflets.

Pour aller plus loin
