E9 — Plugins (Fab, Quixel Bridge)

Débutant 30 min Piste 4 · Unreal Engine
PrérequisE3 — Créer un projet. Utile : L5 — Bibliothèques d'assets.
🎯Objectifs d'apprentissage

À la fin de cette leçon, tu sauras :

  • Activer et désactiver des plugins dans Unreal 5.7.
  • Comprendre ce qu'est Fab et pourquoi on ne dit plus « Unreal Marketplace ».
  • Utiliser Quixel Bridge, notamment pour MetaHuman et les Megascans.
  • Installer un plugin tiers et éviter les pièges de compatibilité et de licence.

Pourquoi c'est important

Unreal est extensible : au-delà de son cœur, des plugins ajoutent des outils (modélisation in-editor, rendu cinéma, importateurs…). Et la quasi-totalité du contenu prêt à l'emploi — modèles, matériaux, personnages — passe aujourd'hui par Fab, la marketplace unifiée d'Epic. Savoir activer le bon plugin et importer proprement un asset (en lisant sa licence !) fait gagner un temps précieux et évite des erreurs juridiques. Attention au vocabulaire : beaucoup de tutoriels anciens parlent d'« Unreal Engine Marketplace » — ce nom n'existe plus.

Théorie en profondeur

1. Les plugins : activer / désactiver

On gère les plugins via Edit▸Plugins. La fenêtre liste les plugins par catégorie, avec une case Enabled. Activer ou désactiver un plugin demande en général un redémarrage de l'éditeur. Unreal fournit déjà de nombreux plugins intégrés : Modeling Tools (modélisation/retopo dans l'éditeur), Niagara (renvoi E8), Quixel Bridge, Water, Movie Render Queue (rendu haute qualité)… On distingue les plugins du moteur (disponibles partout) et ceux du projet (rangés dans le dossier Plugins/ du projet).

💡N'active que ce dont tu as besoin

Chaque plugin actif consomme de la mémoire et peut introduire des bugs ou allonger les temps de compilation. Active à la demande, désactive ce que tu n'utilises pas.

2. Fab : la marketplace unifiée d'Epic

Fab (fab.com, lancée en octobre 2024) est la marketplace unique d'Epic. Elle a remplacé à elle seule quatre anciennes plateformes : l'Unreal Engine Marketplace, le Sketchfab Store, Quixel.com et l'ArtStation Marketplace. Points clés :

  • Elle sert Unreal ET Unity (et d'autres moteurs) : ce n'est plus un magasin « Unreal only ».
  • Megascans (la bibliothèque de scans) y est désormais hébergée. Depuis 2025, la majorité des Megascans est payante ; certains assets restent gratuits — d'où l'importance de vérifier (renvoi L5).
  • Intégration moteur : on récupère un asset via Fab puis on l'ajoute au projet (Add to Project / téléchargement).

3. Quixel Bridge et MetaHuman

Quixel Bridge est intégré à Unreal (plugin) : il donne accès aux Megascans (désormais via Fab) et, surtout, sert de passerelle pour MetaHuman — le système d'Epic pour créer et importer des personnages humains photoréalistes (visage, peau, cheveux, rig facial). On crée un MetaHuman (via le créateur en ligne / l'outil dédié) puis on l'importe dans le projet via Bridge. C'est la voie royale pour obtenir un humain crédible sans le sculpter de zéro — en gardant à l'esprit le coût (assets lourds) et les conditions d'utilisation.

4. Installer un plugin tiers

  1. Via Fab : le plus simple — l'ajout au projet se fait automatiquement.
  2. Manuellement : on copie le dossier du plugin (contenant un fichier .uplugin) dans le dossier Plugins/ du projet (ou du moteur). S'il contient du code C++, Unreal proposera de le recompiler au prochain lancement.
  3. Puis on l'active dans Edit▸Plugins et on redémarre.

🔧 Démonstration pas à pas

  1. Ouvre Edit▸Plugins. Cherche « Modeling » : vérifie que Modeling Tools Editor Mode est activé (il ajoute un mode de modélisation dans la barre Modes). Redémarre si nécessaire.
  2. Ouvre Window▸Quixel Bridge (ou via la barre d'outils). Connecte-toi avec ton compte Epic. Parcours les Megascans : repère bien l'étiquette gratuit/payant avant de télécharger.
  3. Télécharge un asset gratuit et clique Add : il apparaît dans ton Content Browser, prêt à être placé.
  4. Sur fab.com (dans le navigateur), repère un asset compatible Unreal et utilise Add to Project. Note la licence affichée avant tout usage (renvoi L5).

⚠️ Pièges & erreurs courantes

SymptômeJe cherche l'« Unreal Engine Marketplace » et je ne le trouve pas.
CauseIl n'existe plus : Fab l'a remplacé (ainsi que Quixel.com, Sketchfab Store, ArtStation Marketplace).
CorrectionVa sur fab.com (ou Quixel Bridge intégré). Parle de « Fab ».
SymptômeLe plugin tiers que j'ai installé fait planter le projet ou n'apparaît pas.
CausePlugin incompatible avec UE 5.7, ou code C++ non recompilé pour cette version.
CorrectionVérifie la version supportée par le plugin ; laisse Unreal recompiler ; désactive-le si conflit.
SymptômeJ'ai utilisé un Megascan « gratuit » mais on me réclame une licence.
CauseDepuis 2025, la majorité des Megascans sur Fab est payante ; seuls certains restent gratuits.
CorrectionVérifie systématiquement le statut et la licence de chaque asset avant de l'intégrer (renvoi L5).
SymptômeMon projet est devenu lent / instable après avoir ajouté plusieurs plugins.
CauseTrop de plugins actifs, dont certains inutiles ou en conflit.
CorrectionDésactive ceux que tu n'utilises pas ; n'active que le nécessaire.

🔧Exercice pratique

  1. Active le plugin Modeling Tools et vérifie qu'un nouveau mode apparaît dans la barre Modes.
  2. Via Quixel Bridge, importe un asset Megascan gratuit et place-le dans la scène.
  3. Sur Fab, identifie un asset et lis sa licence : note s'il est utilisable en commercial et s'il exige une attribution.
Critères de réussite
  • Tu actives/désactives un plugin et comprends le redémarrage requis.
  • Tu importes un asset depuis Bridge/Fab dans le Content Browser.
  • Tu dis « Fab » (et non « Unreal Marketplace ») et tu vérifies une licence avant usage.

Exercices

Exercice 1Prise en main

Ouvre Edit ▸ Plugins, active un plugin intégré utile (ex. un plugin déjà fourni) et redémarre l'éditeur.

Voir une piste

Parcours la liste, coche un plugin (par ex. un outil d'édition fourni), accepte le redémarrage. Vérifie ensuite que ses options apparaissent dans les menus/panneaux correspondants.

Exercice 2Application

Désactive les plugins inutilisés de ton projet pour alléger l'éditeur et le build.

Voir une piste

Repère les plugins activés non nécessaires à ton projet et désactive-les (redémarrage). Moins de plugins = démarrage plus rapide, build plus léger et moins de risques d'incompatibilité.

Exercice 3Défi

Évalue un plugin tiers sur Fab (compatibilité version, maintenance, licence) avant une éventuelle intégration.

Voir une piste

Vérifie : versions d'Unreal supportées, date de dernière mise à jour, avis, et conditions de licence (usage commercial, redistribution). Un plugin récent, maintenu et compatible avec ta version est un bon candidat.

Exercices express — auto-corrigésTape ta réponse et vérifie-la immédiatement.
📌À retenir
  • Les plugins se gèrent dans Edit ▸ Plugins (redémarrage souvent nécessaire) ; n'active que le nécessaire.
  • Fab a remplacé l'Unreal Marketplace, Quixel.com, le Sketchfab Store et l'ArtStation Marketplace ; il sert Unreal et Unity.
  • Megascans est sur Fab (majorité payante depuis 2025) ; Quixel Bridge reste la passerelle, notamment pour MetaHuman.
  • Installe un plugin tiers via Fab ou via le dossier Plugins/ ; vérifie compatibilité UE 5.7 et licence.

Glossaire des termes introduits

Plugin (Unreal) — module activable ajoutant des fonctionnalités, géré dans Edit ▸ Plugins.

Fab — marketplace unifiée d'Epic (depuis oct. 2024), remplaçant plusieurs anciennes plateformes, pour Unreal et Unity.

Quixel Bridge — passerelle intégrée vers les Megascans et MetaHuman.

MetaHuman — système d'Epic pour créer/importer des personnages humains photoréalistes.

.uplugin — fichier descripteur d'un plugin, placé dans le dossier Plugins/.
