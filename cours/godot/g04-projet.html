G4 — Créer & configurer un projet

Débutant 30 min Piste 5 · Godot
PrérequisG3 — Renderers.
🎯Objectifs d'apprentissage

À la fin de cette leçon, tu sauras :

  • Créer un projet via le Project Manager et choisir son renderer.
  • Te repérer dans les Project Settings (notamment Rendering).
  • Comprendre la structure d'un projet : res://, le fichier project.godot, le dossier .godot.
  • Organiser proprement tes dossiers et versionner ton projet.

Pourquoi c'est important

Un projet bien rangé dès le départ évite le chaos plus tard. Godot stocke tout sous une racine virtuelle res://, et un seul fichier (project.godot) décrit le projet. Comprendre cette structure, c'est savoir ce qu'on versionne, ce qu'on partage, et où vivent les réglages — y compris le renderer (renvoi G3).

Théorie en profondeur

1. Le Project Manager

Au lancement de Godot s'ouvre le Project Manager : la liste de tes projets, avec Create, Import (ouvrir un projet existant via son project.godot) et l'onglet Asset Library (modèles de projets, renvoi G10). À la création, on choisit : le nom, un dossier vide, et le renderer (Forward+ par défaut).

2. La structure d'un projet

  • project.godot : le fichier qui définit le projet (réglages, autoloads…). Sa présence = « ce dossier est un projet Godot ».
  • res:// : la racine virtuelle qui pointe vers le dossier du projet. Tous tes chemins internes commencent par res://.
  • .godot/ : dossier de cache généré (imports, vignettes, données dérivées). On ne le versionne pas (il se régénère).
  • Tes dossiers : scenes/, assets/ (ou models/, textures/), materials/, scripts/, etc.

3. Les Project Settings

Project▸Project Settings centralise la configuration. Les sections utiles côté graphisme :

  • Rendering : le Renderer (renvoi G3), la qualité des ombres, l'anti-aliasing (MSAA, TAA, FXAA), les options de textures.
  • Display ▸ Window : résolution de base, mode d'étirement (stretch) pour l'adaptation d'écran.
  • Application : nom, icône, scène principale (lancée au démarrage).
  • Onglet Advanced Settings à activer pour voir toutes les options.

4. Organisation & versioning

Range par type ou par feature, avec des noms clairs (Godot tolère bien les deux). Renomme/déplace les fichiers depuis le dock FileSystem de Godot (pas dans l'explorateur de l'OS) : il met à jour les références automatiquement. Pour Git, un .gitignore doit au minimum ignorer .godot/ (cache régénéré). Versionne project.godot, tes scènes, scripts et assets sources.

🔧 Démonstration pas à pas

  1. Project Manager ▸ Create. Nomme le projet, choisis un dossier vide, renderer Forward+, puis Create & Edit.
  2. Dans le FileSystem, crée les dossiers scenes/, assets/, materials/, scripts/ (clic droit ▸ New Folder).
  3. Ouvre Project▸Project Settings ▸ section Rendering : repère le renderer et l'anti-aliasing. Active Advanced Settings pour voir l'étendue des options.
  4. Définis la scène principale : crée une scène, sauvegarde-la dans scenes/, puis Project Settings▸Application▸Run▸Main Scene.
  5. Ajoute un .gitignore ignorant .godot/ si tu utilises Git.

⚠️ Pièges & erreurs courantes

SymptômeJ'ai déplacé des fichiers et tout est cassé (références introuvables).
CauseFichiers déplacés/renommés dans l'explorateur de l'OS, hors de Godot.
CorrectionRenomme/déplace toujours depuis le dock FileSystem de Godot, qui met à jour les chemins.
SymptômeMon dépôt Git est énorme / plein de fichiers inutiles.
CauseLe dossier de cache .godot/ a été versionné.
CorrectionAjoute .godot/ au .gitignore ; il se régénère tout seul.
SymptômeLancer le jeu ouvre une scène vide / la mauvaise scène.
CauseAucune Main Scene définie, ou la mauvaise.
CorrectionDéfinis la Main Scene dans Project Settings ▸ Application ▸ Run.
SymptômeJe ne trouve pas un réglage de rendu.
CauseLes options avancées sont masquées par défaut.
CorrectionActive Advanced Settings en haut à droite des Project Settings.

🔧Exercice pratique

  1. Crée un projet propre avec une arborescence de dossiers claire.
  2. Configure une Main Scene et lance le jeu (F5) pour vérifier.
  3. Mets en place un .gitignore ignorant .godot/ et fais un premier commit (si tu utilises Git).
Critères de réussite
  • Le projet a une structure de dossiers cohérente.
  • Lancer le jeu ouvre la bonne scène.
  • Tu sais ce qui se versionne et ce qui s'ignore (.godot/).
📌À retenir
  • Le Project Manager crée/ouvre les projets ; le renderer se choisit ici (renvoi G3).
  • Tout vit sous res:// ; project.godot définit le projet ; .godot/ est un cache à ne pas versionner.
  • Déplace/renomme depuis le FileSystem de Godot pour préserver les références.
  • Configure la Main Scene et explore les Project Settings ▸ Rendering.

Glossaire des termes introduits

Project Manager — fenêtre d'accueil de Godot pour créer/ouvrir/importer des projets.

project.godot — fichier qui définit un projet Godot et ses réglages.

.godot/ — dossier de cache généré, à ignorer dans le versioning.

Main Scene — scène lancée au démarrage du jeu.
