G2 — Anatomie de l'éditeur Godot

Débutant 35 min Piste 5 · Godot
PrérequisG1 — Installation.
🎯Objectifs d'apprentissage

À la fin de cette leçon, tu sauras :

  • Comprendre le modèle fondateur de Godot : Node + Scene.
  • Te repérer dans les principaux docks (Scene, Inspector, FileSystem) et le viewport.
  • Naviguer dans la vue 3D et placer/transformer des nodes.
  • Saisir l'idée des scènes instanciées (réutilisation), équivalent des prefabs.

Pourquoi c'est important

Godot a une approche unique et très cohérente : tout est un Node, et on assemble des nodes en scènes qu'on imbrique. Une fois cette logique comprise, l'éditeur devient limpide. C'est différent du modèle GameObject+Components d'Unity ou Actor d'Unreal ; comprendre la nuance dès le départ évite la confusion la plus fréquente des nouveaux venus.

Théorie en profondeur

1. Le modèle Node + Scene

Un Node est une brique qui fait une chose : afficher un mesh (MeshInstance3D), émettre de la lumière (DirectionalLight3D), une caméra (Camera3D), un corps physique, du son, etc. Il existe des centaines de types de nodes spécialisés. On les organise en arbre (parent/enfant) : un node enfant hérite de la transformation de son parent.

Une Scene (scène) est simplement un arbre de nodes sauvegardé dans un fichier (.tscn). Point clé : une scène peut être instanciée à l'intérieur d'une autre. Un « personnage » (scène avec mesh + collision + animation) peut être déposé 50 fois dans une scène « niveau ». C'est l'équivalent des prefabs d'Unity, mais c'est ici le concept central : scène = brique réutilisable.

ℹ️Comparaison express

Unity : un GameObject + des Components. Unreal : un Actor + des Components. Godot : un arbre de Nodes, chaque node étant déjà spécialisé (pas de « composant » à ajouter — on choisit le bon type de node). La composition se fait par imbrication de nodes et de scènes.

2. Les docks de l'éditeur

  • Viewport (centre) : la vue d'édition. Bascule 2D / 3D / Script / AssetLib via les boutons en haut au centre.
  • Scene (dock haut-gauche) : l'arbre des nodes de la scène ouverte. On y ajoute des nodes (bouton + ou Ctrl+A).
  • FileSystem (dock bas-gauche) : les fichiers du projet (assets, scènes, scripts) rangés sous res://.
  • Inspector (dock droit) : toutes les propriétés du node sélectionné (transform, mesh, matériau, paramètres…).
  • Bottom panel : Output, Debugger, Audio, Animation… selon le contexte.

3. Naviguer et transformer

Dans le viewport 3D : clic-molette pour orbiter, Maj+molette pour le panoramique, rouleau pour zoomer ; le mode « survol » avec clic droit + ZQSD/WASD (comme un FPS). On transforme les nodes via les gizmos (déplacer/tourner/mettre à l'échelle) ou les valeurs dans l'Inspector (section Transform). F recentre la vue sur le node sélectionné.

4. Scènes instanciées (réutilisation)

Pour réutiliser : on glisse une scène (.tscn) depuis le FileSystem dans l'arbre d'une autre scène → elle devient une instance. Modifier la scène source met à jour toutes ses instances (comme un prefab). On peut aussi cliquer droit sur un node ▸ Save Branch as Scene pour extraire une sous-partie réutilisable.

🔧 Démonstration pas à pas

  1. Nouvelle scène : dans le dock Scene, choisis un node racine 3D Scene (crée un Node3D).
  2. Ajoute des enfants (bouton + / Ctrl+A) : Camera3D, DirectionalLight3D, et MeshInstance3D (assigne-lui un mesh primitif dans l'Inspector ▸ Mesh ▸ New BoxMesh).
  3. Sélectionne le mesh, déplace-le avec le gizmo ; observe ses valeurs dans l'Inspector ▸ Transform.
  4. Sauvegarde la scène (Ctrl+S) dans res://scenes/ sous un nom clair.
  5. Crée une seconde scène « niveau », puis glisse ta première scène depuis le FileSystem dans son arbre : tu viens d'instancier une scène.

⚠️ Pièges & erreurs courantes

SymptômeJe cherche où « ajouter un composant » à mon objet.
CauseGodot n'a pas de composants : on ajoute des nodes spécialisés en enfants.
CorrectionAjoute le bon type de node (MeshInstance3D, light, collision…) comme enfant.
SymptômeModifier un objet n'affecte pas ses copies (ou les affecte toutes).
CauseConfusion entre instance d'une scène et nodes dupliqués localement.
CorrectionPour partager les modifs, utilise une scène instanciée ; édite la scène source.
SymptômeMon node n'apparaît pas dans la vue.
CauseVue en mode 2D, node hors champ, ou visibilité désactivée.
CorrectionPasse en mode 3D, sélectionne le node et appuie F pour le cadrer.
SymptômeJe ne retrouve plus une propriété dans l'Inspector.
CauseLes propriétés dépendent du type de node sélectionné.
CorrectionVérifie quel node est sélectionné ; les sections de l'Inspector changent en conséquence.

🔧Exercice pratique

  1. Construis une scène « Prop » (Node3D + MeshInstance3D + collision) et sauvegarde-la.
  2. Crée une scène « Niveau » et instancie ta scène « Prop » trois fois à des positions différentes.
  3. Modifie la scène « Prop » (change le mesh) et constate la mise à jour des trois instances.
Critères de réussite
  • Tu construis un arbre de nodes cohérent et le sauvegardes en .tscn.
  • Tu sais instancier une scène et expliquer pourquoi modifier la source propage le changement.
  • Tu navigues sans peine dans le viewport 3D.

Vérifie ta compréhension

Clique sur la réponse que tu penses correcte : la bonne réponse et une explication s'affichent aussitôt.

Exercices

Exercice 1Prise en main

Construis une petite scène 2D : un node racine, un Sprite2D enfant, et règle sa texture dans l'Inspector.

Voir une piste

Ajoute un Node2D racine, puis un Sprite2D enfant. Glisse une image dans la propriété Texture de l'Inspector. La hiérarchie de l'arbre détermine l'organisation et les transformations relatives.

Exercice 2Application

Transforme un groupe de nodes en scène réutilisable (.tscn) et instancie-la deux fois.

Voir une piste

Sélectionne le node racine du groupe, clic droit ▸ Save Branch as Scene. Puis « Instantiate Child Scene » pour en placer plusieurs copies. Modifier la scène source met à jour toutes les instances.

Exercice 3Défi

Organise ton interface (docks) et bascule entre les éditeurs 2D, 3D et Script.

Voir une piste

Les boutons 2D / 3D / Script en haut changent le contexte d'édition. Tu peux déplacer/réorganiser les docks (Scene, FileSystem, Inspector) selon ton flux de travail.

📌À retenir
  • Dans Godot, tout est un Node spécialisé ; on les organise en arbre.
  • Une Scene = un arbre de nodes sauvegardé ; elle s'instancie (≈ prefab) pour être réutilisée.
  • Docks clés : Scene (arbre), Inspector (propriétés), FileSystem (res://), viewport au centre.
  • Pas de « composant » à ajouter : on choisit le bon type de node.

Glossaire des termes introduits

Node — brique spécialisée de Godot (mesh, lumière, caméra…), organisée en arbre.

Scene (.tscn) — arbre de nodes sauvegardé ; réutilisable par instanciation.

Instance de scène — copie d'une scène intégrée dans une autre, mise à jour depuis la source (≈ prefab).

res:// — racine du système de fichiers du projet Godot.
