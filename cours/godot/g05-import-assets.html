G5 — Import d'assets

Intermédiaire 45 min Piste 5 · Godot
PrérequisG4, F3 — Formats et F7 — PBR.
🎯Objectifs d'apprentissage

À la fin de cette leçon, tu sauras :

  • Importer des modèles 3D — et pourquoi glTF est le format de prédilection de Godot.
  • Utiliser le dock Import pour régler comment un fichier est interprété.
  • Comprendre la scène importée et le mécanisme de réimport automatique.
  • Régler l'import des textures (compression VRAM, mipmaps, sRGB vs données).

Pourquoi c'est important

Godot importe automatiquement tout fichier déposé sous res:// et le convertit en ressource interne optimisée. Bien régler cet import — surtout le format des modèles et les réglages de textures — évite des assets trop lourds, des normal maps cassées ou des échelles fausses (renvoi C2). Bonne nouvelle : Godot est particulièrement à l'aise avec le glTF, le format ouvert recommandé.

Théorie en profondeur

1. glTF, le format de prédilection

Godot supporte le FBX, l'OBJ, mais privilégie le glTF 2.0 (renvoi F3) : ouvert, moderne, PBR natif, bonne gestion des unités et des axes. Depuis Blender, exporte en glTF (renvoi B11) pour le moins de surprises. Le .glb (binaire) embarque mesh + matériaux + textures dans un fichier ; le .gltf + dossier sépare les ressources. Le FBX fonctionne aussi (Godot le convertit), mais glTF reste le chemin le plus fiable.

2. Import automatique et dossier .godot

Dès qu'un fichier arrive sous res://, Godot l'importe : il génère un fichier .import compagnon (qui stocke les réglages) et une version optimisée dans le cache .godot/ (renvoi G4). Tu travailles toujours avec la ressource importée, pas le fichier source directement. Modifier le source (réexporter depuis Blender) déclenche un réimport automatique.

3. Le dock Import

Quand tu sélectionnes un fichier dans le FileSystem, le dock Import (souvent à côté de Scene) affiche ses réglages d'import. Selon le type :

  • Modèle (glTF/FBX) : options d'Import As (scène), génération de collisions/LOD, root type, échelle. Après réglage, clique Reimport.
  • Image (texture) : mode de compression, mipmaps, repeat/filter, et surtout l'usage sRGB vs données (voir plus bas).

4. Scène importée vs scène héritée

Un modèle importé devient une scène (un arbre de nodes : MeshInstance3D, etc.). Tu ne dois pas l'éditer directement (un réimport l'écraserait). Pour l'adapter (ajouter collision, script, matériau override), tu crées une scène héritée (clic droit ▸ New Inherited Scene) ou tu l'instancies dans une autre scène. Ainsi tes ajouts survivent aux réimports.

5. Réglages de textures

Comme dans les autres moteurs (renvoi U5, E4), l'import d'image expose :

  • Compression : VRAM Compressed (formats GPU type BCn/ASTC, renvoi F12) pour les textures de jeu ; Lossless pour l'UI/pixel art.
  • Mipmaps : activées pour la 3D (anti-aliasing au loin, renvoi F12).
  • sRGB vs données : les textures de couleur (albedo) sont traitées en sRGB ; les maps de données (normal, roughness…) en linéaire. En pratique, quand tu utilises un StandardMaterial3D (renvoi G6), Godot gère l'espace selon le slot où tu branches la texture — mais reste vigilant (renvoi F1).
  • Filter / Repeat : linéaire vs nearest (pixel art = nearest), répétition pour les textures tileable.

🔧 Démonstration pas à pas

  1. Exporte un asset en .glb depuis Blender (renvoi B11), glisse-le dans res://assets/models/ via le FileSystem.
  2. Sélectionne-le : le dock Import s'affiche. Active la génération de collision si besoin, vérifie l'échelle, clique Reimport.
  3. Crée une New Inherited Scene à partir de ce modèle pour pouvoir l'éditer (collision, script) sans risquer un écrasement.
  4. Importe une texture : sélectionne-la, règle Compression = VRAM Compressed, Mipmaps activées, puis Reimport.
  5. Instancie ta scène héritée dans une scène de niveau et vérifie le rendu (matériau, échelle, orientation).

⚠️ Pièges & erreurs courantes

SymptômeMes modifications sur un modèle importé disparaissent après réexport.
CauseTu as édité la ressource importée directement ; le réimport l'a écrasée.
CorrectionTravaille dans une scène héritée ou une instance ; tes ajouts y survivent.
SymptômeMon import FBX se comporte mal (échelle, matériaux).
CauseFBX moins fiable que glTF dans Godot ; échelle/axes non gérés.
CorrectionPrivilégie le glTF ; applique les transforms côté Blender (renvoi C2).
SymptômeMes textures saturent la mémoire vidéo.
CauseTextures non compressées (Lossless) et/ou trop grandes.
CorrectionPasse en VRAM Compressed, active les mipmaps, réduis la taille (renvoi F12).
SymptômeMon pixel art est flou en jeu.
CauseFiltre linéaire appliqué à une texture pixel art.
CorrectionRègle le Filter sur Nearest et désactive les mipmaps pour ces textures.

🔧Exercice pratique

  1. Importe un asset glTF complet (mesh + textures) dans un projet Godot.
  2. Crée une scène héritée et ajoute-lui une collision ; vérifie qu'un réimport ne casse pas tes ajouts.
  3. Règle l'import des textures (VRAM Compressed + mipmaps) et observe l'impact mémoire.
Critères de réussite
  • L'asset arrive à la bonne échelle/orientation et s'affiche correctement.
  • Tes ajouts (collision/script) survivent à un réimport (scène héritée).
  • Les textures sont compressées en VRAM avec mipmaps.

Exercices

Exercice 1Prise en main

Importe une image et un modèle glTF en les déposant dans le projet, puis place-les dans une scène.

Voir une piste

Copie les fichiers dans le dossier du projet (ou glisse-les dans FileSystem). Godot les importe automatiquement. Glisse l'image dans un Sprite2D et le glTF dans la scène 3D ; vérifie l'échelle.

Exercice 2Application

Modifie les réglages d'import d'une texture (filtre, mipmaps) et réimporte.

Voir une piste

Sélectionne la texture ▸ onglet Import ▸ change le filtre (Nearest pour du pixel art, Linear sinon) et les mipmaps, puis Reimport. Tu peux enregistrer un preset par défaut pour ce type d'asset.

Exercice 3Défi

Importe un modèle glTF animé et vérifie que ses animations sont accessibles dans Godot.

Voir une piste

Un glTF animé arrive avec un AnimationPlayer. Double-clic sur le modèle importé pour configurer (séparer les clips, boucles). Place-le en scène et lis une animation pour vérifier le rig et le skinning.

📌À retenir
  • glTF est le format de prédilection de Godot (FBX/OBJ aussi, mais glTF d'abord).
  • Tout fichier sous res:// est importé automatiquement (fichier .import + cache).
  • Édite via une scène héritée/instance, pas la ressource importée (sinon écrasée au réimport).
  • Textures : VRAM Compressed + mipmaps pour la 3D ; attention sRGB vs données (renvoi F1).

Glossaire des termes introduits

Dock Import — panneau de réglages d'import du fichier sélectionné dans Godot.

Scène importée — ressource générée à partir d'un modèle ; non éditable directement.

Scène héritée — scène dérivée d'un import, où l'on ajoute ses propres nodes sans risque d'écrasement.

Réimport — regénération automatique de la ressource quand le fichier source change.
