G3 — Renderers (Forward+ / Mobile / Compatibility)

Intermédiaire 40 min Piste 5 · Godot
PrérequisG2 — Éditeur. Utile : F9.
🎯Objectifs d'apprentissage

À la fin de cette leçon, tu sauras :

  • Distinguer les trois renderers de Godot 4 et leur API graphique (Vulkan / OpenGL).
  • Choisir le bon selon ta cible (PC, mobile, web) et tes besoins visuels.
  • Comprendre les conséquences du choix (fonctionnalités disponibles, perfs).
  • Changer de renderer et savoir pourquoi on évite d'en changer en cours de projet.

Pourquoi c'est important

Comme Unity a ses pipelines (URP/HDRP, renvoi U3), Godot 4 propose trois backends de rendu. Le choix conditionne la fidélité visuelle, les performances et les fonctionnalités disponibles (certaines, comme le SDFGI, ne marchent que sur le renderer haut de gamme). C'est une décision à prendre tôt, au moment de créer le projet.

Théorie en profondeur

1. Les trois renderers

RendererAPICibleFonctionnalités
Forward+VulkanPC / consoles modernesLe plus complet : SDFGI, VoxelGI, SSR, SSIL, volumétriques… Idéal pour la qualité 3D (défaut sur desktop).
MobileVulkan (mobile)Smartphones / tablettes récentsOptimisé pour le mobile : certaines fonctions avancées en moins, meilleures perfs sur GPU mobiles.
CompatibilityOpenGL / WebGLVieux matériel, web (HTML5), mobiles bas de gammeLe plus compatible et léger ; moins de fonctionnalités avancées, mais tourne presque partout.

En clair : Forward+ = qualité maximale sur PC ; Mobile = compromis pour téléphones ; Compatibility = « ça marche partout », notamment pour les exports web.

2. Vulkan vs OpenGL

Vulkan est l'API graphique moderne (bas niveau, performante) utilisée par Forward+ et Mobile ; elle exige un GPU et des pilotes compatibles (la quasi-totalité du matériel récent). OpenGL (Compatibility) est plus ancienne et universelle, et c'est elle qui permet l'export web via WebGL. Le rendu de Godot 4 a été reconstruit autour de Vulkan, d'où le saut de qualité par rapport à Godot 3.

3. Conséquences du choix

  • Certaines fonctionnalités d'éclairage et de post-process (SDFGI, SSIL, SSR de qualité — renvoi G7, G8) ne sont disponibles qu'en Forward+ (parfois Mobile).
  • Les performances diffèrent selon le matériel cible : Forward+ peut être plus lourd sur un GPU mobile que le renderer Mobile dédié.
  • Le rendu peut légèrement différer visuellement entre renderers : teste sur ta cible réelle.

4. Choisir et changer

On choisit le renderer à la création du projet (Project Manager) et on peut le changer ensuite dans Project Settings▸Rendering▸Renderer (un menu en haut à droite de l'éditeur le bascule aussi). Évite d'en changer tard : des matériaux/effets configurés pour Forward+ peuvent ne pas se comporter pareil en Compatibility. Décide selon ta plateforme principale dès le début.

🔧 Démonstration pas à pas

  1. Crée trois projets de test, un par renderer (Forward+, Mobile, Compatibility) avec la même scène simple (un mesh + une lumière + un WorldEnvironment).
  2. Compare l'aspect : ajoute un effet avancé (par ex. SDFGI dans le WorldEnvironment — renvoi G7) et constate qu'il n'est disponible/efficace qu'en Forward+.
  3. Dans le projet Compatibility, fais un export web (HTML5) de test pour voir l'intérêt de ce renderer.
  4. Note dans chaque projet, via Project Settings▸Rendering, les options disponibles selon le renderer.

⚠️ Pièges & erreurs courantes

SymptômeUne fonctionnalité (SDFGI, SSR…) est grisée ou sans effet.
CauseElle n'est pas supportée par le renderer choisi (souvent Compatibility).
CorrectionPasse en Forward+ si ta cible le permet, ou utilise une alternative compatible (lightmaps).
SymptômeMon jeu ne se lance pas / écran noir au démarrage.
CauseVulkan non supporté par le GPU/pilotes (Forward+/Mobile).
CorrectionMets à jour les pilotes, ou bascule en Compatibility (OpenGL).
SymptômeMon rendu change d'aspect après avoir changé de renderer.
CauseLes renderers ne gèrent pas les effets exactement pareil.
CorrectionFixe ton renderer tôt ; re-règle/teste les matériaux et l'environnement après un changement.
SymptômeMon export web ne marche pas en Forward+.
CauseLe web passe par WebGL (OpenGL), donc le renderer Compatibility.
CorrectionPour cibler le web, choisis Compatibility dès le départ.

🔧Exercice pratique

  1. Pour un projet imaginaire (ex. « jeu 3D PC stylisé » ou « petit jeu web »), justifie le renderer à choisir.
  2. Crée la scène test dans le renderer choisi et liste deux fonctionnalités d'éclairage disponibles.
  3. Bascule temporairement en Compatibility et note ce qui change/disparaît.
Critères de réussite
  • Ton choix de renderer est justifié par la cible (PC/mobile/web).
  • Tu sais citer une fonctionnalité exclusive à Forward+.
  • Tu comprends pourquoi on fixe le renderer tôt.

Exercices

Exercice 1Observation

Crée deux projets (Forward+ et Compatibility) avec une même scène simple et compare l'aspect et les options disponibles.

Voir une piste

En Forward+, des effets comme SDFGI sont disponibles dans l'Environment ; en Compatibility, ils manquent. Note les différences d'éclairage et de réflexions pour comprendre le compromis fonctionnalités/portabilité.

Exercice 2Application

Change le renderer d'un projet dans les Project Settings et observe l'impact sur une scène existante.

Voir une piste

Project Settings ▸ Rendering ▸ Renderer (Rendering Method). Après changement et redémarrage, certains effets peuvent ne plus s'afficher : c'est pourquoi le choix initial compte.

Exercice 3Défi

Pour un projet web ciblé, justifie le choix du renderer et liste une contrainte à accepter.

Voir une piste

Pour le web : Compatibility (WebGL) est requis. Contrainte : pas de SDFGI ni de certains effets avancés ; on adapte la DA (lightmaps, styles) pour rester performant et compatible navigateur.

📌À retenir
  • Trois renderers : Forward+ (Vulkan, PC, qualité max), Mobile (Vulkan mobile), Compatibility (OpenGL/web).
  • Les fonctions avancées (SDFGI, SSR, SSIL…) sont surtout en Forward+.
  • Le web passe par Compatibility (WebGL).
  • Choisis ton renderer à la création selon ta cible ; évite d'en changer tard.

Glossaire des termes introduits

Forward+ — renderer Vulkan haut de gamme de Godot (desktop), le plus complet.

Mobile (renderer) — renderer Vulkan optimisé pour les GPU mobiles.

Compatibility — renderer OpenGL/WebGL, très portable, requis pour le web.

Vulkan — API graphique moderne et performante utilisée par Forward+/Mobile.
