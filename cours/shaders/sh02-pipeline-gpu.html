SH02 — Le pipeline graphique

Débutant 40 min Piste 10 · Shaders
PrérequisSH1 — Introduction.
🎯Objectifs d'apprentissage

À la fin de cette leçon, tu sauras :

  • Décrire les étapes du pipeline : vertex → rasterisation → fragment → sortie.
  • Comprendre la rasterisation (triangles → pixels) et l'interpolation.
  • Comprendre le test de profondeur (depth) et le blending (transparence).
  • Situer ce que tu contrôles (shaders) vs ce que le GPU fait pour toi.

Pourquoi c'est important

Pour écrire des shaders, il faut savoir à quel moment ton code intervient et ce que le GPU fait automatiquement autour. Comprendre le pipeline explique des comportements clés : pourquoi les données sont interpolées, pourquoi un objet en cache un autre (profondeur), pourquoi la transparence demande un réglage spécial (blending).

Théorie en profondeur

1. Les grandes étapes

  1. Entrée : les sommets du mesh (position, normale, UV…).
  2. Vertex shader : transforme chaque sommet vers l'espace écran (clip space).
  3. Rasterisation : le GPU « remplit » chaque triangle de pixels candidats (fragments).
  4. Fragment shader : calcule la couleur de chaque fragment.
  5. Tests & sortie : test de profondeur, blending, écriture dans l'image.

2. La rasterisation & l'interpolation

Le vertex shader ne s'exécute que pour quelques sommets ; mais un triangle couvre des milliers de pixels. La rasterisation détermine quels pixels sont dans le triangle, puis interpole les données des 3 sommets (couleur, UV, normale) pour chaque pixel. C'est pourquoi un dégradé apparaît entre deux sommets de couleurs différentes : l'interpolation fait le mélange (vu en SH4).

3. Le test de profondeur (depth)

Plusieurs objets peuvent se projeter sur le même pixel. Le depth buffer mémorise la profondeur du pixel le plus proche déjà dessiné : un fragment plus lointain est rejeté. C'est ce qui fait qu'un objet proche en cache un autre, sans qu'on ait à trier manuellement chaque pixel.

4. Le blending (transparence)

Par défaut, un fragment remplace la couleur existante (opaque). Pour la transparence, on active le blending : la couleur du fragment se mélange avec le fond selon l'alpha. Les objets transparents demandent souvent un tri par distance et coûtent plus cher (overdraw, vu en SH14).

🔧 Démonstration pas à pas

  1. Crée deux objets qui se chevauchent : observe que le plus proche cache l'autre (test de profondeur).
  2. Donne des couleurs différentes aux sommets d'un triangle (vertex color) : observe le dégradé interpolé.
  3. Passe un matériau en mode transparent (blending alpha) : observe le mélange avec le fond.
  4. Empile plusieurs surfaces transparentes : note la baisse de perf (overdraw).

⚠️ Pièges & erreurs courantes

SymptômeMa transparence s'affiche dans le mauvais ordre.
CauseLes transparents se trient par distance ; le depth seul ne suffit pas.
CorrectionUtilise un mode/queue de rendu transparent et limite les superpositions.
SymptômeUne donnée « saute » entre pixels.
CauseTu attendais une valeur constante alors qu'elle est interpolée.
CorrectionComprends que les sorties vertex sont interpolées par pixel.
SymptômeDu « z-fighting » (surfaces qui clignotent).
CauseDeux surfaces quasi à la même profondeur : le depth hésite.
CorrectionDécale légèrement les surfaces ou ajuste le near/far de la caméra.
SymptômeFPS qui chute avec beaucoup de transparents.
CauseOverdraw : les pixels transparents sont recalculés plusieurs fois.
CorrectionRéduis les couches transparentes (voir SH14).

🔧Exercice pratique

  1. Mets en évidence le test de profondeur avec deux objets qui se croisent.
  2. Crée un dégradé via des vertex colors (interpolation).
  3. Active la transparence sur un objet et observe le blending.
Critères de réussite
  • Tu expliques pourquoi un objet en cache un autre.
  • Tu relies le dégradé à l'interpolation.
  • Tu sais que la transparence = blending (et son coût).

Exercices

Exercice 1Observation

Fais se croiser deux objets opaques et identifie le rôle du depth buffer.

Voir une piste

Le pixel affiché est celui du fragment le plus proche de la caméra ; les fragments plus lointains sont rejetés. Aucun tri manuel n'est nécessaire pour les opaques : le depth s'en charge.

Exercice 2Application

Crée un dégradé sur une face en attribuant des couleurs différentes à ses sommets.

Voir une piste

Avec des vertex colors différentes par sommet, la rasterisation interpole la couleur entre eux : le fragment shader reçoit une couleur lissée par pixel, d'où le dégradé.

Exercice 3Défi

Empile 3 plans transparents et mesure l'impact sur les performances (overdraw).

Voir une piste

Chaque pixel couvert par les 3 plans exécute le fragment 3 fois (overdraw ×3) et blend à chaque couche. Le coût grimpe vite : c'est pourquoi on limite les couches transparentes plein écran.

📌À retenir
  • Pipeline : entrée → vertex → rasterisation (+ interpolation) → fragment → depth/blend → image.
  • La rasterisation interpole les données des sommets pour chaque pixel.
  • Le depth gère qui cache qui ; le blending gère la transparence (et son coût).
  • Tu écris vertex & fragment ; le GPU fait la rasterisation, le depth et le blending.

Glossaire des termes introduits

Rasterisation — conversion des triangles en fragments (pixels), avec interpolation.

Interpolation — mélange des données des sommets pour chaque pixel intermédiaire.

Depth buffer — mémoire de profondeur qui détermine quel fragment est visible.

Blending — mélange d'un fragment avec le fond selon l'alpha (transparence).
