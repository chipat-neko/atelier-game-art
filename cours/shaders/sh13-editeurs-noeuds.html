SH13 — Éditeurs à nœuds (Graph)

Intermédiaire 40 min Piste 10 · Shaders
PrérequisSH7 et F9.
🎯Objectifs d'apprentissage

À la fin de cette leçon, tu sauras :

  • Reconnaître Shader Graph (Unity), Material Editor (Unreal), VisualShader (Godot).
  • Faire le lien nœud ↔ fonction de code (un nœud = une opération).
  • Construire un petit effet par nœuds (Fresnel, panning…).
  • Choisir entre code et nœuds selon le contexte.

Pourquoi c'est important

Les éditeurs à nœuds rendent les shaders accessibles aux artistes et accélèrent l'itération. Comme tu connais déjà les opérations (dot, lerp, step, Fresnel…), passer aux nœuds est immédiat : chaque nœud correspond à une fonction vue dans les leçons précédentes.

Théorie en profondeur

1. Un nœud = une opération

Dans un graph, on relie des nœuds : une texture (sample), une multiplication, un Lerp, un Fresnel, un Time… jusqu'au nœud de sortie (Base Color, Emission, etc.). C'est exactement le code de fragment, dessiné. Le flux de données va de gauche (entrées) à droite (sortie).

2. Les trois grands éditeurs

MoteurÉditeurNotes
UnityShader GraphMaster node (URP/HDRP) ; sous-graphes réutilisables.
UnrealMaterial EditorMaster Material + Material Instances (paramètres).
GodotVisualShaderAlternative au code de shader Godot.

3. Correspondance nœud ↔ code

Pour penser en nœuds, traduis ce que tu sais : lerp(a,b,t) → nœud Lerp ; dot(N,V) → nœud Dot ; pow → nœud Power ; échantillonner → Sample Texture 2D ; frac → Fraction. Le Fresnel et le Time ont souvent leur nœud dédié.

4. Quand code vs nœuds ?

Les nœuds sont parfaits pour itérer vite, rester lisibles et collaborer avec des artistes. Le code reste utile pour la logique très dense, la réutilisation fine ou des fonctions absentes du graph. Beaucoup d'éditeurs permettent d'insérer un nœud « Custom Function/Code » pour mélanger les deux.

🔧 Démonstration pas à pas

  1. Crée un Shader Graph/Material/VisualShader et relie une couleur à la sortie Base Color.
  2. Ajoute un nœud Fresnel relié à l'Emission : contour lumineux (rappel SH9).
  3. Ajoute un Time + Tiling/Offset pour faire défiler une texture (panning, SH6).
  4. Mélange deux textures avec un nœud Lerp piloté par un masque.

⚠️ Pièges & erreurs courantes

SymptômeJe ne trouve pas le bon nœud.
CauseTu cherches par syntaxe code, pas par nom de nœud.
CorrectionTraduis l'opération (lerp → Lerp, pow → Power) et cherche ce mot.
SymptômeMon graph ne compile pas.
CauseTypes incompatibles entre deux nœuds (float vs vecteur).
CorrectionAdapte les dimensions (split/combine, ou un seul canal).
SymptômeLe graph devient illisible.
CauseTrop de nœuds sans regroupement.
CorrectionRegroupe (groups), nomme, et crée des sous-graphes réutilisables.
SymptômeEffet correct mais lourd.
CauseBeaucoup de nœuds = beaucoup d'instructions par pixel.
CorrectionSimplifie ; les nœuds génèrent du code (voir SH14).

🔧Exercice pratique

  1. Refais en nœuds un effet déjà fait en code (Fresnel ou panning).
  2. Expose des paramètres (couleur, vitesse) réglables sans rouvrir le graph.
  3. Regroupe et nomme tes nœuds pour la lisibilité.
Critères de réussite
  • L'effet par nœuds correspond à la version code.
  • Les paramètres sont exposés.
  • Le graph reste lisible.

Exercices

Exercice 1Prise en main

Recrée un rim light (Fresnel) en nœuds et branche-le sur l'Emission.

Voir une piste

Nœud Fresnel Effect → Multiply (par une couleur) → Emission. Expose la puissance et la couleur en paramètres pour régler sans rouvrir le graph.

Exercice 2Application

Fais défiler une texture avec un nœud Time relié au Tiling/Offset (panning).

Voir une piste

Time → Multiply (vitesse) → entrée Offset du nœud Tiling And Offset → UV du Sample Texture. La texture défile ; ajoute une 2ᵉ couche pour casser la répétition.

Exercice 3Défi

Construis un sous-graphe réutilisable (ex. « blend deux textures par masque ») et utilise-le dans deux matériaux.

Voir une piste

Regroupe Sample A, Sample B, masque et Lerp dans un sous-graphe avec des entrées exposées. Réutilise-le : tu factorises la logique, comme une fonction en code.

📌À retenir
  • Un nœud = une opération ; un graph = le fragment shader dessiné.
  • Shader Graph (Unity), Material Editor (Unreal), VisualShader (Godot) — même logique.
  • Traduis le code en nœuds : lerp→Lerp, pow→Power, dot→Dot…
  • Nœuds pour itérer/collaborer ; code pour la logique dense (les deux se mélangent).

Glossaire des termes introduits

Shader Graph / Material Editor / VisualShader — éditeurs de shaders à nœuds (Unity/Unreal/Godot).

Nœud — bloc visuel réalisant une opération, relié à d'autres.

Sous-graphe — groupe de nœuds réutilisable (≈ fonction).
