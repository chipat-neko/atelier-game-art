GD06 — Fonctions (func)

Intermédiaire 40 min Piste 9 · GDScript (Godot)
PrérequisGD2 et GD5.
🎯Objectifs d'apprentissage

À la fin de cette leçon, tu sauras :

  • Déclarer une fonction avec func.
  • Passer des paramètres et fournir des valeurs par défaut.
  • Renvoyer une valeur avec return.
  • Typer les paramètres et le retour pour plus de sûreté.

Pourquoi c'est important

Les fonctions découpent ton code en morceaux nommés et réutilisables. Tu en as déjà rencontré (_ready, _process) : ce sont des fonctions que le moteur appelle. Apprends à créer les tiennes.

Théorie en profondeur

1. Déclarer une fonction

func tirer():
    print("Pan !")

func _ready():
    tirer()           # on appelle la fonction

2. Paramètres & valeur de retour

func calculer_degats(base, multiplicateur):
    return base * multiplicateur     # renvoie une valeur

func _ready():
    var d = calculer_degats(10, 3)   # d = 30
    print(d)

3. Typer paramètres & retour (recommandé)

Comme pour les variables (GD2), on peut typer : la flèche -> indique le type de retour :

func calculer_degats(base: int, multiplicateur: float = 1.0) -> int:
    return int(base * multiplicateur)

func saluer(nom: String) -> void:    # 'void' : ne renvoie rien
    print("Bonjour ", nom)

Le multiplicateur: float = 1.0 est une valeur par défaut : l'appelant peut l'omettre.

4. Passage : valeur vs référence

Les types simples (int, float, bool, String, Vector2) sont passés par valeur (copie). Les objets et conteneurs (Array, Dictionary, nodes) sont passés par référence : la fonction modifie l'original.

func ajouter_objet(inventaire: Array, objet: String) -> void:
    inventaire.append(objet)   # modifie le tableau de l'appelant (référence)

var sac = []
ajouter_objet(sac, "potion")
print(sac)                     # ["potion"]

5. Découper pour clarifier

Une bonne fonction fait une chose et porte un nom clair. Au lieu d'un gros _process illisible, découpe en gerer_entrees(), appliquer_mouvement(), verifier_collisions().

🔧 Démonstration pas à pas

  1. Écris func calculer_score(base: int, multi: int = 1) -> int et appelle-la.
  2. Écris func soigner(montant: int = 10) -> void avec valeur par défaut.
  3. Écris une fonction qui ajoute un objet à un Array (passage par référence).
  4. Découpe un _process en plusieurs petites fonctions nommées.

⚠️ Pièges & erreurs courantes

Symptôme« Function expects N arguments ».
CauseNombre d'arguments incorrect à l'appel.
CorrectionRespecte la signature ; utilise des valeurs par défaut si besoin.
Symptômereturn renvoie null.
CauseChemin de code sans return.
CorrectionAssure un return sur tous les chemins (ou type -> void).
SymptômeUn Array passé est modifié à mon insu.
CauseLes conteneurs sont passés par référence.
CorrectionPasse une copie (.duplicate()) si tu ne veux pas le modifier.
SymptômeType de retour incohérent non détecté.
CausePas de type de retour déclaré.
CorrectionAjoute -> Type pour activer la vérification.

🔧Exercice pratique

  1. Une fonction typée calculer_degats avec un multiplicateur par défaut.
  2. Une fonction qui remplit un Array via référence.
  3. Refactore un gros bloc en 2–3 fonctions à rôle unique.
Critères de réussite
  • Paramètres et retour typés.
  • Les valeurs par défaut fonctionnent.
  • Le code est découpé en fonctions claires.

Exercice 3Défi

Découpe une logique en deux fonctions à rôle unique (est_vivant, soigner) et combine-les.

Voir une solution 
var pv := 0

func est_vivant() -> bool:
    return pv > 0

func soigner(montant: int = 10) -> void:
    pv += montant

func _ready():
    if not est_vivant():
        soigner()
    print("PV : ", pv, " vivant : ", est_vivant())

📌À retenir
  • func nom(params): déclare une fonction ; return renvoie une valeur.
  • Type les paramètres et le retour (param: Type, -> Type).
  • Types simples par valeur ; conteneurs/objets par référence.
  • Une fonction = une chose, un nom clair.

Glossaire des termes introduits

func — mot-clé déclarant une fonction en GDScript.

Valeur par défaut — paramètre optionnel (x: int = 0).

-> Type — type de retour déclaré d'une fonction.

Passage par référence — Array/Dictionary/objets : la fonction modifie l'original.
