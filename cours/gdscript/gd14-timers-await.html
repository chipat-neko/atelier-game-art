GD14 — Timers, await & gestion du temps

Avancé 40 min Piste 9 · GDScript (Godot)
PrérequisGD7 et GD10.
🎯Objectifs d'apprentissage

À la fin de cette leçon, tu sauras :

  • Comprendre pourquoi on ne peut pas « attendre » avec une boucle.
  • Utiliser le node Timer et son signal timeout.
  • Créer un timer rapide avec get_tree().create_timer().
  • Étaler une action avec await (coroutines à la Godot).

Pourquoi c'est important

« Attends 3 s puis ouvre la porte », « fais apparaître un ennemi toutes les 2 s », « fais clignoter l'ennemi touché » : ces séquences étalées dans le temps sont partout. Godot est mono-thread : une boucle d'attente gèlerait le jeu. Les timers et await sont la solution propre — l'équivalent des coroutines d'Unity (CS14).

🛑Ce qu'il ne faut JAMAIS faire
# ❌ GÈLE le jeu : la frame ne se termine jamais pendant l'attente
func ouvrir():
    var t = 0.0
    while t < 3.0:
        t += get_process_delta_time()   # boucle bloquante sur la même frame
    porte.ouvrir()

Théorie en profondeur

1. Le node Timer

Ajoute un node Timer, règle sa durée (wait_time), coche autostart ou démarre-le par code ; il émet timeout à échéance (signal, GD10) :

@onready var minuteur: Timer = $Timer

func _ready():
    minuteur.timeout.connect(_on_timeout)
    minuteur.start(2.0)      # déclenche timeout dans 2 s

func _on_timeout():
    print("Temps écoulé !")
    # si one_shot est désactivé, le Timer reboucle automatiquement

2. Un timer rapide : create_timer

Sans ajouter de node, pour un délai ponctuel :

func ouvrir_apres_delai():
    await get_tree().create_timer(3.0).timeout   # attend 3 s SANS bloquer
    porte.ouvrir()

3. await : attendre proprement

await suspend la fonction jusqu'à ce qu'un signal soit émis, puis reprend juste après — pendant ce temps, le jeu continue de tourner. C'est la coroutine de Godot :

func clignoter(fois: int):
    for i in range(fois):
        sprite.visible = false
        await get_tree().create_timer(0.1).timeout
        sprite.visible = true
        await get_tree().create_timer(0.1).timeout

func attendre_signal():
    await mort                     # attend qu'un signal "mort" soit émis
    print("Le joueur est mort")

Une fonction qui contient await rend la main au moteur à chaque attente : la frame se termine normalement.

4. Un spawner en boucle

func lancer_vagues():
    while continuer:
        faire_apparaitre_ennemi()
        await get_tree().create_timer(2.0).timeout   # 1 ennemi toutes les 2 s

Grâce à await, cette boucle while n'est pas bloquante : elle s'interrompt à chaque attente.

🔧 Démonstration pas à pas

  1. Une fonction ouvrir_apres_delai() qui await un timer de 3 s puis logge.
  2. Un effet clignoter(fois) qui alterne la visibilité avec des await.
  3. Un spawner en while + await qui crée un ennemi toutes les 2 s.
  4. Une fonction qui await un signal personnalisé (ex. mort).

⚠️ Pièges & erreurs courantes

SymptômeLe jeu se fige pendant « l'attente ».
CauseBoucle bloquante au lieu d'await.
CorrectionUtilise await ...timeout.
SymptômeLe code après await ne s'exécute jamais.
CauseLe signal attendu n'est jamais émis.
CorrectionAssure-toi que le signal/timeout est bien déclenché.
SymptômeCrash : node détruit après await.
CauseLe node a été queue_free pendant l'attente.
CorrectionVérifie is_instance_valid(self) après l'await si besoin.
SymptômeMon Timer ne reboucle pas (ou reboucle).
CauseRéglage one_shot mal positionné.
Correctionone_shot = true pour une fois, false pour boucler.

🔧Exercice pratique

  1. Un spawner qui crée un ennemi toutes les 2 s, arrêtable via un booléen.
  2. Un effet de clignotement de 1 s quand un objet est « touché ».
  3. Une porte qui s'ouvre 3 s après un déclencheur, via await.
Critères de réussite
  • Le jeu n'est jamais gelé pendant une attente.
  • Le spawner s'arrête proprement.
  • Les délais utilisent Timer ou await ...timeout.
📌À retenir
  • Godot est mono-thread : ne bloque jamais une frame pour « attendre ».
  • Node Timer + signal timeout pour les délais récurrents.
  • await get_tree().create_timer(t).timeout pour un délai rapide.
  • await suspend la fonction jusqu'à un signal : c'est la coroutine de Godot.

Glossaire des termes introduits

Timer — node émettant timeout après une durée (une fois ou en boucle).

create_timer — crée un timer ponctuel sans ajouter de node.

await — suspend une fonction jusqu'à l'émission d'un signal, sans bloquer le jeu.

timeout — signal émis par un Timer à échéance.
