GD03 — Opérateurs & expressions

Débutant 30 min Piste 9 · GDScript (Godot)
PrérequisGD2 — Variables & types.
🎯Objectifs d'apprentissage

À la fin de cette leçon, tu sauras :

  • Utiliser les opérateurs arithmétiques, de comparaison et logiques.
  • Employer l'affectation composée (+=).
  • Comprendre la division entière et le modulo.
  • Gérer la priorité des opérateurs.

Pourquoi c'est important

Dégâts, score, distances, conditions de victoire : tout calcul de gameplay combine des opérateurs en expressions. GDScript se lit presque comme du français (and, or, not), ce qui rend la logique très claire.

Théorie en profondeur

1. Arithmétique

var a = 7
var b = 2
print(a + b)     # 9
print(a * b)     # 14
print(a / b)     # 3   -> division ENTIÈRE (deux int)
print(a % b)     # 1   -> modulo (reste)
print(7.0 / 2)   # 3.5 -> au moins un float => division flottante

2. Comparaison & logique

var egal = a == b          # ==  != < <= > >=
var peut_tirer = a_des_munitions and not en_rechargement   # 'and', 'not'
var bloque = mur_devant or ennemi_devant                   # 'or'

GDScript utilise les mots and, or, not (et accepte aussi &&, ||, !). Ils sont court-circuités : dans A and B, si A est faux, B n'est pas évalué.

3. Affectation composée

score += 10      # score = score + 10
vie -= degats
munitions -= 1
# Note : GDScript n'a PAS de ++ ni de -- ; utilise += 1 / -= 1

4. Priorité & parenthèses

var r1 = 2 + 3 * 4          # 14 (multiplication d'abord)
var r2 = (2 + 3) * 4        # 20 (parenthèses d'abord)
var deg = base * (1.0 + bonus) - reduction

5. Conversions & maths utiles

var ratio = float(pv) / pv_max          # cast en float avant de diviser
var pourcent = roundi(ratio * 100)      # arrondi à l'entier
var v = clamp(vitesse, 0.0, 600.0)      # borne une valeur entre min et max
var proche = is_equal_approx(a, b)      # comparer deux floats sans piège

Comme partout, pv / pv_max entre deux int ferait une division entière (73/100 = 0). Caste en float d'abord.

🔧 Démonstration pas à pas

  1. Formule de dégâts : var total = base * (1.0 + bonus_crit).
  2. Condition composée : peut attaquer si munitions > 0 and not en_pause.
  3. Pourcentage de vie correct (cast + roundi).
  4. Détecte un tour pair avec tour % 2 == 0.

⚠️ Pièges & erreurs courantes

SymptômeUn ratio vaut toujours 0.
CauseDivision entière entre deux int.
CorrectionCaste en float avant de diviser.
SymptômeDeux floats « égaux » testés faux.
CauseImprécision des flottants.
CorrectionUtilise is_equal_approx(a, b).
Symptôme« ++ n'existe pas ».
CauseGDScript n'a pas d'opérateur d'incrément.
CorrectionUtilise x += 1.
SymptômeRésultat de calcul inattendu.
CausePriorité des opérateurs mal anticipée.
CorrectionAjoute des parenthèses.

🔧Exercice pratique

  1. Une formule de dégâts avec bonus critique et réduction (en floats).
  2. Le pourcentage de vie restant (cast + arrondi).
  3. Une condition « peut tirer » court-circuitée.
Critères de réussite
  • Aucune division entière non voulue.
  • Les conditions composées sont claires (and/or/not).
  • Les floats sont comparés avec is_equal_approx.

Exercices

Exercice 1Échauffement

Voir une solution 
var base_degats := 10.0
var bonus := 0.5
var total := base_degats * (1.0 + bonus)   # 15.0

Exercice 2Application

Écris une condition « peut tirer » avec and et not.

Voir une solution 
var a_des_munitions := true
var en_pause := false
var peut_tirer := a_des_munitions and not en_pause   # true
Exercice 3Défi

Calcule un pourcentage de vie (cast + roundi) et compare deux flottants avec is_equal_approx.

Voir une solution 
var pourcent := roundi(float(73) / 100 * 100)   # 73
var proche := is_equal_approx(0.1 + 0.2, 0.3)   # true
print(pourcent, " ", proche)

Glossaire des termes introduits

Modulo (%) — reste d'une division entière (parité, cycles).

and / or / not — opérateurs logiques en GDScript (court-circuités).

clamp — borne une valeur entre un minimum et un maximum.

is_equal_approx — compare deux floats en tenant compte de l'imprécision.
