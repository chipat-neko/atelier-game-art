GD09 — Nodes, scènes & instanciation

Intermédiaire 45 min Piste 9 · GDScript (Godot)
PrérequisGD7 et GD8.
🎯Objectifs d'apprentissage

À la fin de cette leçon, tu sauras :

  • Comprendre l'arbre de nodes et les chemins (NodePath).
  • Référencer un node avec $ / get_node et @onready.
  • Instancier une scène (PackedScene) et l'ajouter avec add_child.
  • Supprimer un node proprement avec queue_free.

Pourquoi c'est important

Tout dans Godot est node et scène. Faire communiquer un script avec les autres nodes (récupérer le sprite, la barre de vie) et faire apparaître des objets (un ennemi, un projectile) est la base du gameplay. Une scène réutilisable est l'équivalent du prefab d'Unity.

Théorie en profondeur

1. L'arbre de nodes & les chemins

Une scène est un arbre. Depuis un script, on désigne un autre node par son chemin relatif :

var barre = get_node("UI/BarreVie")   # chemin depuis ce node
var barre2 = $UI/BarreVie             # $ = raccourci de get_node
var parent = get_parent()             # le node parent

Le $ est la forme courte et lisible. Les chemins suivent la hiérarchie de l'éditeur (comme des dossiers).

2. @onready : récupérer une référence au bon moment

On ne peut référencer un enfant qu'une fois la scène prête. L'annotation @onready assigne la variable juste avant _ready, proprement :

@onready var sprite: Sprite2D = $Sprite2D
@onready var barre_vie = $UI/BarreVie

func _ready():
    sprite.modulate = Color.RED       # déjà assigné, prêt à l'emploi

3. Instancier une scène

On crée une scène réutilisable (ex. : ennemi.tscn), on la charge puis on l'instancie à l'exécution :

const ENNEMI = preload("res://ennemi.tscn")   # charge à la compilation (rapide)

func faire_apparaitre():
    var e = ENNEMI.instantiate()              # crée une instance de la scène
    e.position = Vector2(100, 50)
    add_child(e)                              # l'ajoute à l'arbre (sinon invisible)

preload charge la ressource au démarrage (rapide) ; load charge à l'exécution. instantiate() crée une copie, et add_child l'insère dans la scène.

4. Supprimer un node : queue_free

queue_free()              # supprime CE node en fin de frame (sûr)
mon_ennemi.queue_free()   # supprime un autre node

Utilise queue_free() plutôt que free() : la suppression est différée en fin de frame, ce qui évite les crashs si le node est encore utilisé.

🔧 Démonstration pas à pas

  1. Référence un enfant Sprite2D avec @onready var sprite = $Sprite2D et change sa couleur.
  2. Crée une scène ennemi.tscn, fais const ENNEMI = preload(...).
  3. Dans une fonction, instantiate() l'ennemi, place-le et add_child(e).
  4. Détruis-le plus tard avec queue_free().

⚠️ Pièges & erreurs courantes

SymptômeL'objet instancié n'apparaît pas.
Causeadd_child oublié : il n'est pas dans l'arbre.
CorrectionAppelle add_child(instance) après instantiate().
Symptôme« Node not found: UI/BarreVie ».
CauseChemin incorrect ou node renommé.
CorrectionVérifie la hiérarchie ; glisse le node dans le script pour générer le chemin.
Symptôme$Sprite vaut null dans _ready.
CauseRéférence prise trop tôt (avant que l'enfant existe).
CorrectionUtilise @onready pour assigner au bon moment.
SymptômeCrash après suppression d'un node.
Causefree() immédiat alors que le node est encore utilisé.
CorrectionUtilise queue_free() (suppression différée).

🔧Exercice pratique

  1. Un spawner qui instancie une scène d'ennemi et l'ajoute à l'arbre.
  2. Une référence @onready à une barre de vie mise à jour au Start.
  3. Auto-destruction de l'ennemi via queue_free() après une action.
Critères de réussite
  • L'instanciation et l'ajout à l'arbre fonctionnent.
  • Les références passent par @onready + $.
  • La suppression utilise queue_free().

📌À retenir
  • Les nodes forment un arbre ; on les référence par chemin avec $ / get_node.
  • @onready assigne une référence juste avant _ready (sûr).
  • preload + instantiate() + add_child() pour faire apparaître une scène.
  • queue_free() pour supprimer proprement (différé).

Glossaire des termes introduits

NodePath / $ — chemin désignant un node dans l'arbre ; $ = raccourci de get_node.

@onready — assigne une variable juste avant _ready.

PackedScene — scène sauvegardée (.tscn) réutilisable par instanciation (≈ prefab).

instantiate / add_child / queue_free — créer une instance / l'ajouter à l'arbre / la supprimer.
