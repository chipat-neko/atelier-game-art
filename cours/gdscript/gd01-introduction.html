GD01 — Introduction à GDScript & premier script

Débutant 35 min Piste 9 · GDScript (Godot)
Prérequisaucune expérience de la programmation requise. Avoir installé Godot aide (voir la piste Godot).
🎯Objectifs d'apprentissage

À la fin de cette leçon, tu sauras :

  • Ce qu'est GDScript et pourquoi il est idéal pour Godot.
  • Attacher un script à un node.
  • Utiliser _ready() et print().
  • Comprendre le cycle éditer → jouer et l'importance de l'indentation.

Pourquoi c'est important

GDScript est le langage natif de Godot : conçu pour le moteur, proche de Python (lisible, sans accolades), il s'apprend vite et s'intègre parfaitement aux nodes et aux scènes. C'est la porte d'entrée idéale vers la programmation de jeu : peu de cérémonie, résultat immédiat.

ℹ️GDScript ou C# ?

Godot supporte aussi le C# (.NET) et le C++ (GDExtension). Mais GDScript est le plus simple, le plus intégré et le mieux documenté pour débuter — c'est celui qu'on apprend ici. Les concepts (variables, POO, signaux) se transposent ensuite à n'importe quel langage.

Théorie en profondeur

1. Script = comportement attaché à un node

Dans Godot, une scène est un arbre de nodes (rappel G2). Un script ajoute du comportement à un node : clic droit sur un node ▸ Attach Script. Godot génère un squelette :

extends Node          # ce script ÉTEND le node auquel il est attaché

# Appelé une fois, quand le node entre dans la scène
func _ready():
    print("Bonjour depuis GDScript !")

# Appelé à chaque frame (delta = temps écoulé depuis la dernière frame)
func _process(delta):
    pass              # 'pass' = ne rien faire (placeholder)

Le mot-clé extends indique de quel type de node ton script hérite (ici Node) : ton script devient ce node, avec tous ses pouvoirs. On approfondira l'héritage en GD8.

Un fichier GDScript porte l'extension .gd (par exemple joueur.gd) : c'est un simple fichier texte, versionnable, que Godot associe au node auquel tu l'attaches.

2. print() : ton premier outil

print() écrit dans le panneau Output en bas de l'éditeur — l'équivalent du Debug.Log d'Unity :

print("Score : ", 42)            # affiche : Score : 42
print("PV restants : %d" % pv)   # formatage avec %d (entier), %s (texte), %f (flottant)

3. L'indentation fait la structure

Point crucial venant de Python : en GDScript, l'indentation définit les blocs (pas d'accolades). Tout ce qui est indenté sous func _ready(): appartient à cette fonction. Utilise des tabulations (ou des espaces) de façon cohérente :

func _ready():
    var nom = "Héros"        # indenté → DANS la fonction
    print(nom)               # indenté → DANS la fonction
# cette ligne n'est plus indentée → HORS de la fonction

4. Le cycle éditer → jouer

GDScript est interprété par le moteur : pas de compilation longue. Tu modifies le script, tu appuies sur F5 (jouer le projet) ou F6 (jouer la scène courante), et le résultat apparaît immédiatement. Cette boucle rapide est l'un des grands plaisirs de Godot.

🔧 Démonstration pas à pas

  1. Crée une scène avec un node racine Node (ou Node2D).
  2. Clic droit ▸ Attach Script, garde le nom proposé, clique sur Create.
  3. Dans _ready(), ajoute print("Mon premier script !").
  4. Appuie sur F6 pour jouer la scène et observe le message dans le panneau Output.

⚠️ Pièges & erreurs courantes

Symptôme« Indentation error » / « Expected indented block ».
CauseMélange d'espaces et de tabulations, ou bloc non indenté.
CorrectionIndente de façon cohérente sous chaque func ...:.
SymptômeMon script ne fait rien.
CauseScript non attaché à un node présent dans la scène jouée.
CorrectionVérifie l'icône de script sur le node et joue la bonne scène.
Symptôme« Function _ready expects 0 arguments ».
CauseMauvaise signature d'une fonction du moteur.
CorrectionRespecte les signatures : _ready(), _process(delta).
SymptômeRien dans l'Output.
CauseLe panneau Output est masqué.
CorrectionOuvre l'onglet Output en bas de l'éditeur.

🔧Exercice pratique

  1. Attache un script à un node et affiche ton prénom dans _ready().
  2. Ajoute un _process(delta) qui affiche delta (tu verras des petits nombres défiler).
  3. Formate un message avec %d pour afficher un score fictif.
Critères de réussite
  • Le script s'exécute sans erreur d'indentation.
  • Les messages apparaissent dans l'Output au bon moment.
  • Tu sais relancer la scène avec F6.

📌À retenir
  • GDScript est le langage natif de Godot, proche de Python, interprété (itération immédiate).
  • Un script extends un node et lui ajoute du comportement.
  • _ready() au démarrage, _process(delta) chaque frame ; print() pour déboguer.
  • L'indentation définit les blocs : sois cohérent.

Glossaire des termes introduits

GDScript — langage de script natif de Godot, proche de Python.

Node — brique de base d'une scène Godot, à laquelle on attache un script.

extends — mot-clé indiquant le type de node dont le script hérite.

_ready / _process — fonctions appelées au démarrage / à chaque frame.
