GD08 — Classes & objets (POO)

Intermédiaire 45 min Piste 9 · GDScript (Godot)
PrérequisGD6 — Fonctions.
🎯Objectifs d'apprentissage

À la fin de cette leçon, tu sauras :

  • Comprendre qu'un script EST une classe (extends).
  • Nommer une classe avec class_name pour la réutiliser.
  • Hériter, redéfinir une fonction et appeler super.
  • Initialiser un objet avec _init et créer des classes internes.

Pourquoi c'est important

La POO structure le code autour d'objets qui regroupent données et comportements. Dans Godot, chaque script est déjà une classe : comprendre la POO te permet de créer tes propres types (un Ennemi, une Arme) et de réutiliser du comportement par héritage.

Théorie en profondeur

1. Un script EST une classe

Le extends du haut indique la classe parente. Les var de premier niveau sont des membres (propriétés), les func sont des méthodes :

extends Node

var pv: int = 100
var nom: String = "Héros"

func subir_degats(montant: int) -> void:
    pv -= montant
    if pv < 0:
        pv = 0

2. class_name : nommer sa classe

class_name enregistre ta classe globalement : elle apparaît dans la liste des nodes/types et peut être instanciée par son nom :

class_name Ennemi
extends CharacterBody2D

var pv: int = 30

func _init():            # constructeur : appelé à la création de l'objet
    print("Un ennemi est né")

Ailleurs : var e = Ennemi.new() crée une instance (pour les objets non-node) ; pour un node, on instancie plutôt sa scène (GD9).

3. Héritage & super

Une classe peut étendre une de tes classes et la spécialiser. super appelle la version parente d'une méthode :

class_name Boss
extends Ennemi              # Boss EST un Ennemi (avec ses pv, etc.)

func _init():
    super()                # appelle Ennemi._init()
    pv = 200               # le boss a plus de vie

func subir_degats(montant: int) -> void:
    super(montant / 2)     # le boss ne subit que la moitié des dégâts

4. Encapsulation à la Godot

GDScript n'a pas de private strict : la convention est de préfixer les membres internes d'un underscore (_compteur) pour signaler « ne pas toucher de l'extérieur ». On expose plutôt des méthodes claires.

5. Classes internes

class Stats:                # classe interne (dans le même fichier)
    var force: int = 10
    var agilite: int = 5

func _ready():
    var s = Stats.new()
    print(s.force)

🔧 Démonstration pas à pas

  1. Crée une classe Ennemi (class_name) avec pv et subir_degats.
  2. Crée Boss extends Ennemi qui double les PV dans _init via super().
  3. Redéfinis subir_degats pour halver les dégâts, en appelant super(...).
  4. Préfixe un membre interne d'un _ pour marquer l'intention « privé ».

🔧Exercice pratique

  1. Une classe Arme (class_name) avec degats et tirer().
  2. Une classe Fusil extends Arme qui spécialise tirer() avec super().
  3. Une classe interne Munition avec quelques membres.
Critères de réussite
  • class_name et extends corrects.
  • L'override appelle super quand nécessaire.
  • Les membres internes sont marqués par convention.

Glossaire des termes introduits

class_name — nomme une classe GDScript et l'enregistre globalement.

_init — constructeur appelé à la création d'un objet.

super — appelle la version parente d'une méthode.

Classe interne — classe définie dans le même fichier (class Nom:).
