GD10 — Signaux & communication

Avancé 45 min Piste 9 · GDScript (Godot)
PrérequisGD8 et GD9.
🎯Objectifs d'apprentissage

À la fin de cette leçon, tu sauras :

  • Comprendre le couplage et pourquoi le réduire.
  • Déclarer un signal et l'émettre.
  • Connecter un signal dans l'éditeur ou par code.
  • Appliquer le patron observateur pour découpler.

Pourquoi c'est important

Quand le joueur meurt, l'UI, le son et le score doivent réagir. Si le node Sante appelle directement chacun, il devient couplé à tout le projet. Les signaux sont la solution native de Godot : Sante émet « je suis mort », et qui veut réagir s'y connecte. C'est le patron observateur, équivalent des events C# (CS13) et des delegates Unreal (CPP13).

Théorie en profondeur

1. Déclarer & émettre un signal

extends Node

signal mort                         # signal sans donnée
signal vie_changee(nouvelle_vie)    # signal qui transporte une valeur

var pv: int = 100

func subir(degats: int) -> void:
    pv = max(0, pv - degats)
    vie_changee.emit(pv)            # émet le signal vers tous les connectés
    if pv == 0:
        mort.emit()

2. Connecter par code

Un autre node s'abonne avec .connect() en passant la fonction à appeler (un Callable) :

extends Control

@onready var sante = $"../Joueur/Sante"

func _ready():
    sante.vie_changee.connect(_on_vie_changee)   # s'abonner
    sante.mort.connect(_on_mort)

func _on_vie_changee(nouvelle_vie: int) -> void:
    print("Barre de vie : ", nouvelle_vie)

func _on_mort() -> void:
    print("GAME OVER")

Convention : nommer la fonction réceptrice _on_<signal>.

3. Connecter dans l'éditeur

Godot permet aussi de connecter visuellement : sélectionne le node émetteur ▸ onglet Node ▸ Signals ▸ double-clic sur le signal ▸ choisis le node récepteur. Godot génère la fonction _on_.... Idéal pour les boutons d'UI (pressed) et les zones (body_entered).

4. Pourquoi c'est puissant

L'émetteur ne connaît pas ses abonnés : ajouter une réaction (un son, un effet) ne modifie pas le node Sante. C'est la clé d'un code maintenable. Beaucoup de nodes Godot émettent déjà des signaux intégrés : Button.pressed, Area2D.body_entered, Timer.timeout…

🔧 Démonstration pas à pas

  1. Dans un node Sante, déclare signal vie_changee(v) et signal mort, et émets-les dans subir().
  2. Dans l'UI, connecte-toi par code à vie_changee et mets à jour un affichage.
  3. Ajoute un node Audio qui se connecte aussi à mort — sans toucher à Sante.
  4. Connecte un Button.pressed dans l'éditeur pour comparer les deux méthodes.

⚠️ Pièges & erreurs courantes

SymptômeRien ne se passe à l'émission.
CauseAucun node connecté au signal.
CorrectionConnecte le signal (par code ou éditeur).
Symptôme« Signal is already connected ».
CauseConnexion faite deux fois (code + éditeur, ou _ready rappelé).
CorrectionConnecte une seule fois ; vérifie avec is_connected si besoin.
Symptôme« Nonexistent function _on_... ».
CauseLa fonction réceptrice n'existe pas / mauvaise signature.
CorrectionCrée la fonction avec les bons paramètres du signal.
SymptômeCrash après destruction d'un node connecté.
CauseRécepteur détruit alors que le signal s'émet encore.
CorrectionGodot nettoie en général à queue_free ; évite d'émettre vers un node libéré.

🔧Exercice pratique

  1. Un node Sante émettant vie_changee(int) et mort.
  2. Une UI qui se connecte par code et met à jour une barre de vie.
  3. Un 2ᵉ abonné (audio/score) ajouté sans modifier Sante.
Critères de réussite
  • L'émetteur ne référence aucun abonné.
  • Les connexions sont faites une seule fois.
  • Les fonctions réceptrices ont la bonne signature.

📌À retenir
  • Les signaux réduisent le couplage : l'émetteur emit, les abonnés réagissent.
  • signal nom(params) ; émettre avec nom.emit(...).
  • Connecter par code (signal.connect(fonction)) ou dans l'éditeur (onglet Node ▸ Signals).
  • Beaucoup de nodes émettent des signaux intégrés (pressed, body_entered, timeout).

Glossaire des termes introduits

signal — annonce qu'un node émet ; les autres s'y connectent.

emit — déclenche un signal vers tous les connectés.

connect — abonne une fonction (Callable) à un signal.

Observateur — patron où des abonnés réagissent à un émetteur sans qu'il les connaisse.
