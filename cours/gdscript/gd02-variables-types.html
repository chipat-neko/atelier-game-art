GD02 — Variables & types

Débutant 35 min Piste 9 · GDScript (Godot)
PrérequisGD1 — Introduction.
🎯Objectifs d'apprentissage

À la fin de cette leçon, tu sauras :

  • Déclarer une variable avec var et une constante avec const.
  • Connaître les types de base : int, float, bool, String, Vector2.
  • Utiliser le typage statique optionnel et l'inférence.
  • Exposer une variable dans l'éditeur avec @export.

Pourquoi c'est important

Les variables stockent l'état du jeu : vie, score, vitesse, position. GDScript est dynamiquement typé par défaut (souple) mais permet un typage statique optionnel qui rend le code plus sûr et plus rapide — une force à adopter dès le début.

Théorie en profondeur

1. var et const

var score = 0                # variable : peut changer
var nom = "Héros"
var en_vie = true

const VIE_MAX = 100          # constante : ne changera jamais (MAJUSCULES par convention)

2. Les types de base

TypePourExemple
intEntiervar pv = 100
floatDécimalvar vitesse = 300.0
boolVrai/fauxvar en_vie = true
StringTextevar nom = "Boss"
Vector2Position 2D (x, y)var pos = Vector2(10, 5)

Pour la 3D, Vector3 (x, y, z). Ces types vecteur sont au cœur du gameplay (positions, directions, vitesses).

3. Typage statique optionnel

On peut préciser le type avec : Type. Le moteur vérifie alors la cohérence (erreur si tu mets un texte dans un int) et le code tourne plus vite :

var pv: int = 100            # typé explicitement
var vitesse: float = 300.0
var nom: String = "Héros"

var degats := 25            # ':=' : INFÉRENCE — le type (int) est déduit de la valeur

Le := (« deux-points égal ») laisse Godot déduire le type tout en le verrouillant. Recommandation : type tes variables — on y reviendra en GD15.

4. Exposer une variable : @export

L'annotation @export rend une variable réglable dans l'Inspector de Godot, sans toucher au code : parfait pour ajuster une vitesse, une couleur, etc. (détaillé en GD13) :

extends Node

@export var vitesse: float = 300.0   # apparaît dans l'Inspector
@export var nom_ennemi: String = "Slime"

func _ready():
    print("Vitesse réglée : ", vitesse)

5. Conversion & affichage

var texte = str(score)       # convertir en String
var nombre = int("42")       # convertir en int
var f = float(3)             # 3.0
print("PV : %d / %d" % [pv, VIE_MAX])   # formatage avec une liste de valeurs

🔧 Démonstration pas à pas

  1. Déclare des variables typées pour un personnage : nom: String, pv: int, vitesse: float, en_vie: bool.
  2. Ajoute const VIE_MAX = 100 et tente de la modifier : observe l'erreur.
  3. Expose vitesse avec @export et règle-la dans l'Inspector.
  4. Affiche une fiche récap formatée avec %.

⚠️ Pièges & erreurs courantes

SymptômeDivision qui « perd » les décimales.
CauseDeux int : la division est entière (voir GD3).
CorrectionUtilise un float (100.0).
Symptôme« Invalid operands String and int ».
CauseConcaténer texte et nombre sans conversion.
CorrectionConvertis avec str(n) ou utilise %.
SymptômeMa variable @export n'apparaît pas.
CauseAnnotation mal écrite ou script non rechargé.
CorrectionÉcris @export var ... en début de classe ; resélectionne le node.
SymptômeType incohérent non détecté.
CauseVariable non typée (dynamique).
CorrectionAjoute : Type pour activer la vérification.

🔧Exercice pratique

  1. Crée des variables typées pour un ennemi (nom, pv, vitesse, agressif).
  2. Expose deux d'entre elles avec @export et règle-les dans l'Inspector.
  3. Calcule et affiche le pourcentage de vie (attention aux types !).
Critères de réussite
  • Toutes les variables sont typées et initialisées.
  • Les @export apparaissent et fonctionnent dans l'Inspector.
  • Le pourcentage est calculé en flottant.

Exercice 3Défi

Calcule le pourcentage de vie en flottant et affiche-le formaté.

Voir une solution 
var pv := 73
const VIE_MAX := 100
var pourcent := roundi(float(pv) / VIE_MAX * 100)
print("Vie : %d %%" % pourcent)   # Vie : 73 %
📌À retenir
  • var pour une variable, const pour une constante (MAJUSCULES).
  • Types clés : int, float, bool, String, Vector2/Vector3.
  • Type tes variables (: Type ou inférence :=) : plus sûr et plus rapide.
  • @export expose une variable dans l'Inspector.

Glossaire des termes introduits

Typage statique optionnel — préciser le type d'une variable (: int) pour plus de sûreté/perf.

Inférence (:=) — Godot déduit et verrouille le type à partir de la valeur.

@export — annotation exposant une variable dans l'Inspector.

Vector2 / Vector3 — types de position/direction en 2D / 3D.
