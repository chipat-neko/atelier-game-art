GD11 — Collections (Array, Dictionary)

Intermédiaire 40 min Piste 9 · GDScript (Godot)
PrérequisGD5 — Boucles et GD6.
🎯Objectifs d'apprentissage

À la fin de cette leçon, tu sauras :

  • Utiliser un Array : ajouter, parcourir, retirer.
  • Typer un tableau (Array[int]) pour plus de sûreté.
  • Utiliser un Dictionary (clé → valeur).
  • Choisir la bonne collection selon le besoin.

Pourquoi c'est important

Un jeu manipule des collections partout : la liste des ennemis, l'inventaire, les scores par joueur. GDScript propose l'Array (liste) et le Dictionary (table clé-valeur) — équivalents des List/Dictionary de C# (CS10).

Théorie en profondeur

1. Array : la liste

var ennemis = []                 # tableau vide
ennemis.append(nouvel_ennemi)    # ajouter à la fin
var n = ennemis.size()           # nombre d'éléments
var premier = ennemis[0]         # accès par index (attention aux bornes !)

ennemis.erase(mort)              # retirer par valeur
ennemis.remove_at(2)             # retirer par index
var present = cible in ennemis   # test d'appartenance

for e in ennemis:                # parcours (GD5)
    if is_instance_valid(e):
        pass

2. Tableaux typés

Comme pour les variables, on peut typer un tableau : Godot refuse alors les éléments du mauvais type :

var scores: Array[int] = [100, 75, 50]
var noms: Array[String] = []
noms.append("Alice")
# noms.append(42)   # ERREUR : 42 n'est pas un String

3. Dictionary : clé → valeur

var scores_par_joueur = {
    "Alice": 100,
    "Bob": 75
}

scores_par_joueur["Carla"] = 60          # ajouter / modifier
var s = scores_par_joueur.get("Alice", 0) # lire avec valeur par défaut si absent
var existe = scores_par_joueur.has("Bob") # tester une clé

for nom in scores_par_joueur:             # parcourt les CLÉS
    print(nom, " = ", scores_par_joueur[nom])

4. Quelques méthodes utiles

ennemis.is_empty()        # vide ?
ennemis.clear()           # tout vider
ennemis.sort()            # trier
ennemis.has(cible)        # contient ?
ennemis.pick_random()     # élément au hasard

5. Choisir le bon conteneur

BesoinConteneurPourquoi
Liste ordonnée, parcourue souventArrayAccès par index, ajout en fin efficace.
Retrouver par clé (nom, id)DictionaryRecherche rapide clé → valeur.
Données homogènes sûresArray[T]Type vérifié, moins de bugs.

🔧 Démonstration pas à pas

  1. Un Array d'ennemis : ajoute-en, parcours-les, retire les invalides.
  2. Un tableau typé Array[int] de scores et tente d'y mettre un texte (observe l'erreur).
  3. Un Dictionary « nom → score » : ajoute, lis avec get, parcours les clés.
  4. Utilise pick_random() pour choisir une cible au hasard.

⚠️ Pièges & erreurs courantes

Symptôme« Index out of bounds ».
CauseAccès à un index ≥ size() ou tableau vide.
CorrectionVérifie size() / is_empty() avant l'accès.
Symptôme« Invalid key in dictionary ».
CauseAccès direct à une clé absente.
CorrectionUtilise has(cle) ou get(cle, defaut).
SymptômeErreur en retirant pendant un parcours.
CauseModification du tableau pendant l'itération.
CorrectionItère sur une copie (.duplicate()) ou par index à l'envers.
SymptômeUn tableau accepte n'importe quoi.
CauseTableau non typé.
CorrectionDéclare Array[Type] pour la vérification.

🔧Exercice pratique

  1. Un inventaire en Array[String] : ajouter/retirer et afficher le nombre.
  2. Un Dictionary « objet → quantité » avec mise à jour et recherche.
  3. Choisis un ennemi au hasard avec pick_random().
Critères de réussite
  • Aucun accès hors bornes ni clé absente non gérée.
  • Les tableaux homogènes sont typés.
  • Le bon conteneur est choisi pour chaque besoin.
📌À retenir
  • Array (liste indexée) et Dictionary (clé→valeur) couvrent l'essentiel.
  • Type tes tableaux (Array[int]) pour la sûreté.
  • Vérifie bornes (size) et clés (has/get) avant l'accès.
  • Parcours avec for x in collection ; n'modifie pas pendant l'itération.

Glossaire des termes introduits

Array — liste dynamique indexée (append, size, erase…).

Array[T] — tableau typé : éléments d'un type vérifié.

Dictionary — table associant des clés à des valeurs.

has / get — tester une clé / la lire avec une valeur par défaut.
