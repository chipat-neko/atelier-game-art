GD05 — Boucles (for, while, range)

Débutant 35 min Piste 9 · GDScript (Godot)
PrérequisGD4 — Conditions.
🎯Objectifs d'apprentissage

À la fin de cette leçon, tu sauras :

  • Répéter avec for et range.
  • Parcourir un tableau avec for x in tableau.
  • Utiliser while, break et continue.
  • Comprendre pourquoi une boucle ne doit pas bloquer une frame.

Pourquoi c'est important

Traiter « tous les ennemis », « chaque case d'une grille » ou répéter une action N fois : les boucles sont l'outil de la répétition.

Théorie en profondeur

1. for avec range

range(n) génère les nombres de 0 à n-1 :

for i in range(5):           # 0, 1, 2, 3, 4
    print("Spawn n°", i)

for i in range(1, 4):        # 1, 2, 3 (début, fin exclue)
    print(i)

for i in range(0, 10, 2):    # 0, 2, 4, 6, 8 (pas de 2)
    print(i)

2. Parcourir un tableau (le plus utile en jeu)

for ennemi in ennemis:       # "pour chaque ennemi dans ennemis"
    if is_instance_valid(ennemi):
        ennemi.queue_free()

for nom in ["Alice", "Bob"]:
    print(nom)

Pas d'index à gérer : GDScript te donne directement chaque élément. (Les tableaux sont détaillés en GD11.)

3. while

while munitions > 0:
    tirer()
    munitions -= 1

4. break & continue

for a in acteurs:
    if not is_instance_valid(a):
        continue             # passe au tour suivant
    if a.is_in_group("boss"):
        cible = a
        break                # sort de la boucle
🛑Ne bloque jamais une frame

Une boucle très longue ou while true sans sortie s'exécute entièrement dans la même frame et gèle le jeu. Pour étaler dans le temps, on utilise await et les timers (GD14), pas une boucle qui attend.

🔧 Démonstration pas à pas

  1. Avec range, logge les nombres de 1 à 5.
  2. Avec while, vide un stock de munitions en tirant.
  3. Parcours un tableau d'ennemis et détruis chaque élément valide.
  4. Cherche le premier acteur du groupe « boss » avec break, en sautant les invalides par continue.

⚠️ Pièges & erreurs courantes

CauseBoucle infinie ou condition de sortie jamais atteinte.
CorrectionGarantis la progression vers la sortie ; pour le temps, await/timer.
SymptômeErreur en modifiant un tableau pendant le parcours.

🔧Exercice pratique

  1. Une boucle for qui calcule la somme des entiers de 1 à 100.
  2. Parcours un tableau de PV et compte combien sont « morts » (≤ 0).
  3. Trouve le premier ennemi à portée avec break et saute les invalides avec continue.
Critères de réussite
  • Aucune boucle ne bloque la frame.
  • Le parcours utilise for x in collection.
  • Pas d'erreur de borne ni de modification du tableau pendant l'itération.
📌À retenir
  • for i in range(n) pour un nombre connu ; range(début, fin, pas).
  • for x in collection pour parcourir chaque élément (idiomatique).
  • while tant que vrai ; break sort, continue passe au tour suivant.
  • Ne bloque jamais une frame : pour le temps, await/timers, pas une boucle d'attente.

Glossaire des termes introduits

range — génère une suite d'entiers (début, fin exclue, pas).

for x in collection — boucle visitant chaque élément d'un tableau/dictionnaire.

break / continue — sortir de la boucle / passer au tour suivant.

is_instance_valid — teste qu'un objet existe encore avant de l'utiliser.
