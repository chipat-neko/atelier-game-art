GD12 — Entrées & mouvement

Intermédiaire 45 min Piste 9 · GDScript (Godot)
PrérequisGD7 et GD9.
🎯Objectifs d'apprentissage

À la fin de cette leçon, tu sauras :

  • Définir des actions dans l'Input Map et les lire avec Input.
  • Distinguer « action maintenue » et « action ponctuelle ».
  • Déplacer un CharacterBody2D avec velocity et move_and_slide.
  • Choisir entre _process, _physics_process et _input.

Pourquoi c'est important

Contrôler un personnage, c'est le premier « ça marche ! » d'un débutant. Godot propose un système d'actions propre : tu nommes une intention (« sauter ») et tu y associes des touches — le code lit l'action, pas la touche physique.

Théorie en profondeur

1. L'Input Map : nommer les actions

Dans Project Settings ▸ Input Map, crée des actions (move_left, move_right, saut…) et associe-leur des touches. Avantages : remapping facile, support clavier/manette, code lisible. Godot fournit déjà ui_left, ui_right, etc.

2. Lire les entrées

if Input.is_action_pressed("move_right"):     # VRAI tant que maintenue
    pass
if Input.is_action_just_pressed("saut"):      # VRAI seulement à l'instant de l'appui
    pass

# Axe pratique : renvoie un vecteur normalisé selon 4 actions
var direction = Input.get_vector("move_left", "move_right", "move_up", "move_down")

Distinction clé : is_action_pressed (continu, pour le déplacement) vs is_action_just_pressed (instant, pour sauter/tirer). get_vector donne directement une direction prête à l'emploi.

3. Déplacer un CharacterBody2D

Pour un personnage qui respecte les collisions, on utilise un CharacterBody2D, sa velocity et move_and_slide() — dans _physics_process (GD7) :

extends CharacterBody2D

@export var vitesse: float = 300.0

func _physics_process(delta):
    var direction = Input.get_vector("move_left", "move_right", "move_up", "move_down")
    velocity = direction * vitesse
    move_and_slide()             # déplace en gérant les collisions (utilise velocity)

move_and_slide() applique velocity en glissant le long des murs. Note : il intègre déjà delta en interne — pas besoin de multiplier velocity par delta ici.

4. Un saut (plateforme 2D)

@export var force_saut: float = -400.0   # vers le haut (y négatif en 2D)
var gravite: float = 980.0

func _physics_process(delta):
    if not is_on_floor():
        velocity.y += gravite * delta
    if Input.is_action_just_pressed("saut") and is_on_floor():
        velocity.y = force_saut
    velocity.x = Input.get_axis("move_left", "move_right") * vitesse
    move_and_slide()

🔧 Démonstration pas à pas

  1. Crée les actions move_left/right/up/down dans l'Input Map.
  2. Sur un CharacterBody2D, lis get_vector, règle velocity et appelle move_and_slide() dans _physics_process.
  3. Ajoute un mur (collision) et vérifie que le personnage glisse dessus.
  4. Variante plateforme : ajoute gravité et saut avec is_action_just_pressed + is_on_floor().

⚠️ Pièges & erreurs courantes

Symptôme« Action move_right not found ».
CauseAction non définie dans l'Input Map.
CorrectionCrée l'action dans Project Settings ▸ Input Map.
SymptômeLe personnage traverse les murs.
CauseDéplacement via position au lieu de move_and_slide.
CorrectionUtilise un CharacterBody2D + colliders + move_and_slide().
SymptômeMon saut se déclenche plusieurs fois.
Causeis_action_pressed (continu) au lieu de just_pressed.
CorrectionUtilise is_action_just_pressed pour les actions ponctuelles.
SymptômeMouvement saccadé.
CauseCode de déplacement dans _process.
CorrectionMets-le dans _physics_process.

🔧Exercice pratique

  1. Déplace un CharacterBody2D dans 4 directions à vitesse réglable.
  2. Ajoute des murs et vérifie le glissement (collisions).
  3. Variante plateforme : gravité + saut au sol uniquement.
Critères de réussite
  • Le déplacement utilise des actions (Input Map), pas des touches en dur.
  • Le code de mouvement est dans _physics_process.
  • Le saut se déclenche une seule fois et seulement au sol.

📌À retenir
  • Définis des actions dans l'Input Map ; lis-les avec Input (jamais la touche en dur).
  • is_action_pressed (maintenu) vs is_action_just_pressed (instant) ; get_vector pour une direction.
  • CharacterBody2D + velocity + move_and_slide() dans _physics_process.
  • move_and_slide intègre delta : ne le multiplie pas en plus.

Glossaire des termes introduits

Input Map — table associant des actions nommées à des touches/boutons.

Input — classe lisant l'état des actions (pressed, just_pressed, get_vector).

CharacterBody2D — corps contrôlable gérant les collisions via move_and_slide.

velocity / move_and_slide — vélocité du corps / déplacement avec glissement sur les obstacles.
