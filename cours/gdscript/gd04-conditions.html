GD04 — Conditions (if, match)

Débutant 35 min Piste 9 · GDScript (Godot)
PrérequisGD3 — Opérateurs.
🎯Objectifs d'apprentissage

À la fin de cette leçon, tu sauras :

  • Brancher le code avec if / elif / else.
  • Respecter l'indentation des blocs.
  • Utiliser l'opérateur ternaire (... if ... else ...).
  • Choisir entre plusieurs cas avec match.

Pourquoi c'est important

Un jeu décide en permanence : « si la vie atteint 0, mourir », « selon l'état de l'IA, patrouiller ou attaquer ». Les conditions sont le mécanisme de base de toute logique.

Théorie en profondeur

1. if / elif / else

if pv <= 0:
    mourir()
elif pv < 30:
    afficher_alerte_vie()    # entre 1 et 29
else:
    pass                     # tout va bien

Noter le deux-points : en fin de ligne et l'indentation du bloc (pas d'accolades). C'est elif (pas « else if »).

2. L'opérateur ternaire

GDScript écrit le ternaire « à la Python » : la valeur d'abord, puis la condition :

var etat = "Vivant" if pv > 0 else "Mort"
var vitesse = 600.0 if court else 300.0

3. match : choisir parmi des cas

match est l'équivalent du switch, mais plus puissant et sans piège de break (chaque cas est isolé) :

match etat_ia:
    "patrouille":
        patrouiller()
    "poursuite":
        poursuivre()
    "attaque":
        attaquer()
    _:                       # '_' = cas par défaut (tous les autres)
        attendre()

match peut aussi tester des motifs (plages, plusieurs valeurs avec ,, tableaux) :

match degats:
    0:
        print("aucun dégât")
    1, 2, 3:
        print("dégâts légers")    # plusieurs valeurs
    _:
        print("gros dégâts")

🔧 Démonstration pas à pas

  1. Une fonction evaluer_vie(pv) qui affiche « Mort », « Critique » ou « OK » selon des seuils.
  2. Un ternaire qui choisit une vitesse selon un booléen court.
  3. Un match sur une chaîne d'état d'IA qui déclenche l'action correspondante (avec _).

⚠️ Pièges & erreurs courantes

Symptôme« Expected indented block ».
CauseBloc non indenté après le :.
CorrectionIndente le corps du if/elif/else.
Symptôme« else if » refusé.
CauseGDScript utilise elif, pas « else if ».
CorrectionÉcris elif condition:.
SymptômeUn cas non prévu n'est pas géré.
CausePas de cas par défaut dans le match.

SymptômeLa condition est toujours vraie/fausse.
CauseConfusion entre = (affectation) et == (comparaison).
CorrectionUtilise == dans les conditions.

🔧Exercice pratique

  1. Un « rang » selon le score (Bronze/Argent/Or) via if/elif/else.
  2. Un match sur un état d'IA qui déclenche l'action correspondante.
  3. Un ternaire qui choisit un message court selon que le joueur est vivant.
Critères de réussite
  • Indentation correcte, elif bien utilisé.
  • Le match a un cas par défaut _.
  • Aucune confusion =/==.

📌À retenir

  • Ternaire « à la Python » : valeur_si_vrai if cond else valeur_si_faux.

Glossaire des termes introduits

elif — « sinon si » de GDScript.

match — branchement multi-cas par motif, sans break.

_ (motif joker) — cas par défaut d'un match.
