LUA13 — Intégration dans un moteur

Avancé 45 min Piste 11 · Lua
PrérequisLUA12 — Le scripting de jeu, LUA9 — Modules.
🎯Objectifs d'apprentissage

À la fin de cette leçon, tu sauras :

  • Ce que signifie « Lua embarqué » et le pont moteur ↔ Lua.
  • La structure d'un projet Roblox (Luau, Instances, services, :Connect).
  • La structure d'un jeu LÖVE (love.load/update/draw).
  • Le principe de Defold et des autres hôtes Lua.

Pourquoi c'est important

Lua est rarement utilisé seul : il est embarqué dans un moteur écrit en C/C++. Le moteur expose une API (dessin, physique, audio) que tu pilotes depuis Lua. Chaque hôte a ses conventions ; les reconnaître t'évite de réapprendre à zéro.

Théorie en profondeur

1. Le pont moteur ↔ Lua

Le moteur (C++) appelle ton Lua au bon moment, et ton Lua appelle l'API du moteur. Les objets du moteur (un sprite, un Part) arrivent côté Lua sous forme de tables/userdata munies de méthodes : tu manipules des références, pas des copies.

2. Roblox / Luau

local Players = game:GetService("Players")
local ServerStorage = game:GetService("ServerStorage")

local function onJoin(player)
  local arme = ServerStorage.Epee:Clone()   -- Instance du moteur
  arme.Parent = player.Backpack
  print(player.Name .. " équipé")
end

Players.PlayerAdded:Connect(onJoin)

Roblox = Luau (Lua typé optionnellement). On obtient des fonctionnalités via game:GetService, on agit sur des Instances (arbre d'objets) et on s'abonne aux événements avec :Connect. Scripts Server vs LocalScript séparent serveur et client.

3. LÖVE (love2d)

function love.load()
  perso = love.graphics.newImage("hero.png")
  x = 100
end
function love.update(dt) x = x + 80 * dt end
function love.draw() love.graphics.draw(perso, x, 200) end
function love.keypressed(k) if k == "escape" then love.event.quit() end end

LÖVE est minimaliste : tu remplis les callbacks love.*. Tout le reste est du Lua pur (tes modules, tes tables).

4. Defold & autres hôtes

Defold structure le jeu en game objects + scripts attachés (callbacks init, update, on_message, on_input). On retrouve la même idée partout : des callbacks + une API moteur. Lua embarque aussi dans Solar2D, des plugins (Neovim), des serveurs (OpenResty)…

🔧 Démonstration pas à pas

  1. Écris un script Roblox qui équipe chaque joueur à l'arrivée.
  2. Écris un squelette LÖVE complet (load/update/draw/keypressed).
  3. Repère, dans chaque cas, ce qui vient du moteur et ce qui est du Lua pur.

⚠️ Pièges & erreurs courantes

SymptômeCode copié d'un moteur qui échoue dans un autre.
CauseL'API moteur diffère (love.* ≠ game:GetService).
CorrectionSépare ton Lua pur (réutilisable) de l'API moteur.
SymptômeDans Roblox, mon code client touche au serveur.
CauseConfusion LocalScript / Script.
CorrectionRespecte la frontière client/serveur (RemoteEvents).
SymptômeDu Luau typé ne tourne pas hors Roblox.
CauseLuau ≠ Lua standard (types, syntaxe étendue).
CorrectionGarde la logique portable en Lua standard.
SymptômeRessource introuvable (image/son).
CauseChemin relatif au mauvais dossier du projet moteur.
CorrectionSuis l'arborescence imposée par le moteur.

🔧Exercice pratique

  1. Liste 3 fonctions appelées par le moteur dans LÖVE.
  2. Écris un callback Roblox branché sur PlayerAdded.
  3. Identifie quelle partie de ton code serait réutilisable d'un moteur à l'autre.
Critères de réussite
  • Tu distingues l'API moteur du Lua pur.
  • Tu sais qu'un moteur appelle des callbacks.
  • Tu connais au moins deux hôtes Lua (Roblox, LÖVE).

Clique sur la réponse que tu penses correcte : la bonne réponse et une explication s'affichent aussitôt.

Exercices

Exercice 1Prise en main

Écris un squelette LÖVE minimal qui affiche un texte.

Voir une piste 
function love.draw()
  love.graphics.print("Bonjour LÖVE !", 100, 100)
end
Exercice 2Application

Script Roblox : donne 100 pièces à chaque joueur qui rejoint.

Voir une solution 
local Players = game:GetService("Players")
Players.PlayerAdded:Connect(function(player)
  local stats = Instance.new("IntValue")
  stats.Name = "Pieces"
  stats.Value = 100
  stats.Parent = player
end)
Exercice 3Défi

Conçois un module de logique de score réutilisable, indépendant du moteur.

Voir une piste 
-- score.lua : aucun appel à love.* ni game.* → portable
local M = {valeur = 0}
function M.ajouter(n) M.valeur = M.valeur + n end
function M.reset() M.valeur = 0 end
return M
-- Le code moteur (LÖVE ou Roblox) se contente d'appeler M.ajouter().
📌À retenir
  • Lua est embarqué : le moteur expose une API, ton Lua la pilote.
  • Roblox = Luau : services, Instances, :Connect, client/serveur.
  • LÖVE : callbacks love.load/update/draw + Lua pur.
  • Garde ta logique de jeu portable (Lua pur), isolée de l'API moteur.

Glossaire des termes introduits

embarqué — Lua intégré à un programme hôte (le moteur) qui l'exécute.

Luau — dialecte de Lua de Roblox (types optionnels, extensions).

Instance — objet de l'arbre Roblox (Part, Script, Value…).

API moteur — ensemble de fonctions exposées par le moteur (love.*, game:*).
