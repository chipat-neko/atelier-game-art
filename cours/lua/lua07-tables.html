LUA07 — Tables

Intermédiaire 40 min Piste 11 · Lua
PrérequisLUA6 — Fonctions.
🎯Objectifs d'apprentissage

À la fin de cette leçon, tu sauras :

  • Que la table est la seule structure de données de Lua.
  • L'utiliser comme tableau (séquence) et comme dictionnaire (clés nommées).
  • Ajouter/retirer avec table.insert / table.remove.
  • Imbriquer des tables pour modéliser un objet de jeu.

Pourquoi c'est important

En Lua, tout ce qui est structuré est une table : inventaires, ennemis, configurations, et même les « objets » (POO, LUA8). Maîtriser les tables, c'est maîtriser Lua.

Théorie en profondeur

1. Table comme tableau (séquence)

local inventaire = {"épée", "potion", "clé"}
print(inventaire[1])   -- épée  (indices commencent à 1 !)
print(#inventaire)     -- 3

⚠️ Rappel majeur : les indices Lua commencent à 1, pas à 0.

2. Table comme dictionnaire

local joueur = {
  nom = "Léa",
  pv = 100,
  niveau = 5,
}
print(joueur.nom)        -- Léa  (notation point)
print(joueur["pv"])      -- 100  (notation crochet, équivalente)
joueur.pv = 80           -- modifier
joueur.xp = 0            -- ajouter une clé

3. insert / remove

local sac = {}
table.insert(sac, "torche")     -- ajoute à la fin
table.insert(sac, "corde")
table.insert(sac, 1, "carte")   -- insère en position 1
table.remove(sac, 2)            -- retire l'élément 2
print(#sac)                     -- 2

4. Imbrication (modéliser un objet de jeu)

local ennemi = {
  nom = "Gobelin",
  pv = 30,
  position = {x = 12, y = 4},
  loot = {"or", "dague"},
}
print(ennemi.position.x)   -- 12
print(ennemi.loot[2])      -- dague

🖐Essaie maintenant~1 min

Teste une table : local inv = { or_val = 5 }; inv.or_val = inv.or_val + 3; print(inv.or_val). Ajoute inv.potion = 2. La table est LA structure de Lua (liste ET dictionnaire).

🔧 Démonstration pas à pas

  1. Crée un inventaire (séquence) et affiche le 1er élément.
  2. Crée un joueur (dictionnaire) et lis/modifie ses champs.
  3. Ajoute et retire des objets avec table.insert/remove.
  4. Imbrique une position {x, y} dans un ennemi.

⚠️ Pièges & erreurs courantes

Symptômet[0] est nil.
CauseLes séquences Lua commencent à 1.
CorrectionAccède à t[1] pour le premier élément.
Symptôme#t renvoie un nombre faux.
CauseLa table a un trou (nil au milieu).
CorrectionGarde une séquence dense ; table.remove décale proprement.
SymptômeDeux variables modifient « la même » table.
CauseLes tables sont des références.
CorrectionCopie explicitement si tu veux deux tables distinctes.
Symptômejoueur.x est nil alors que la clé existe.
CauseFaute de casse ou clé non-identifiant.
CorrectionUtilise t["ma clé"] pour les clés à espaces/chiffres.

🔧Exercice pratique

  1. Crée un inventaire et ajoute 3 objets avec table.insert.
  2. Crée un ennemi avec une position imbriquée et affiche ennemi.position.x.
  3. Retire un objet et vérifie #inventaire.
Critères de réussite
  • Tu accèdes correctement aux indices à partir de 1.
  • Tu sais lire une table imbriquée (a.b.c).
  • Tu comprends que les tables sont des références.

📌À retenir
  • La table est l'unique structure : tableau + dictionnaire en un.
  • Les indices de séquence commencent à 1.
  • table.insert / table.remove pour gérer une liste ; # pour la taille.
  • Les tables sont des références (attention aux copies).

Glossaire des termes introduits

table — unique structure de données de Lua (tableau + dictionnaire).

séquence — table indexée 1..n sans trou.

référence — une table n'est pas copiée par affectation ; les deux variables pointent dessus.
