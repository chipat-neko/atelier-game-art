LUA12 — Le scripting de jeu

Intermédiaire 45 min Piste 11 · Lua
PrérequisLUA6 — Fonctions, LUA11 — Coroutines.
🎯Objectifs d'apprentissage

À la fin de cette leçon, tu sauras :

  • Ce qu'est la boucle de jeu et la séparation update / draw.
  • Pourquoi multiplier les vitesses par le delta time.
  • Le pattern des callbacks : le moteur appelle ton code.
  • Réagir à des événements (entrées, collisions).

Pourquoi c'est important

Jusqu'ici tu écrivais des scripts qui s'exécutent une fois. Un jeu, lui, tourne en continu : 60 fois par seconde, il met à jour l'état puis dessine. Comprendre cette boucle est le passage obligé vers le vrai gameplay.

Théorie en profondeur

1. La boucle de jeu : update puis draw

Une frame = deux temps : mettre à jour la logique (positions, PV, IA), puis dessiner l'image. On ne mélange jamais les deux : update change l'état, draw l'affiche sans rien modifier.

local x = 0

function update(dt)
  x = x + 100 * dt     -- logique : avancer de 100 px/s
end

function draw()
  dessinerCercle(x, 150, 20)   -- affichage seul
end

2. Le delta time (dt)

Le dt est la durée de la frame précédente (en secondes). Sans lui, un jeu va deux fois plus vite sur une machine à 120 fps qu'à 60 fps. En multipliant par dt, le mouvement devient indépendant du framerate.

-- 100 * dt → 100 px PAR SECONDE, quel que soit le framerate
x = x + vitesse * dt

3. Le pattern callback : « ne nous appelez pas… »

Tu ne lances pas la boucle : le moteur la possède et appelle tes fonctions au bon moment. Tu fournis des callbacks (rappel de LUA6 : une fonction est une valeur qu'on passe).

-- tu DÉFINIS le comportement ; le moteur DÉCIDE quand l'appeler
function surTouche(touche)
  if touche == "space" then sauter() end
end

4. Événements

-- un système d'événements simple en Lua pur
local abonnes = {}
local function on(nom, fn) abonnes[nom] = abonnes[nom] or {}; table.insert(abonnes[nom], fn) end
local function emit(nom, ...) for _, fn in ipairs(abonnes[nom] or {}) do fn(...) end end

on("mort", function(qui) print(qui .. " est mort") end)
emit("mort", "Gobelin")   -- Gobelin est mort

🔧 Démonstration pas à pas

  1. Sépare clairement update(dt) (logique) et draw() (affichage).
  2. Fais avancer un objet avec vitesse * dt.
  3. Définis un callback d'entrée surTouche.
  4. Crée un mini bus d'événements on/emit.

⚠️ Pièges & erreurs courantes

SymptômeLe jeu va trop vite/trop lentement selon le PC.
CauseTu déplaces de valeurs fixes par frame, sans dt.
CorrectionMultiplie toujours par dt.
SymptômeDes bugs visuels intermittents.
CauseTu modifies l'état dans draw.
Correctiondraw ne fait qu'afficher ; la logique va dans update.
SymptômeMon code de jeu ne s'exécute jamais.
CauseTu attends d'appeler une boucle toi-même.
CorrectionDéfinis les callbacks que le moteur attend.
SymptômeCalcul lourd qui fait chuter le framerate.
CauseTravail coûteux fait chaque frame.
CorrectionÉtale-le (coroutine, cache) — voir LUA14.

🔧Exercice pratique

  1. Écris update(dt) qui déplace un joueur à 150 px/s.
  2. Écris draw() qui n'affiche que la position.
  3. Ajoute un bus d'événements et émets « score ».
Critères de réussite
  • update et draw ont des rôles distincts.
  • Tes vitesses sont en unités/seconde via dt.
  • Tu comprends que le moteur appelle tes callbacks.

📌À retenir
  • Une frame : update (logique) puis draw (affichage), sans mélange.
  • Multiplie les vitesses par dt pour être indépendant du framerate.
  • Le moteur appelle tes callbacks (inversion de contrôle).
  • Les événements découplent l'émetteur et les réactions.

Glossaire des termes introduits

boucle de jeu — cycle répété (entrées → update → draw) ≈ 60 fois/s.

delta time (dt) — durée de la frame précédente.

callback — fonction que le moteur appelle pour toi.

événement — signal auquel des fonctions s'abonnent.
