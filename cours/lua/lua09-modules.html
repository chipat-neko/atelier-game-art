LUA09 — Modules & require

Intermédiaire 30 min Piste 11 · Lua
PrérequisLUA8 — Métatables & POO.
🎯Objectifs d'apprentissage

À la fin de cette leçon, tu sauras :

  • Écrire un module qui retourne une table.
  • Le charger avec require.
  • Comprendre que require met en cache (un seul chargement).
  • Organiser un projet Lua en fichiers.

Pourquoi c'est important

Un jeu, ce n'est pas un seul fichier. On découpe en modules : player.lua, enemy.lua, utils.lua… require permet de les assembler proprement, sans variables globales partout.

Théorie en profondeur

1. Un module = un fichier qui retourne une table

-- fichier : mathutils.lua
local M = {}

function M.clamp(x, mini, maxi)
  if x < mini then return mini end
  if x > maxi then return maxi end
  return x
end

return M   -- on RENVOIE la table du module

2. Charger avec require

-- fichier : main.lua
local mu = require("mathutils")   -- sans l'extension .lua
print(mu.clamp(150, 0, 100))      -- 100

On nomme le module par son chemin de fichier (le point sert de séparateur de dossier : require("jeu.player") → jeu/player.lua).

3. Le cache de require

require ne charge un module qu'une seule fois : les appels suivants renvoient la même table (mise en cache dans package.loaded). C'est voulu — les modules sont des singletons partagés.

local a = require("config")
local b = require("config")
print(a == b)   -- true : même table

🔧 Démonstration pas à pas

  1. Crée mathutils.lua avec une table locale M et return M.
  2. Ajoute une fonction M.clamp.
  3. Charge-le depuis main.lua avec require.
  4. Vérifie que deux require renvoient la même table.

⚠️ Pièges & erreurs courantes

Symptômerequire renvoie true au lieu du module.
CauseLe fichier ne fait pas return M.
CorrectionTermine le module par return de sa table.
Symptôme« module not found ».
CauseMauvais chemin ou extension .lua incluse.
Correctionrequire("dossier.fichier") sans .lua.
SymptômeMes modifications d'un module ne sont pas reprises.
CauseLe module est en cache.
CorrectionRecharge le jeu, ou vide package.loaded[nom].
SymptômeVariables qui fuient entre fichiers.
CauseTu déclares en global au lieu de la table du module.
CorrectionTout passe par local M + champs de M.

🔧Exercice pratique

  1. Écris un module vecteur avec longueur(x, y).
  2. Charge-le et calcule la longueur de (3, 4).
  3. Explique pourquoi deux require renvoient la même table.
Critères de réussite
  • Ton module se termine par return de sa table.
  • Tu charges avec require sans l'extension.
  • Tu n'utilises aucune variable globale.

Exercices

Exercice 1Prise en main

Écris un module saluts avec une fonction bonjour(nom).

Voir une piste 
-- saluts.lua
local M = {}
function M.bonjour(nom) return "Bonjour " .. nom end
return M
Exercice 2Application

Module vecteur avec longueur(x, y) (théorème de Pythagore).

Voir une solution 
-- vecteur.lua
local M = {}
function M.longueur(x, y) return math.sqrt(x*x + y*y) end
return M

-- main.lua
local v = require("vecteur")
print(v.longueur(3, 4))   -- 5.0
Exercice 3Défi

Crée un module compteurGlobal qui garde un état entre deux require.

Voir une piste 
-- compteurGlobal.lua
local M = {total = 0}
function M.ajouter(n) M.total = M.total + n end
return M
-- Grâce au cache, tous les require partagent la même table M.
📌À retenir
  • Un module = un fichier qui retourne une table.
  • require("dossier.fichier") charge sans l'extension .lua.
  • require met en cache : chargement unique, table partagée.
  • Évite les globales : tout passe par la table du module.

Glossaire des termes introduits

module — fichier Lua qui expose une table de fonctions/données.

require — charge un module (une seule fois) et renvoie sa table.

package.loaded — cache interne des modules déjà chargés.

Voir aussi le glossaire global.
