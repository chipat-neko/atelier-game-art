LUA04 — Conditions

Débutant 30 min Piste 11 · Lua
PrérequisLUA3 — Opérateurs.
🎯Objectifs d'apprentissage

À la fin de cette leçon, tu sauras :

  • Écrire if / elseif / else / end.
  • Comprendre ce qui est « vrai » en Lua (seuls false et nil sont faux).
  • Simuler un ternaire avec cond and a or b.

Pourquoi c'est important

Une condition fait réagir ton jeu : ouvrir une porte, infliger des dégâts, déclencher un dialogue. La syntaxe Lua se distingue par le mot-clé then et la fermeture explicite end.

Théorie en profondeur

1. if / elseif / else / end

local pv = 30

if pv <= 0 then
  print("Game Over")
elseif pv < 50 then
  print("Attention, vie basse !")
else
  print("Tout va bien")
end

Pas d'accolades : le bloc commence après then et se ferme avec end. elseif s'écrit en un seul mot.

2. Ce qui est « vrai » en Lua

Règle d'or rappelée de LUA3 : seuls false et nil sont faux. Tout le reste est vrai, y compris 0 et la chaîne vide.

if 0 then print("0 est VRAI en Lua") end       -- s'affiche
local cible = trouverCible()
if cible then                                  -- nil = faux → pratique pour tester l'existence
  attaquer(cible)
end

3. Ternaire simulé avec and/or

Lua n'a pas d'opérateur ternaire (?:). On l'imite avec cond and a or b :

local etat = (pv > 0) and "vivant" or "mort"
print(etat)

⚠️ Cette astuce échoue si a peut valoir false/nil : dans ce cas, utilise un vrai if.

🔧 Démonstration pas à pas

  1. Écris une chaîne if/elseif/else sur les PV.
  2. Teste l'existence d'une valeur avec if cible then.
  3. Compose un état avec le ternaire and/or.

⚠️ Pièges & erreurs courantes

Symptôme« 'end' expected ».
CauseTu as oublié end ou then.
CorrectionChaque if a un then et se ferme par end.
SymptômeMon test sur 0 ne se comporte pas comme attendu.
CauseEn Lua, 0 est vrai.
CorrectionCompare explicitement : if n == 0 then.
Symptômeelse if provoque une erreur de structure.
CauseTu sépares en deux mots.
CorrectionUtilise elseif (un seul mot).
SymptômeMon ternaire renvoie la mauvaise valeur.
Causea vaut false/nil dans cond and a or b.
CorrectionRepasse à un vrai if dans ce cas.

🔧Exercice pratique

  1. Affiche un message selon une note (>= 16 « Excellent », >= 10 « Reçu », sinon « À revoir »).
  2. Teste si une variable arme existe avant de l'utiliser.
  3. Compose etat = pv > 0 and "vivant" or "mort".
Critères de réussite
  • Ta chaîne if/elseif/else est correctement fermée par end.
  • Tu utilises elseif en un mot.
  • Tu testes l'existence avec if x then.

Vérifie ta compréhension

Clique sur la réponse que tu penses correcte : la bonne réponse et une explication s'affichent aussitôt.

Exercices

Exercice 1Prise en main

Affiche « Majeur » ou « Mineur » selon un âge.

Voir une piste 
local age = 20
if age >= 18 then
  print("Majeur")
else
  print("Mineur")
end
Exercice 2Application

Classe une note en trois catégories avec elseif.

Voir une solution 
local note = 13
if note >= 16 then
  print("Excellent")
elseif note >= 10 then
  print("Reçu")
else
  print("À revoir")
end
Exercice 3Défi

Donne un état textuel en une ligne via le ternaire simulé.

Voir une piste 
local pv = 0
local etat = (pv > 0) and "vivant" or "mort"
print(etat)   -- mort
📌À retenir
  • if … then … elseif … then … else … end (pas d'accolades).
  • Seuls false et nil sont faux ; 0 est vrai.
  • elseif s'écrit en un seul mot.
  • Ternaire simulé : cond and a or b (attention si a peut être faux).

Glossaire des termes introduits

then / end — ouvrent et ferment un bloc conditionnel.

elseif — branche alternative (un seul mot).

truthy — valeur considérée comme vraie ; en Lua, tout sauf false et nil.
