LUA08 — Métatables & POO

Avancé 45 min Piste 11 · Lua
PrérequisLUA7 — Tables, LUA6 — Fonctions.
🎯Objectifs d'apprentissage

À la fin de cette leçon, tu sauras :

  • Ce qu'est une métatable et le rôle de __index.
  • Le pattern « classe » avec setmetatable et Classe.new.
  • Définir des méthodes avec function Classe:methode() et self.
  • Faire un héritage simple entre classes.

Pourquoi c'est important

Lua n'a pas de mot-clé class. La POO se construit à partir des tables et des métatables. C'est ainsi que fonctionnent les « objets » dans Roblox, LÖVE et la plupart des moteurs Lua.

Théorie en profondeur

1. Métatable & __index

Quand on lit une clé absente d'une table, Lua consulte sa métatable. Si celle-ci a un champ __index pointant vers une autre table, la recherche s'y poursuit : c'est le mécanisme d'héritage.

local base = {salut = function() print("Hello") end}
local obj = setmetatable({}, {__index = base})
obj.salut()   -- Hello  (trouvé via __index → base)

2. Le pattern « classe »

local Personnage = {}
Personnage.__index = Personnage

function Personnage.new(nom, pv)
  local self = setmetatable({}, Personnage)
  self.nom = nom
  self.pv = pv
  return self
end

function Personnage:recevoir(degats)   -- : injecte self
  self.pv = self.pv - degats
  print(self.nom .. " : " .. self.pv .. " PV")
end

local héros = Personnage.new("Léa", 100)
héros:recevoir(30)   -- Léa : 70 PV

Le : à la déclaration et à l'appel ajoute automatiquement self. Personnage.__index = Personnage rend les méthodes accessibles aux instances.

3. Héritage simple

local Ennemi = setmetatable({}, {__index = Personnage})
Ennemi.__index = Ennemi

function Ennemi.new(nom, pv, butin)
  local self = Personnage.new(nom, pv)   -- réutilise le parent
  setmetatable(self, Ennemi)
  self.butin = butin
  return self
end

local gob = Ennemi.new("Gobelin", 30, "or")
gob:recevoir(10)   -- méthode héritée de Personnage → Gobelin : 20 PV

🔧 Démonstration pas à pas

  1. Crée une « classe » Personnage avec new et __index.
  2. Ajoute une méthode :recevoir(degats) utilisant self.
  3. Crée une instance et appelle la méthode avec :.
  4. Dérive Ennemi qui hérite de Personnage.

⚠️ Pièges & erreurs courantes

Symptômeself est nil dans une méthode.
CauseTu appelles avec . au lieu de :.
CorrectionAppelle obj:methode() (le : passe self).
SymptômeLes instances ne voient pas les méthodes.
CauseTu as oublié Classe.__index = Classe.
CorrectionAjoute cette ligne avant setmetatable.
SymptômeToutes les instances partagent la même valeur.
CauseTu as mis l'état dans la classe, pas dans self.
CorrectionStocke les données par instance dans self.
SymptômeL'héritage ne remonte pas au parent.
CauseChaîne __index mal configurée.
Correctionsetmetatable(Enfant, {__index = Parent}).

Glossaire des termes introduits

métatable — table décrivant le comportement d'une autre table (opérations, recherche).

__index — métaméthode indiquant où chercher une clé absente.

self — l'instance courante, passée par le :.

setmetatable — attache une métatable à une table.
