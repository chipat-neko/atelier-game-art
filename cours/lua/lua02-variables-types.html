LUA02 — Variables & types

Débutant 35 min Piste 11 · Lua
PrérequisLUA1 — Introduction.
🎯Objectifs d'apprentissage

À la fin de cette leçon, tu sauras :

  • Les types de base : nil, boolean, number, string, table.
  • Que Lua est typé dynamiquement (le type suit la valeur).
  • La différence cruciale entre local et global.
  • Inspecter un type avec type().

Pourquoi c'est important

Lua a peu de types, mais ils sont puissants (surtout la table, vue en LUA7). Et la distinction local/global est le piège n°1 du débutant : l'oublier ralentit et pollue l'espace global.

Théorie en profondeur

1. Les types de base

TypeExempleRôle
nilnil« rien » : variable non définie/absence de valeur.
booleantrue, falseVrai/faux.
number42, 3.14Nombres (entiers et décimaux, même type).
string"Héros"Texte.
table{1, 2, 3}LA structure de données (tableau + dictionnaire).

(Il existe aussi function et thread — les coroutines, LUA11.)

2. Typage dynamique

On ne déclare pas de type : une variable prend le type de sa valeur, et peut même changer. Pratique mais à manier avec soin :

local pv = 100          -- number
local nom = "Slime"     -- string
local vivant = true     -- boolean
local rien              -- nil (non initialisée)
print(type(pv), type(nom), type(vivant), type(rien))
-- number  string  boolean  nil

3. local vs global (essentiel !)

Sans local, une variable est globale : visible partout, plus lente d'accès, et source de bugs (collisions de noms). Toujours préfixer par local sauf raison précise.

local score = 0     -- ✅ local : portée limitée, rapide
total = 0           -- ❌ global implicite : à éviter

4. nil et l'absence

nil signifie « pas de valeur ». Accéder à une variable jamais définie renvoie nil (pas une erreur). Mettre une variable à nil la « supprime ». On teste souvent if x ~= nil then (ou simplement if x then, vu en LUA4).

🔧 Démonstration pas à pas

  1. Déclare une variable de chaque type (number, string, boolean) en local.
  2. Affiche leur type avec type(...).
  3. Mets une variable à nil et observe qu'elle « disparaît ».
  4. Compare l'accès à une globale vs une locale (juste pour comprendre la portée).

📌À retenir

  • Lua est typé dynamiquement : le type suit la valeur.

Glossaire des termes introduits

nil — valeur représentant l'absence ; une variable non définie vaut nil.

local / global — portée limitée (recommandé) vs visible partout (à éviter).

Typage dynamique — le type est porté par la valeur, pas déclaré.

type() — fonction renvoyant le type d'une valeur sous forme de texte.
