LUA10 — Gestion des erreurs

Intermédiaire 30 min Piste 11 · Lua
PrérequisLUA6 — Fonctions.
🎯Objectifs d'apprentissage

À la fin de cette leçon, tu sauras :

  • Lever une erreur avec error et valider avec assert.
  • Exécuter du code risqué sans planter grâce à pcall.
  • Récupérer le message d'erreur et réagir proprement.
  • Utiliser xpcall pour une trace détaillée.

Pourquoi c'est important

Dans un jeu, une erreur non gérée fige tout. pcall (« protected call ») permet d'isoler un bout de code risqué (chargement de sauvegarde, script de mod) et de continuer même s'il échoue.

Théorie en profondeur

1. error & assert

local function diviser(a, b)
  if b == 0 then
    error("division par zéro")
  end
  return a / b
end

-- assert : lève une erreur si la condition est fausse
local fichier = assert(ouvrir("save.dat"), "sauvegarde introuvable")

assert(v, msg) renvoie v s'il est vrai, sinon lève une erreur avec msg. Idéal pour valider des préconditions.

2. pcall : appel protégé

local ok, resultat = pcall(diviser, 10, 0)
if ok then
  print("Résultat : " .. resultat)
else
  print("Erreur attrapée : " .. resultat)   -- resultat = message d'erreur
end
-- Le jeu continue, pas de crash.

pcall(f, ...) renvoie true, retours… si tout va bien, ou false, message en cas d'erreur. Le programme ne plante pas.

3. xpcall & trace

local function gestionnaire(err)
  return "LOG : " .. err .. "\n" .. debug.traceback()
end

local ok, info = xpcall(function()
  error("boom")
end, gestionnaire)
print(ok, info)

xpcall est comme pcall mais ajoute un gestionnaire qui s'exécute au moment de l'erreur (utile pour capturer la pile avec debug.traceback).

🔧 Démonstration pas à pas

  1. Écris diviser(a, b) qui lève une erreur si b == 0.
  2. Appelle-la via pcall et gère le cas d'échec.
  3. Valide une précondition avec assert.
  4. Ajoute une trace avec xpcall + debug.traceback.

⚠️ Pièges & erreurs courantes

SymptômeLe jeu plante malgré le code « protégé ».
CauseTu as appelé f() avant de la passer à pcall.
CorrectionPasse la fonction : pcall(f, arg), pas pcall(f(arg)).
SymptômeJe perds le résultat de la fonction.
CauseTu oublies que pcall ajoute ok en première valeur.
Correctionlocal ok, res = pcall(...).
SymptômeErreurs avalées silencieusement.
CauseTu ignores le false de pcall.
CorrectionJournalise le message quand ok est faux.
Symptômeassert masque la vraie cause.
CauseMessage d'assert trop vague.
CorrectionDonne un message explicite en 2e argument.

🔧Exercice pratique

  1. Protège un chargement de « sauvegarde » avec pcall.
  2. Affiche un message clair si l'opération échoue.
  3. Valide un argument non nul avec assert.
Critères de réussite
  • Tu passes la fonction (et ses args) à pcall, sans l'appeler avant.
  • Tu lis ok et le message d'erreur.
  • Le programme continue après un échec.

Exercices

Exercice 1Prise en main

Protège un appel qui peut lever une erreur et affiche le message.

Voir une piste 
local ok, err = pcall(function() error("oops") end)
if not ok then print("Attrapé : " .. err) end
Exercice 2Application

Écris racine(n) qui refuse les négatifs via assert.

Voir une solution 
local function racine(n)
  assert(n >= 0, "n doit être positif")
  return math.sqrt(n)
end
local ok, res = pcall(racine, -4)
print(ok, res)   -- false  ...: n doit être positif
Exercice 3Défi

Charge une « config » avec valeur de repli si le chargement échoue.

Voir une piste 
local function chargerConfig() error("fichier manquant") end
local ok, config = pcall(chargerConfig)
if not ok then
  config = {volume = 50, plein_ecran = false}   -- repli
end
print(config.volume)   -- 50

📌À retenir
  • error(msg) lève ; assert(v, msg) valide une précondition.
  • pcall(f, ...) isole le code risqué : true, res ou false, msg.
  • Passe la fonction à pcall, ne l'appelle pas avant.
  • xpcall + debug.traceback pour une trace complète.

Glossaire des termes introduits

error — lève une erreur (interrompt le flux courant).

assert — vérifie une condition, lève une erreur sinon.

pcall — « protected call » : exécute sans laisser l'erreur planter le programme.

xpcall — pcall avec un gestionnaire d'erreur (pour la trace).
