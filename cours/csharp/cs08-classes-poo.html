CS8 — Classes & objets (POO)

Intermédiaire 45 min Piste 7 · C# (Unity)
PrérequisCS6 — Méthodes et CS2 — Variables.
🎯Objectifs d'apprentissage

À la fin de cette leçon, tu sauras :

  • Définir une classe avec ses champs et méthodes.
  • Créer des objets (instances) et un constructeur.
  • Appliquer l'encapsulation (public/private, propriétés).
  • Distinguer une classe normale d'un MonoBehaviour, et un membre static.

Pourquoi c'est important

La programmation orientée objet (POO) organise le code en classes : des « moules » qui regroupent des données (champs) et des actions (méthodes) qui vont ensemble. Un Ennemi a une vie, une vitesse, et sait Attaquer(). C'est la façon dont tout code Unity est structuré (chaque script est une classe). Maîtriser la POO, c'est passer de scripts fourre-tout à un code clair et réutilisable.

Théorie en profondeur

1. Classe = moule, objet = exemplaire

Une classe décrit ce qu'est et ce que sait faire une catégorie de choses. Un objet (ou instance) est un exemplaire concret créé à partir de ce moule. La classe Ennemi est le plan ; « cet ennemi-ci à 30 PV » est un objet.

// Une classe « normale » (PAS un MonoBehaviour) — un simple modèle de données
public class Ennemi
{
    public string nom;        // champ
    public int vie;           // champ

    public void Attaquer()    // méthode
    {
        Debug.Log(nom + " attaque !");
    }
}

On crée des objets avec le mot-clé new :

Ennemi gobelin = new Ennemi();   // un objet
gobelin.nom = "Gobelin";
gobelin.vie = 30;
gobelin.Attaquer();              // "Gobelin attaque !"

Ennemi dragon = new Ennemi();    // un AUTRE objet, indépendant
dragon.nom = "Dragon";
dragon.vie = 500;

2. Constructeur : initialiser à la création

Un constructeur est une méthode spéciale (même nom que la classe, sans type de retour) appelée par new :

public class Ennemi
{
    public string nom;
    public int vie;

    public Ennemi(string nomInitial, int vieInitiale)  // constructeur
    {
        nom = nomInitial;
        vie = vieInitiale;
    }
}

// Création en une ligne :
Ennemi gobelin = new Ennemi("Gobelin", 30);
ℹ️Les MonoBehaviour ne s'instancient pas avec new

Un script Unity (qui hérite de MonoBehaviour) n'est jamais créé avec new : il est ajouté comme Component (Add Component) ou via Instantiate/AddComponent (CS11), et n'a pas de constructeur — c'est Awake/Start qui jouent ce rôle. new est pour tes classes « normales » (données, logique pure).

3. Encapsulation : public vs private

L'encapsulation consiste à protéger les données d'une classe : private = accessible seulement dans la classe ; public = accessible de partout. Bonne pratique : champs private, et on expose des méthodes ou des propriétés contrôlées pour y toucher.

public class Compte
{
    private int pieces = 0;              // protégé

    public void Ajouter(int montant)
    {
        if (montant > 0) pieces += montant;   // on contrôle ce qui entre
    }

    public int Pieces => pieces;         // propriété en lecture seule (raccourci)
}

Une propriété ressemble à un champ vu de l'extérieur mais cache du code (un get/set). public int Pieces => pieces; expose la valeur en lecture seule : on peut la lire, pas l'écrire directement.

4. Membres static : partagés par la classe

Un membre static appartient à la classe entière, pas à un objet précis : il existe en un seul exemplaire. Utile pour des constantes ou un compteur global :

public class Ennemi
{
    public static int nombreVivants = 0;  // partagé par TOUS les ennemis

    public Ennemi() { nombreVivants++; }  // chaque création incrémente
}
// Accès via la CLASSE, pas un objet :  Debug.Log(Ennemi.nombreVivants);

À utiliser avec parcimonie : le static crée un état global, source de couplage si on en abuse (on en reparle en CS15).

🔧 Démonstration pas à pas

  1. Crée une classe Item (PAS un MonoBehaviour) avec nom (string), prix (int), et un constructeur.
  2. Dans un script attaché (MonoBehaviour), crée deux Item avec new et affiche leurs infos.
  3. Passe le champ prix en private et expose-le via une propriété en lecture seule ; ajoute une méthode AppliquerReduction(int pourcent).

⚠️ Pièges & erreurs courantes

SymptômeNullReferenceException en utilisant un objet.
CauseTu utilises une variable d'objet jamais initialisée avec new (elle vaut null).
CorrectionCrée l'objet (x = new MaClasse();) avant de l'utiliser.
Symptôme« MonoBehaviour … you can't use new ».
CauseTu fais new sur une classe qui hérite de MonoBehaviour.
CorrectionAjoute-la via Add Component / AddComponent (CS11), pas new.
SymptômeModifier un objet change aussi un autre.
CauseDeux variables pointent vers le même objet (les classes sont des références).
CorrectionCrée un nouvel objet avec new si tu veux une copie indépendante.
Symptôme« an object reference is required for the non-static field ».
CauseTu accèdes à un membre d'instance via la classe (ou inversement pour un static).
CorrectionMembre d'instance → via un objet ; membre static → via la classe.

🔧Exercice pratique

  1. Crée une classe Arme avec nom, degats (privés), un constructeur, et une méthode Decrire().
  2. Expose degats en lecture seule via une propriété ; ajoute une méthode Ameliorer(int bonus).
  3. Ajoute un static int nombreArmes incrémenté à chaque création ; affiche-le après avoir créé 3 armes.
Critères de réussite
  • Tes objets sont créés avec new et indépendants.
  • Les champs sensibles sont private, exposés par propriété/méthode.
  • Le compteur static reflète le nombre d'objets créés.

Encapsule les munitions : champ private, un constructeur qui les initialise, et une méthode publique Tirer() qui les décrémente sans descendre sous 0.

Voir une solution 
public class Arme
{
    private int munitions;

    public Arme(int munitionsInitiales)
    {
        munitions = munitionsInitiales;
    }

    public void Tirer()
    {
        if (munitions > 0)
        {
            munitions--;
            Debug.Log("Pan ! Reste : " + munitions);
        }
        else Debug.Log("Plus de munitions");
    }
}

    }

📌À retenir
  • Une classe = moule (champs + méthodes) ; un objet = instance créée avec new.
  • Le constructeur initialise à la création ; les MonoBehaviour n'utilisent pas new (Add Component).
  • Encapsulation : champs private, accès contrôlé par méthodes/propriétés.
  • Un membre static appartient à la classe (partagé), à utiliser avec parcimonie.

Glossaire des termes introduits

POO — organisation du code en classes regroupant données et comportements.

Classe / objet (instance) — le moule / un exemplaire concret créé avec new.

Constructeur — méthode spéciale appelée à la création d'un objet.

Encapsulation — protection des données (private) avec un accès contrôlé (propriétés/méthodes).

static — membre appartenant à la classe entière, pas à un objet.
