CS2 — Variables & types

Débutant 35 min Piste 7 · C# (Unity)
PrérequisCS1 — Introduction & premier script.
🎯Objectifs d'apprentissage

À la fin de cette leçon, tu sauras :

  • Déclarer une variable et comprendre la notion de type.
  • Utiliser les types essentiels : int, float, bool, string (et var).
  • Respecter les conventions de nommage du C#.
  • Exposer une variable dans l'Inspector avec public ou [SerializeField].

Pourquoi c'est important

Un jeu manipule sans cesse des données : points de vie, vitesse, score, nom du joueur, « est-il en vie ? ». Une variable est une boîte étiquetée qui stocke une de ces valeurs et qu'on peut lire et modifier. Bien choisir son type (entier, nombre à virgule, vrai/faux, texte) évite des bugs subtils — comme une division d'entiers qui donne 0, ou une vitesse « collée » parce qu'on a oublié le suffixe f.

Théorie en profondeur

1. Déclarer une variable

La forme de base : type nom = valeur;

int pointsDeVie = 100;      // un entier
float vitesse = 5.5f;       // un nombre à virgule (note le 'f' !)
bool estEnVie = true;       // vrai ou faux
string nomJoueur = "Léa";   // du texte (entre guillemets doubles)

Le type précise quelle sorte de donnée la boîte contient. Le C# est typé : une variable int ne contiendra que des entiers — y mettre du texte provoque une erreur de compilation. C'est une sécurité : le compilateur attrape tes étourderies avant le jeu.

2. Les types essentiels

TypeContientExempleUsage jeu
intNombre entier3, -7, 0Score, munitions, niveau
floatNombre à virgule (suffixe f)5.5f, 0.1fVitesse, temps, distance
boolVrai ou fauxtrue, false« est en vie », « porte ouverte »
stringTexte"Game Over"Noms, dialogues, UI
charUn seul caractère (guillemets simples)'A', 'x'Touche pressée, initiale
longEntier 64 bits (très grands nombres)9000000000LScore cumulé, identifiants
Vector3Position/direction 3D (x,y,z)transform.positionPosition, déplacement (Unity)

Le float exige le suffixe f (sinon C# pense que 5.5 est un double, un type plus précis qu'Unity n'utilise pas par défaut → erreur ou avertissement). Le Vector3 est un type d'Unity très courant (positions), vu en pratique dès CS11/CS12.

3. var : laisser C# deviner le type

Le mot-clé var demande au compilateur de déduire le type à partir de la valeur :

var score = 0;          // C# en déduit : int
var nom = "Boss";       // C# en déduit : string

var est pratique quand le type est évident, mais reste fortement typé (ce n'est PAS du « sans type »). En débutant, écris le type explicitement : c'est plus clair.

4. Conventions de nommage

  • Les variables « locales » et les champs privés : camelCase (pointsDeVie, vitesseMax).
  • Pas d'espaces ni d'accents, pas de chiffre en première position. Choisis des noms parlants : vitesse vaut mieux que v.
  • Un nom = une intention claire. Du code lisible se relit comme une phrase.

5. Exposer une variable dans l'Inspector

Énorme avantage d'Unity : tu peux régler une variable depuis l'éditeur, sans toucher au code. Deux façons :

using UnityEngine;

public class Joueur : MonoBehaviour
{
    public float vitesse = 5f;          // public → visible ET modifiable dans l'Inspector
    [SerializeField] private int vie = 100;   // privé MAIS exposé à l'Inspector (recommandé)
    private bool estInvincible = false; // privé, NON visible dans l'Inspector

    void Start()
    {
        Debug.Log("Vitesse = " + vitesse + ", Vie = " + vie);
    }
}

Préfère [SerializeField] private à public : tu gardes le réglage dans l'Inspector tout en protégeant la variable contre des modifications accidentelles par d'autres scripts (principe d'encapsulation, voir CS8). Détail crucial : une valeur réglée dans l'Inspector écrase la valeur du code — l'Inspector « gagne ».

🖐Essaie maintenant~1 min

Prédis : int x = 7 / 2; puis float y = 7f / 2;. (Réponses : x = 3, y = 3.5.) La division de deux int tronque ; le suffixe f force un float.

🔧 Démonstration pas à pas

  1. Crée un script Joueur, colle le code ci-dessus, sauvegarde.
  2. Attache-le à un GameObject : dans l'Inspector, tu vois maintenant les champs Vitesse et Vie (mais pas estInvincible).
  3. Change Vitesse à 10 dans l'Inspector, Play : la Console affiche 10 — la valeur de l'Inspector a écrasé le 5 du code.
  4. Ajoute une variable string nomJoueur = "Inconnu"; en [SerializeField] private, sauvegarde, et règle-la dans l'Inspector.

⚠️ Pièges & erreurs courantes

SymptômeErreur « Literal of type double cannot be implicitly converted to float ».
CauseUn float sans le suffixe f (ex. float v = 5.5;).
CorrectionAjoute le f : float v = 5.5f;
SymptômeJe modifie la valeur dans le code mais rien ne change en jeu.
CauseLa variable est public/[SerializeField] et l'Inspector écrase la valeur du code.
CorrectionChange la valeur dans l'Inspector (ou clique les ⋮ ▸ Reset pour revenir au code).
SymptômeMa variable n'apparaît pas dans l'Inspector.
CauseElle est private sans [SerializeField].
CorrectionAjoute [SerializeField] devant, ou passe-la public (moins recommandé).
Symptôme10 / 3 me donne 3 au lieu de 3.33.
CauseDivision de deux int → résultat entier (tronqué). Voir CS3.
CorrectionUtilise des float : 10f / 3f.

  3. Règle chaque champ dans l'Inspector et vérifie l'affichage au Play.
Critères de réussite
  • Les quatre variables ont le bon type et apparaissent dans l'Inspector.
  • Le message reflète les valeurs réglées dans l'Inspector (pas celles du code).
  • Tu sais expliquer pourquoi un float a besoin du suffixe f.

📌À retenir
  • Une variable = type + nom + valeur ; le C# est typé (chaque boîte n'accepte qu'une sorte de donnée).
  • Essentiels : int, float (suffixe f !), bool, string, Vector3.
  • Nommage en camelCase, noms parlants.
  • [SerializeField] private = réglable dans l'Inspector et protégé ; l'Inspector écrase la valeur du code.

Glossaire des termes introduits

Variable — boîte nommée qui stocke une valeur modifiable.

Type — nature de la donnée d'une variable (int, float, bool, string…).

[SerializeField] — attribut qui expose une variable privée dans l'Inspector sans la rendre publique.

var — mot-clé laissant C# déduire le type à partir de la valeur (toujours fortement typé).
