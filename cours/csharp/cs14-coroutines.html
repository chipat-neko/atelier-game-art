CS14 — Coroutines & gestion du temps

Avancé 40 min Piste 7 · C# (Unity)
PrérequisCS6 — Méthodes et CS7.
🎯Objectifs d'apprentissage

À la fin de cette leçon, tu sauras :

  • Comprendre pourquoi on ne peut pas « attendre » avec une boucle classique.
  • Écrire une coroutine (IEnumerator + yield return).
  • Temporiser avec WaitForSeconds et étaler une action sur plusieurs frames.
  • Démarrer/arrêter une coroutine (StartCoroutine/StopCoroutine).

Pourquoi c'est important

« Attends 3 secondes puis ouvre la porte », « fais clignoter l'ennemi quand il est touché », « fais apparaître une vague d'ennemis toutes les 2 s » : ces séquences étalées dans le temps sont partout. Mais Unity est mono-thread : si tu écris while avec une attente, tu gèles tout le jeu. Les coroutines permettent de mettre une fonction « en pause » et de rendre la main au moteur entre-temps.

🛑Ce qu'il ne faut JAMAIS faire
// ❌ GÈLE le jeu : la frame ne se termine jamais pendant l'attente
void Ouvrir()
{
    float t = 0;
    while (t < 3f) { t += Time.deltaTime; }  // boucle infinie sur la même frame
    porte.Ouvrir();
}

Théorie en profondeur

1. Une coroutine : une méthode qui peut se mettre en pause

Une coroutine renvoie IEnumerator et utilise yield return pour « rendre la main » au moteur jusqu'à une condition :

using System.Collections;   // nécessaire pour IEnumerator

IEnumerator OuvrirApresDelai()
{
    Debug.Log("Compte à rebours…");
    yield return new WaitForSeconds(3f);   // PAUSE 3 s, le jeu continue de tourner
    Debug.Log("Porte ouverte !");
}

void Start()
{
    StartCoroutine(OuvrirApresDelai());    // on LANCE la coroutine
}

Au yield return new WaitForSeconds(3f), la méthode est suspendue : Unity continue à afficher les frames, puis reprend la coroutine 3 secondes plus tard, juste après le yield.

2. Les différents yield

yield return null;                        // attendre la frame suivante
yield return new WaitForSeconds(0.5f);    // attendre 0,5 s (temps de jeu)
yield return new WaitForSecondsRealtime(1f); // attendre 1 s réelle (ignore Time.timeScale)
yield return new WaitForEndOfFrame();     // après le rendu de la frame
yield return new WaitUntil(() => pretA);  // attendre qu'une condition soit vraie

3. Faire quelque chose progressivement

Combinée à une boucle, la coroutine étale une action sur plusieurs frames — par exemple un fondu :

IEnumerator Clignoter(int fois)
{
    for (int i = 0; i < fois; i++)
    {
        rendu.enabled = false;
        yield return new WaitForSeconds(0.1f);
        rendu.enabled = true;
        yield return new WaitForSeconds(0.1f);
    }
}

4. Arrêter une coroutine

Coroutine c = StartCoroutine(Clignoter(5));  // garder la référence
// ...
StopCoroutine(c);        // arrêter celle-ci
StopAllCoroutines();     // arrêter toutes celles de ce MonoBehaviour
💡Invoke, l'alternative simple

Pour un simple délai unique sans logique : Invoke("Methode", 3f); appelle Methode dans 3 s, et InvokeRepeating("Tic", 1f, 2f) la répète toutes les 2 s. Plus limité que les coroutines (par nom, pas de paramètres), mais pratique.

5. Time.timeScale : la pause globale

Time.timeScale = 0f; gèle le temps de jeu (utile pour un menu pause) : Update tourne encore mais Time.deltaTime vaut 0 et les WaitForSeconds sont suspendus (utilise WaitForSecondsRealtime pour ignorer la pause). = 1f reprend le cours normal.

🖐Essaie maintenant~1 min

Écris une coroutine : IEnumerator Attendre() { Debug.Log("avant"); yield return new WaitForSeconds(1f); Debug.Log("après"); } et lance-la avec StartCoroutine(Attendre()); : la 2ᵉ ligne arrive 1 s plus tard.

🔧 Démonstration pas à pas

  1. Écris IEnumerator OuvrirApresDelai() qui logge, attend 3 s, puis logge. Lance-la dans Start avec StartCoroutine.
  2. Écris Clignoter(int fois) qui active/désactive le Renderer avec des pauses : l'objet clignote quand il est touché.
  3. Crée un spawner : une coroutine while(true) avec yield return new WaitForSeconds(2f) qui instancie un ennemi (renvoi CS11).
  4. Ajoute un menu pause : bascule Time.timeScale entre 0 et 1.

⚠️ Pièges & erreurs courantes

SymptômeMa coroutine ne se lance pas.
CauseTu l'as appelée comme une méthode (OuvrirApresDelai();).
CorrectionUtilise StartCoroutine(OuvrirApresDelai());.
Symptôme« IEnumerator/yield introuvable ».
CauseIl manque using System.Collections;.
CorrectionAjoute le using en haut du fichier.
SymptômeLa coroutine s'arrête toute seule.
CauseLe GameObject est désactivé/détruit : ses coroutines s'arrêtent.
CorrectionLance-la sur un objet actif, ou gère la reprise après réactivation.
SymptômeMes attentes ne passent pas en menu pause.
CauseWaitForSeconds dépend du temps de jeu (timeScale = 0).
CorrectionUtilise WaitForSecondsRealtime pour ignorer la pause.

🔧Exercice pratique

  1. Une coroutine de spawn : un ennemi toutes les 2 s, arrêtable avec StopCoroutine.
  2. Un effet de clignotement de 1 s quand un objet est « touché ».
  3. Un menu pause via Time.timeScale (0 / 1) avec un timer en temps réel qui continue.
Critères de réussite
  • Le jeu n'est jamais gelé pendant une attente.
  • La coroutine se lance avec StartCoroutine et s'arrête proprement.
  • La pause fonctionne sans casser le timer temps réel.

private Coroutine spawn;

📌À retenir
  • Unity est mono-thread : ne bloque jamais une frame pour « attendre ».
  • Une coroutine = IEnumerator + yield return, lancée par StartCoroutine.
  • WaitForSeconds (temps de jeu) vs WaitForSecondsRealtime (temps réel).
  • StopCoroutine/StopAllCoroutines pour arrêter ; Invoke pour un simple délai.

Glossaire des termes introduits

Coroutine — méthode pouvant se suspendre et reprendre au fil des frames.

IEnumerator / yield return — type de retour et instruction de « pause » d'une coroutine.

WaitForSeconds — instruction d'attente basée sur le temps de jeu.

Time.timeScale — facteur de vitesse du temps de jeu (0 = pause, 1 = normal).
