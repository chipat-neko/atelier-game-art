CS6 — Méthodes (fonctions)

Intermédiaire 40 min Piste 7 · C# (Unity)
PrérequisCS2 — Variables et CS4 — Conditions.
🎯Objectifs d'apprentissage

À la fin de cette leçon, tu sauras :

  • Déclarer et appeler une méthode (fonction).
  • Passer des paramètres et renvoyer une valeur (return).
  • Comprendre void, la portée des variables, et la surcharge.
  • Découper ton code en petites méthodes lisibles.

Pourquoi c'est important

Une méthode est un bloc de code nommé qu'on peut appeler autant de fois qu'on veut. Tu en utilises déjà : Start(), Update(), Debug.Log(...). Écrire tes méthodes, c'est : éviter de répéter, donner un nom clair à une action (InfligerDegats()), et garder un code rangé. C'est la base de tout code maintenable.

Théorie en profondeur

1. Déclarer et appeler

void Sauter()              // 'void' = ne renvoie rien
{
    Debug.Log("Le joueur saute !");
}

void Start()
{
    Sauter();              // APPEL : on exécute la méthode
    Sauter();              // on peut l'appeler autant de fois qu'on veut
}

Définir une méthode ne l'exécute pas : il faut l'appeler par son nom suivi de (). Start/Update sont juste des méthodes qu'Unity appelle pour toi (CS7).

2. Paramètres : donner des informations

Les paramètres (entre parenthèses) permettent de passer des valeurs à la méthode :

void InfligerDegats(int montant)
{
    vie -= montant;
    Debug.Log("Subit " + montant + " dégâts. Vie : " + vie);
}

// Appels avec des arguments différents :
InfligerDegats(10);
InfligerDegats(25);

On peut avoir plusieurs paramètres séparés par des virgules : void Soigner(int montant, bool effetVisuel).

3. Valeur de retour : récupérer un résultat

Si la méthode calcule quelque chose, on déclare le type de retour (au lieu de void) et on utilise return :

int CalculerScore(int ennemis, int bonus)
{
    int total = ennemis * 100 + bonus;
    return total;          // renvoie la valeur ET sort de la méthode
}

void Start()
{
    int monScore = CalculerScore(5, 50);   // récupère 550
    Debug.Log("Score : " + monScore);
}

// Une méthode qui renvoie un booléen se lit comme une question :
bool EstMort()
{
    return vie <= 0;
}

return renvoie la valeur et termine immédiatement la méthode (le code après est ignoré). C'est aussi utile pour sortir tôt : if (estInvincible) return; au début d'une méthode de dégâts.

4. Portée (scope) des variables

Une variable déclarée dans une méthode (variable locale) n'existe que là : les autres méthodes ne la voient pas. Pour une donnée partagée par toute la classe (vie, vitesse…), on la déclare au niveau de la classe (un champ, voir CS8).

5. Surcharge (overload)

On peut avoir plusieurs méthodes du même nom si leurs paramètres diffèrent : C# choisit la bonne selon les arguments. C'est la surcharge.

void Tirer() { /* tir par défaut */ }
void Tirer(int nombre) { /* tirer 'nombre' projectiles */ }
// Tirer();  et  Tirer(3);  appellent deux versions différentes.

🔧 Démonstration pas à pas

  1. Crée une méthode void InfligerDegats(int montant) qui réduit un champ vie et l'affiche.
  2. Crée une méthode bool EstMort() qui renvoie vie <= 0.
  3. Dans Start(), appelle InfligerDegats(60) deux fois, puis if (EstMort()) Debug.Log("Mort");.
  4. Refactorise : déplace l'affichage dans une méthode AfficherVie() appelée à chaque changement.

⚠️ Pièges & erreurs courantes

SymptômeMa méthode ne fait rien.
CauseTu l'as définie mais jamais appelée.
CorrectionAppelle-la (MaMethode();) depuis Start/Update ou une autre méthode.
Symptôme« not all code paths return a value ».
CauseUne méthode avec type de retour (≠ void) où un chemin n'a pas de return.
CorrectionAssure-toi que tous les chemins (if/else) renvoient une valeur.
Symptôme« The name 'x' does not exist in the current context ».
CauseTu utilises une variable locale en dehors de sa méthode (problème de portée).
CorrectionDéclare-la comme champ de la classe si plusieurs méthodes en ont besoin.
SymptômeLe code après mon return ne s'exécute jamais.
CauseC'est normal : return termine la méthode.
CorrectionPlace le code utile avant le return.

🔧Exercice pratique

  1. Écris int Additionner(int a, int b) et affiche Additionner(7, 8).
  2. Écris bool PeutAcheter(int prix, int argent) qui renvoie argent >= prix.
  3. Surcharge une méthode Soigner() (soin par défaut) et Soigner(int montant).
Critères de réussite
  • Tes méthodes renvoient les bonnes valeurs et sont appelées.
  • Tu distingues paramètre (déclaration) et argument (appel).
  • Ton code est découpé en méthodes au nom clair.

Exercices

Exercice 1Échauffement

Écris une méthode int CalculerScore(int base, int multi) qui renvoie base * multi, et appelle-la depuis Start.

Voir une solution 
int CalculerScore(int valeurBase, int multi)
{
    return valeurBase * multi;
}

void Start()
{
    Debug.Log(CalculerScore(10, 3));   // 30
}
Exercice 2Application

Crée deux surcharges Afficher : une pour un int, une pour un string. Appelle les deux et observe que le compilateur choisit la bonne.

Voir une solution 
void Afficher(int valeur) { Debug.Log("Entier : " + valeur); }
void Afficher(string texte) { Debug.Log("Texte : " + texte); }

void Start()
{
    Afficher(42);          // Entier : 42
    Afficher("Bonjour");   // Texte : Bonjour
}
Exercice 3Défi

Découpe une logique en plusieurs méthodes : une méthode bool EstVivant(int pv) et une méthode void Soigner avec un paramètre par défaut. Combine-les.

Voir une solution 
int pv = 0;

bool EstVivant(int valeur) => valeur > 0;

void Soigner(int montant = 10)
{
    pv += montant;
}

void Start()
{
    if (!EstVivant(pv))
        Soigner();          // soigne 10 par défaut
    Debug.Log("PV : " + pv + " — vivant : " + EstVivant(pv));
}
📌À retenir
  • Une méthode = un bloc nommé ; définir ≠ appeler (il faut l'appeler avec ()).
  • Les paramètres reçoivent des valeurs ; void = pas de retour, sinon return valeur;.
  • Une variable locale n'existe que dans sa méthode (portée) ; partage via un champ de classe.
  • Surcharge : même nom, paramètres différents. Découpe ton code en petites méthodes.

Glossaire des termes introduits

Méthode (fonction) — bloc de code nommé, appelable, qui peut prendre des paramètres et renvoyer une valeur.

Paramètre / argument — variable d'entrée d'une méthode (paramètre = déclaration, argument = valeur passée à l'appel).

return — renvoie une valeur et termine la méthode.

Portée (scope) — zone où une variable existe et est accessible.

Surcharge — plusieurs méthodes de même nom différenciées par leurs paramètres.

Voir aussi le glossaire global.
