CS11 — Manipuler GameObjects & Components

Intermédiaire 45 min Piste 7 · C# (Unity)
PrérequisCS8 — Classes, CS7 et U2 — Éditeur.
🎯Objectifs d'apprentissage

À la fin de cette leçon, tu sauras :

  • Accéder à un autre Component avec GetComponent<T>().
  • Lire/modifier la position via transform.
  • Créer et détruire des objets : Instantiate, Destroy.
  • Référencer proprement des objets (Inspector vs Find) et utiliser tags/layers.

Pourquoi c'est important

Jusqu'ici, tes scripts vivaient « en vase clos ». Le vrai gameplay émerge quand un script agit sur la scène : déplacer un objet, lui retirer de la vie via son script, faire apparaître un projectile, détruire un ennemi. Tout cela passe par les GameObjects et leurs Components (rappel U2). Cette leçon est le pont entre le C# et le moteur.

Théorie en profondeur

1. transform : la position de l'objet

Tout script attaché accède à transform (le Component Transform de son propre objet), qui porte position, rotation et échelle :

transform.position = new Vector3(0f, 5f, 0f);   // téléporter
transform.Translate(Vector3.up * 2f);           // déplacer relativement
transform.Rotate(Vector3.up * 90f);             // tourner de 90°
Debug.Log(transform.position.y);                // lire la hauteur

Vector3 (x, y, z) est le type des positions/directions (vu en CS2). Raccourcis utiles : Vector3.up, Vector3.forward, Vector3.zero.

2. GetComponent : accéder à un autre Component

Pour agir sur un autre Component du même objet (ou d'un autre), on le récupère avec GetComponent<Type>() :

// Récupérer le Rigidbody de mon objet
Rigidbody rb = GetComponent<Rigidbody>();
rb.linearVelocity = Vector3.forward * 5f;

// Récupérer le script d'un AUTRE objet (via une référence) :
Ennemi e = autreObjet.GetComponent<Ennemi>();
if (e != null) e.SubirDegats(10);    // null-check : et s'il n'a pas ce script ?
💡Mets GetComponent en cache

GetComponent a un coût : ne l'appelle pas dans Update à chaque frame. Récupère la référence une fois dans Awake/Start et stocke-la dans un champ (renvoi CS15).

3. Instantiate & Destroy : créer et détruire

public GameObject prefabProjectile;   // glissé depuis le Project dans l'Inspector

void Tirer()
{
    // crée une COPIE du prefab à la position/rotation données
    GameObject balle = Instantiate(prefabProjectile, transform.position, transform.rotation);
    Destroy(balle, 3f);   // détruit la balle après 3 secondes
}

void OnDejaTouche()
{
    Destroy(gameObject);  // 'gameObject' = l'objet qui porte CE script
}

Instantiate clone un Prefab (un modèle d'objet réutilisable). Destroy(gameObject) détruit l'objet courant ; Destroy(obj, delai) attend avant de détruire.

4. Référencer un objet : Inspector vs Find

Pour qu'un script connaisse un autre objet, deux voies :

  • Référence sérialisée (recommandé) : un champ [SerializeField] private Transform cible; que tu glisses dans l'Inspector. Rapide, sûr, explicite.
  • Recherche à l'exécution : GameObject.FindWithTag("Player") ou GameObject.Find("Nom"). Pratique mais lent et fragile (dépend du nom) — à éviter dans Update.

Les tags (étiquettes) et layers (couches) classent les objets. On compare un tag avec CompareTag("Enemy") (plus sûr que == "Enemy").

🔧 Démonstration pas à pas

  1. Crée un script Deplacement sur un cube : dans Update, transform.Translate(Vector3.right * 2f * Time.deltaTime); — le cube glisse.
  2. Crée un Prefab « Balle » (un petit objet). Dans un autre script, expose public GameObject prefabBalle;, glisse le prefab dans l'Inspector, et fais Instantiate(prefabBalle, transform.position, Quaternion.identity); au Start.
  3. Ajoute Destroy(gameObject, 5f); sur la balle pour qu'elle disparaisse.
  4. Référence un autre objet via [SerializeField] private Transform cible; (glissé dans l'Inspector) et fais ton cube regarder la cible : transform.LookAt(cible);.

⚠️ Pièges & erreurs courantes

SymptômeNullReferenceException sur un GetComponent.
CauseL'objet n'a pas ce Component : GetComponent renvoie null.
CorrectionVérifie le résultat (if (rb != null)) ou assure-toi que le Component est bien là.
SymptômeMon Instantiate ne crée rien / erreur de référence.
CauseLe champ prefab n'a pas été assigné dans l'Inspector (vaut null).
CorrectionGlisse le prefab dans le champ de l'Inspector.
SymptômeMon jeu rame avec beaucoup d'objets.
CauseGetComponent/Find appelés à chaque frame dans Update.
CorrectionMets ces références en cache dans Awake/Start (renvoi CS15).
Symptôme« MissingReferenceException » après destruction.
CauseTu utilises un objet déjà détruit par Destroy.
CorrectionVérifie qu'il existe encore (if (obj != null)) avant de l'utiliser.

🔧Exercice pratique

  1. Un script qui, au clic (ou au Start), instancie un prefab et le détruit après 2 s.
  2. Un script « ennemi » qui récupère le Rigidbody en cache (Awake) et l'utilise.
  3. Une référence à une cible glissée dans l'Inspector ; ton objet la regarde (LookAt) en continu.
Critères de réussite
  • L'instanciation et la destruction fonctionnent sans erreur null.
  • Tu mets GetComponent en cache plutôt que dans Update.
  • Tu références par l'Inspector plutôt que par Find.
📌À retenir
  • transform = position/rotation/échelle de l'objet ; Vector3 pour les coordonnées.
  • GetComponent<T>() accède aux autres Components — mets-le en cache, jamais dans Update.
  • Instantiate(prefab, pos, rot) crée ; Destroy(obj[, delai]) détruit (gameObject = l'objet courant).
  • Référence via l'Inspector (recommandé) plutôt que Find ; compare un tag avec CompareTag.

Glossaire des termes introduits

transform — Component portant position, rotation et échelle d'un objet.

GetComponent<T>() — récupère un Component d'un type donné sur un objet.

Instantiate / Destroy — créer une copie d'un objet/prefab / le détruire.

Référence sérialisée — champ exposé dans l'Inspector où l'on glisse un objet.
