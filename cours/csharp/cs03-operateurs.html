CS3 — Opérateurs & expressions

Débutant 30 min Piste 7 · C# (Unity)
PrérequisCS2 — Variables & types.
🎯Objectifs d'apprentissage

À la fin de cette leçon, tu sauras :

  • Calculer avec les opérateurs arithmétiques (et le piège de la division d'entiers).
  • Comparer des valeurs (==, !=, <, >…) et combiner avec la logique (&&, ||, !).
  • Utiliser l'affectation composée (+=, ++…).
  • Faire des conversions et casts entre types.

Pourquoi c'est important

Une expression est un calcul qui produit une valeur : vie - degats, score > recordPrecedent. Les opérateurs sont les briques de toute logique de jeu : infliger des dégâts, vérifier une condition de victoire, faire avancer un objet. Deux pièges célèbres y vivent (la division d'entiers et le ==/=) : les connaître t'épargne des heures.

Théorie en profondeur

1. Opérateurs arithmétiques

int a = 10, b = 3;
int somme   = a + b;   // 13
int diff    = a - b;   // 7
int produit = a * b;   // 30
int quotient = a / b;  // 3  ← division d'ENTIERS : la partie décimale est jetée !
int reste   = a % b;   // 1  ← modulo : le reste de la division (10 = 3*3 + 1)

Le modulo % est précieux en jeu : compteur % 2 == 0 teste la parité, et il sert à « boucler » un index (revenir à 0 après le dernier). Le piège : 10 / 3 entre deux int donne 3, pas 3.33. Pour une vraie division, au moins un opérande doit être float :

float precis = 10f / 3f;     // 3.333...
float aussi  = (float)a / b; // on convertit a en float d'abord

2. Affectation et affectation composée

= est l'affectation (mettre une valeur dans une boîte). Les versions composées raccourcissent « modifie-toi avec… » :

score += 10;   // équivaut à : score = score + 10;
vie   -= 25;   // vie = vie - 25;
mult  *= 2;    // mult = mult * 2;
compteur++;    // ajoute 1 (incrément)
compteur--;    // retire 1 (décrément)

3. Comparaison et logique (résultats booléens)

Les opérateurs de comparaison renvoient un bool (true/false) — la base des conditions (CS4) :

bool gagne   = score >= 100;     // supérieur ou égal
bool mort    = vie == 0;         // ÉGALITÉ : deux signes = !
bool different = nom != "Boss";  // différent de
bool vivant  = !mort;            // NON logique : inverse le booléen

// Combiner plusieurs conditions :
bool peutTirer = aMunitions && !estEnRechargement;  // ET : les deux doivent être vrais
bool gameOver  = vie <= 0 || tempsEcoule;            // OU : au moins un vrai

Distinction vitale : == compare (test d'égalité), = affecte (range une valeur). Écrire if (vie = 0) au lieu de if (vie == 0) est une faute classique (heureusement, le compilateur C# la refuse souvent). Le && et le || sont « paresseux » : si le premier opérande suffit à décider, le second n'est pas évalué (utile et performant).

4. Conversions et casts

  • Implicite (sans risque de perte) : float f = monInt; — un int rentre dans un float automatiquement.
  • Explicite (cast) (risque de perte → tu dois le demander) : int i = (int)monFloat; — la partie décimale est tronquée ((int)3.9f donne 3, pas 4).
  • Texte ⇄ nombre : int.Parse("42") (texte → entier), monInt.ToString() (entier → texte). Pour Mathf et l'arrondi, voir CS12 (Mathf.RoundToInt).
float f = 3.9f;
int tronque = (int)f;          // 3  (tronqué, pas arrondi)
int arrondi = Mathf.RoundToInt(f); // 4 (Mathf vient d'Unity)
string txt = "Vie : " + tronque;   // concaténation : "Vie : 3"

Le + avec une string fait de la concaténation (colle du texte) — c'est ce qu'on a utilisé dans les Debug.Log.

🖐Essaie maintenant~1 min

De tête : 3 + 4 * 2, (3 + 4) * 2, 10 % 3 (11, 14, 1). Puis bool b = (5 > 3) && (2 > 4); : vrai ou faux ? (Faux — le && exige les deux.)

🔧 Démonstration pas à pas

  1. Dans un script, déclare int vie = 100; et int degats = 30;. Dans Start(), fais vie -= degats; puis Debug.Log("Vie : " + vie);.
  2. Teste la division : affiche 7 / 2 (int) puis 7f / 2f (float) et compare les résultats dans la Console.
  3. Calcule un booléen : bool estMort = vie <= 0; et affiche-le.
  4. Convertis : float t = 4.7f; puis affiche (int)t et Mathf.RoundToInt(t) pour voir la différence troncature/arrondi.

⚠️ Pièges & erreurs courantes

SymptômeUne moyenne ou un ratio donne toujours un nombre rond bizarre.
CauseDivision entre deux int → résultat tronqué.
CorrectionConvertis au moins un opérande en float ((float)a / b).
SymptômeMa condition d'égalité ne se comporte pas comme prévu.
CauseConfusion = (affecter) et == (comparer).
CorrectionPour tester l'égalité, utilise toujours ==.
SymptômeComparer deux float avec == échoue parfois alors qu'ils « devraient » être égaux.
CauseLes float ont des imprécisions (ex. 0.1f + 0.2f ≠ exactement 0.3f).
CorrectionCompare avec une tolérance : Mathf.Abs(a - b) < 0.001f, ou Mathf.Approximately(a, b).
Symptôme« Cannot implicitly convert type 'float' to 'int' ».
CauseTu ranges un float dans un int sans cast (perte possible).
CorrectionFais un cast explicite (int) ou Mathf.RoundToInt(...).

🔧Exercice pratique

  1. Simule une attaque : vie commence à 100, applique 3 coups de 25 avec -=, affiche la vie après chaque coup.
  2. Calcule le pourcentage de vie restante (attention au type !) et affiche-le.
  3. Écris un booléen peutAgir = estVivant && !estEtourdi; et teste-le avec différentes valeurs.
Critères de réussite
  • Le pourcentage de vie est correct (tu as géré la division float).
  • Tu n'as confondu ni =/==, ni division entière/flottante.
  • Tu sais quand un cast est nécessaire.

À partir de pv = 73 et pvMax = 100 (deux int), calcule le pourcentage de vie en flottant et arrondis-le avec Mathf.RoundToInt.

Voir une solution

📌À retenir
  • Arithmétique : + - * / % ; division d'int = tronquée (passe en float pour un vrai ratio).
  • Affectation composée : += -= *= ++ --.
  • Comparaison == (≠ =) et logique && || ! renvoient des bool.
  • Conversions : int→float auto ; float→int via cast (int) (tronque) ou Mathf.RoundToInt.

Glossaire des termes introduits

Expression — calcul qui produit une valeur (ex. vie - degats).

Modulo (%) — reste d'une division entière ; sert à tester la parité ou à boucler un index.

Cast — conversion explicite d'un type vers un autre (ex. (int)monFloat).

Concaténation — collage de texte avec l'opérateur + sur des string.
