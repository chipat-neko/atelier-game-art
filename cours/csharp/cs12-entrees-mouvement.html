CS12 — Entrées & mouvement

Intermédiaire 45 min Piste 7 · C# (Unity)
PrérequisCS7 — MonoBehaviour et CS11.
🎯Objectifs d'apprentissage

À la fin de cette leçon, tu sauras :

  • Lire le clavier et la souris avec la classe Input.
  • Déplacer un objet avec transform.Translate (et Time.deltaTime).
  • Bouger via la physique (Rigidbody) dans FixedUpdate.
  • Distinguer « touche pressée » (continu) et « vient d'être pressée » (instant).

Pourquoi c'est important

Le contrôle du joueur, c'est le premier « ça marche ! » d'un débutant : lire une touche, faire avancer un personnage. On combine ici tout le cours : variables (vitesse), Update vs FixedUpdate (CS7), transform/Rigidbody (CS11) et Time.deltaTime.

ℹ️Deux systèmes d'entrées

Unity propose l'ancien Input (simple, idéal pour apprendre — utilisé ici) et le nouveau Input System (package, plus puissant pour les manettes/remapping). Pour débuter, la classe Input suffit ; il faut juste que le projet l'autorise (Project Settings ▸ Player ▸ Active Input Handling = Both ou Input Manager).

Théorie en profondeur

1. Lire le clavier

void Update()
{
    if (Input.GetKey(KeyCode.Space))        // VRAI tant que Espace est maintenue
        Debug.Log("Espace maintenue");

    if (Input.GetKeyDown(KeyCode.Space))    // VRAI seulement à l'instant du clic
        Debug.Log("Saut !");                // → idéal pour sauter/tirer

    // Axes : -1 à +1 (flèches / ZQSD / WASD), lissés
    float h = Input.GetAxis("Horizontal");  // gauche/droite
    float v = Input.GetAxis("Vertical");    // avant/arrière
}

Distinction clé : GetKey est vrai tant que la touche est tenue (mouvement continu) ; GetKeyDown est vrai seulement la frame où elle est enfoncée (action ponctuelle : saut, tir). GetAxis renvoie une valeur de -1 à 1 (pratique pour le déplacement).

2. Déplacer avec transform (simple)

On lit l'entrée dans Update et on déplace en multipliant par vitesse * Time.deltaTime (CS7) :

public float vitesse = 5f;

void Update()
{
    float h = Input.GetAxis("Horizontal");
    float v = Input.GetAxis("Vertical");

    Vector3 direction = new Vector3(h, 0f, v);   // un vecteur de déplacement
    transform.Translate(direction * vitesse * Time.deltaTime);
}

Ce mouvement « direct » ignore la physique (l'objet traverse les murs). Parfait pour un prototype ou un objet sans collision.

3. Déplacer avec la physique (Rigidbody)

Pour un personnage qui respecte les collisions et la gravité, on ajoute un Rigidbody et on agit sur lui dans FixedUpdate (rappel CS7 : la physique va dans FixedUpdate) :

public float vitesse = 5f;
private Rigidbody rb;
private Vector3 input;

void Awake()
{
    rb = GetComponent<Rigidbody>();   // mise en cache (CS11/CS15)
}

void Update()  // lire les entrées ici (réactif)
{
    input = new Vector3(Input.GetAxis("Horizontal"), 0f, Input.GetAxis("Vertical"));
}

void FixedUpdate()  // appliquer la physique ici
{
    rb.MovePosition(rb.position + input * vitesse * Time.fixedDeltaTime);
}

On lit l'entrée dans Update (le plus réactif) et on applique le mouvement physique dans FixedUpdate. Pour un saut : rb.AddForce(Vector3.up * forceSaut, ForceMode.Impulse); quand GetKeyDown(Space) et que le personnage touche le sol.

4. La souris

if (Input.GetMouseButtonDown(0))   // 0 = clic gauche, 1 = droit
    Debug.Log("Clic !");

float sourisX = Input.GetAxis("Mouse X");   // déplacement souris (caméra/visée)

🔧 Démonstration pas à pas

  1. Crée un sol (plane) et un cube. Sur le cube, ajoute un Rigidbody.
  2. Crée un script Joueur : déplace le cube avec GetAxis (version transform OU Rigidbody). Règle vitesse dans l'Inspector.
  3. Ajoute un saut : si GetKeyDown(KeyCode.Space), applique une force vers le haut au Rigidbody.
  4. Vérifie que la vitesse est cohérente quel que soit le framerate (Time.deltaTime / fixedDeltaTime).

⚠️ Pièges & erreurs courantes

Symptôme« InvalidOperationException … Input System » ou Input ne marche pas.
CauseLe projet est en « Input System (New) » seul ; l'ancienne classe Input est désactivée.
CorrectionProject Settings ▸ Player ▸ Active Input Handling = Both.
SymptômeMon saut se déclenche plusieurs fois / en continu.
CauseTu utilises GetKey (continu) au lieu de GetKeyDown (instant).
CorrectionUtilise GetKeyDown pour les actions ponctuelles.
SymptômeMon personnage traverse les murs.
CauseTu bouges via transform (ignore la physique) au lieu d'un Rigidbody.
CorrectionUtilise un Rigidbody + colliders et bouge dans FixedUpdate.
SymptômeLe mouvement physique est saccadé.
CauseCode physique dans Update au lieu de FixedUpdate.
CorrectionApplique les forces/MovePosition dans FixedUpdate (renvoi CS7).

🔧Exercice pratique

  1. Déplace un cube au clavier (ZQSD/flèches) à vitesse réglable, framerate-indépendant.
  2. Ajoute un saut avec GetKeyDown(Space) + une force (Rigidbody).
  3. Affiche « Tir ! » au clic gauche (GetMouseButtonDown(0)).
Critères de réussite
  • Le déplacement est fluide et cohérent quel que soit le framerate.
  • Le saut se déclenche une seule fois par appui.
  • Tu lis l'entrée dans Update et appliques la physique dans FixedUpdate.

Glossaire des termes introduits

Input — classe Unity (ancien système) pour lire clavier, souris, manette.

GetKey / GetKeyDown — touche maintenue (continu) / vient d'être pressée (instant).

GetAxis — entrée d'axe lissée renvoyant une valeur de -1 à +1.

Rigidbody — Component de physique ; on le pilote dans FixedUpdate.
