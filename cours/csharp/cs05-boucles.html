CS5 — Boucles (for, while, foreach)

Débutant 35 min Piste 7 · C# (Unity)
PrérequisCS4 — Conditions.
🎯Objectifs d'apprentissage

À la fin de cette leçon, tu sauras :

  • Répéter un bloc avec for, while, do-while.
  • Parcourir une collection avec foreach.
  • Contrôler le flux avec break et continue.
  • Éviter la boucle infinie qui gèle Unity.

Pourquoi c'est important

Dès qu'on doit répéter une action — faire apparaître 10 ennemis, parcourir tous les objets ramassés, additionner des dégâts — on utilise une boucle. C'est ce qui évite de copier-coller 10 fois la même ligne. Mais attention : une boucle qui ne se termine jamais fige Unity (le jeu et l'éditeur « plantent » dans la même frame). On apprend donc aussi à les borner.

Théorie en profondeur

1. La boucle for : « répète N fois »

Le for regroupe trois choses : un compteur de départ, une condition de continuation, et une mise à jour :

for (int i = 0; i < 10; i++)
{
    Debug.Log("Ennemi numéro " + i);   // affiche 0 à 9
}
// 1) int i = 0   → initialisation (une fois)
// 2) i < 10      → condition testée AVANT chaque tour
// 3) i++         → exécutée APRÈS chaque tour

Le compteur i commence souvent à 0 (les index commencent à 0 en programmation, voir CS10). i < 10 fait donc 10 tours (0,1,…,9).

2. while : « tant que »

int munitions = 5;
while (munitions > 0)
{
    Tirer();
    munitions--;        // INDISPENSABLE : sans ça, boucle infinie !
}

Le while répète tant que sa condition est vraie. Il faut que quelque chose, dans le bloc, fasse évoluer la condition vers « faux » — sinon la boucle tourne pour toujours. Le do-while est identique mais teste la condition après le bloc (donc s'exécute au moins une fois).

⚠️La boucle infinie gèle Unity

Une boucle while/for s'exécute entièrement dans la frame courante. Si elle ne se termine pas, Unity se fige (parfois il faut tuer le processus). Pour étaler une répétition dans le temps (un tour par frame/seconde), on n'utilise PAS une boucle mais Update() (CS7) ou une coroutine (CS14).

3. foreach : parcourir une collection

Pour passer sur chaque élément d'un tableau ou d'une liste (CS10), foreach est le plus simple et lisible :

foreach (string nom in noms)   // « pour chaque nom dans noms »
{
    Debug.Log(nom);
}

Pas de compteur à gérer : foreach te donne directement chaque élément. (À noter : on ne modifie pas la collection pendant qu'on la parcourt en foreach — sinon erreur.)

4. break et continue

  • break; — sort immédiatement de la boucle (ex. on a trouvé ce qu'on cherchait).
  • continue; — saute le reste du tour courant et passe au suivant.
for (int i = 0; i < ennemis.Length; i++)
{
    if (ennemis[i] == null) continue;   // ignore les cases vides
    if (ennemis[i].estBoss) { Debug.Log("Boss trouvé !"); break; }  // stop
}

🖐Essaie maintenant~1 min

Prédis for (int i = 0; i < 3; i++) Debug.Log(i); (0, 1, 2). Change la condition en i <= 3 : combien d'itérations maintenant ? (4.)

🔧 Démonstration pas à pas

  1. Dans Start(), écris un for qui affiche « Vague 1 » à « Vague 5 ».
  2. Crée un int munitions = 3; et une boucle while qui « tire » (affiche) jusqu'à épuisement — n'oublie pas munitions--.
  3. Ajoute un if dans le for qui continue; sur les nombres pairs (avec i % 2 == 0) pour n'afficher que les impairs.

⚠️ Pièges & erreurs courantes

SymptômeUnity se fige complètement quand je lance.
CauseBoucle infinie : la condition ne devient jamais fausse (oubli de i++ ou munitions--).
CorrectionAssure-toi que le corps fait évoluer la condition ; pour étaler dans le temps, utilise Update/coroutine.
Symptôme« IndexOutOfRangeException » dans une boucle.
CauseTu dépasses la taille (ex. i <= length au lieu de i < length).
CorrectionLes index vont de 0 à length - 1 : utilise i < collection.Length.
Symptôme« Collection was modified » pendant un foreach.
CauseTu ajoutes/retires des éléments pendant le foreach.
CorrectionParcours avec un for à l'envers, ou collecte d'abord puis modifie après.
SymptômeJe veux faire « un tour par seconde » mais tout se passe en une frame.
CauseUne boucle s'exécute entièrement dans la frame courante.
CorrectionUtilise une coroutine avec yield return new WaitForSeconds(1f); (CS14).

🔧Exercice pratique

  1. Avec un for, calcule la somme des entiers de 1 à 100 et affiche-la.
  2. Avec un while, simule un compte à rebours de 5 à 1 (affiché), puis « Go ! ».
  3. Avec break, parcours un tableau de scores et arrête-toi dès qu'un score dépasse 1000.
Critères de réussite
  • La somme 1→100 vaut 5050.
  • Aucune boucle ne fige Unity (toutes se terminent).
  • Tu sais quand préférer for, while ou foreach.

Exercice 1Échauffement

Avec une boucle for, calcule la somme des entiers de 1 à 100 et affiche-la.

Voir une solution 
int somme = 0;
for (int i = 1; i <= 100; i++)
    somme += i;
Debug.Log(somme);   // 5050
Exercice 2Application

Parcours une liste de PV avec foreach et compte combien d'éléments sont « morts » (≤ 0).

Voir une solution 
int[] pvs = { 30, 0, 12, -5, 8 };
int morts = 0;
foreach (int pv in pvs)
{
    if (pv <= 0)
        morts++;
}
Debug.Log("Morts : " + morts);   // 2
Exercice 3Défi

Dans une boucle, trouve le premier nombre positif d'un tableau en sautant les valeurs nulles avec continue et en sortant avec break.

Voir une solution 
int[] valeurs = { 0, 0, -3, 7, 9 };
int trouve = 0;
foreach (int v in valeurs)
{
    if (v <= 0) continue;   // saute les non-positifs
    trouve = v;
    break;                  // premier positif trouvé
}
Debug.Log(trouve);          // 7

📌À retenir
  • for = « répète N fois » (init ; condition ; mise à jour). Les index commencent à 0.
  • while = « tant que » — fais évoluer la condition, sinon boucle infinie.
  • foreach = parcourir une collection sans gérer d'index.
  • break sort, continue saute au tour suivant ; pour étaler dans le temps → Update/coroutine.

Glossaire des termes introduits

Boucle — structure qui répète un bloc de code.

Itération — un tour de boucle.

Boucle infinie — boucle dont la condition ne devient jamais fausse ; fige le programme.

break / continue — sortir de la boucle / sauter au tour suivant.
