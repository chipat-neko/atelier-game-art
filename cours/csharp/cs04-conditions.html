CS4 — Conditions (if, switch)

Débutant 35 min Piste 7 · C# (Unity)
PrérequisCS3 — Opérateurs (les booléens).
🎯Objectifs d'apprentissage

À la fin de cette leçon, tu sauras :

  • Brancher ton code avec if / else if / else.
  • Écrire un choix court avec l'opérateur ternaire.
  • Gérer plusieurs cas nets avec switch.
  • Éviter les pièges (accolades, == vs =, ordre des conditions).

Pourquoi c'est important

Un jeu prend sans arrêt des décisions : « si la vie atteint 0, alors mort », « si la touche Espace est pressée, alors saute ». La condition est le mécanisme qui exécute un bloc de code seulement si un test (un bool, voir CS3) est vrai. C'est le cœur de toute logique de gameplay.

Théorie en profondeur

1. if / else if / else

if (vie <= 0)
{
    Debug.Log("Game Over");
}
else if (vie < 30)
{
    Debug.Log("Attention, vie basse !");
}
else
{
    Debug.Log("Tout va bien.");
}

Lecture : si la première condition est vraie, on exécute son bloc et on s'arrête ; sinon on teste la suivante (else if) ; le else final attrape « tous les autres cas ». L'ordre compte : une condition large placée trop tôt « avale » les cas plus précis (place le plus spécifique d'abord).

💡Mets toujours les accolades

C# autorise if (x) faireUnTruc(); sans accolades, mais c'est une source de bugs (on ajoute une 2ᵉ ligne en croyant qu'elle est dans le if). Garde les { } même pour une seule instruction.

2. Tester un booléen directement

Une condition attend un bool. Inutile d'écrire == true :

if (estEnVie)        { /* ... */ }   // au lieu de : if (estEnVie == true)
if (!estEnVie)       { /* ... */ }   // « si PAS en vie »
if (aMunitions && !enRechargement) { Tirer(); }   // conditions combinées

3. L'opérateur ternaire (choix court)

Pour choisir une valeur selon une condition, le ternaire condition ? siVrai : siFaux condense un if/else :

string etat = (vie > 0) ? "Vivant" : "Mort";
float vitesse = estAccroupi ? 2f : 5f;

Pratique pour les cas simples ; au-delà, un if/else reste plus lisible.

4. switch : plusieurs cas nets

Quand on compare une même variable à plusieurs valeurs précises, switch est plus clair qu'une cascade de if :

switch (etatJeu)
{
    case "Menu":
        AfficherMenu();
        break;          // break OBLIGATOIRE pour sortir du case
    case "EnJeu":
        Jouer();
        break;
    case "Pause":
        MettreEnPause();
        break;
    default:            // tous les autres cas
        Debug.Log("État inconnu");
        break;
}

Chaque case se termine par break; (sinon erreur en C#). Le default couvre le reste. Le switch marche sur les int, string, enum (énumérations — un type « liste de valeurs nommées », très utilisé pour les états de jeu).

🔧 Démonstration pas à pas

  1. Crée un script avec [SerializeField] private int vie = 100;.
  2. Dans Start(), écris la cascade if/else if/else ci-dessus pour afficher l'état selon la vie.
  3. Règle vie à 0, puis 20, puis 80 dans l'Inspector et relance à chaque fois : vérifie le bon message.
  4. Remplace l'affichage de l'état par un ternaire et compare la lisibilité.

🔧Exercice pratique

  1. Crée un système de « grade » selon un int score : ≥90 « S », ≥75 « A », ≥50 « B », sinon « C ». Affiche le grade.
  2. Réécris-le avec un switch sur un int niveau (1, 2, 3, default).
  3. Ajoute un ternaire qui choisit un message « Victoire ! » / « Réessaie » selon un bool aGagne.
Critères de réussite
  • Chaque tranche de score donne le bon grade (ordre correct).
  • Ton switch a un break; par case et un default.
  • Le ternaire produit le bon message.

📌À retenir
  • if / else if / else = arbre de décision ; une seule branche s'exécute ; l'ordre compte.
  • Teste un bool directement (if (estEnVie), if (!x)) ; garde les accolades.
  • Le ternaire c ? a : b choisit une valeur courte.
  • switch pour plusieurs cas nets d'une même variable ; break; obligatoire, default pour le reste.

Glossaire des termes introduits

Condition — test (booléen) qui décide si un bloc de code s'exécute.

Opérateur ternaire — condition ? valeurSiVrai : valeurSiFaux, un if/else condensé en expression.

switch — structure comparant une variable à plusieurs valeurs (cases) avec un default.

enum — type « liste de valeurs nommées » (ex. états de jeu), idéal avec switch.
