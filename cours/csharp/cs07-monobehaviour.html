CS7 — Le cycle de vie MonoBehaviour

Intermédiaire 45 min Piste 7 · C# (Unity)
PrérequisCS6 — Méthodes et CS1.
🎯Objectifs d'apprentissage

À la fin de cette leçon, tu sauras :

  • Connaître les méthodes que Unity appelle : Awake, Start, Update, FixedUpdate, LateUpdate.
  • Savoir quoi mettre où (init dans Awake/Start, logique par image dans Update).
  • Utiliser Time.deltaTime pour un mouvement indépendant du framerate.
  • Distinguer Update (logique) et FixedUpdate (physique).

Pourquoi c'est important

Tu n'appelles jamais Start() ni Update() toi-même : c'est Unity qui les appelle automatiquement, à des moments précis du cycle de vie du script. Comprendre quand chaque méthode tourne, c'est savoir où placer ton code — et éviter des bugs classiques comme « ma variable est null au démarrage » ou « mon mouvement va deux fois plus vite sur un PC rapide ».

Théorie en profondeur

1. Les méthodes d'événement principales

MéthodeQuand Unity l'appelleÀ y mettre
Awake()Une fois, à la création de l'objet (avant tout)Initialiser ses propres références (GetComponent sur soi).
OnEnable()Chaque fois que l'objet/script est activéS'abonner à des événements (CS13).
Start()Une fois, avant la première Update (après tous les Awake)Init dépendant des autres objets (déjà prêts).
Update()Chaque image (frame)Entrées, logique de jeu, déplacements simples.
FixedUpdate()À intervalle fixe (indépendant du framerate)Tout ce qui touche la physique (Rigidbody).
LateUpdate()Chaque image, après tous les UpdateCaméra qui suit le joueur (après son déplacement).
OnDisable() / OnDestroy()Désactivation / destructionSe désabonner, nettoyer.

2. Awake vs Start : l'ordre qui sauve des bugs

Unity appelle tous les Awake() de la scène avant n'importe quel Start(). Conséquence pratique : dans Awake, initialise ce qui te concerne toi ; dans Start, accède aux autres objets (ils ont déjà fait leur Awake, donc sont prêts). C'est la cause n°1 d'un NullReferenceException au lancement : chercher un autre script dans Awake alors qu'il n'est pas encore initialisé.

3. Time.deltaTime : un mouvement régulier

Update() est appelée à chaque image : sur un PC à 120 fps, deux fois plus souvent qu'à 60 fps. Si tu déplaces « de 5 par image », l'objet ira deux fois plus vite sur la machine rapide ! La solution : multiplier par Time.deltaTime (le temps écoulé depuis l'image précédente), pour raisonner « par seconde » :

public float vitesse = 5f;   // unités PAR SECONDE

void Update()
{
    // avance de 'vitesse' unités par seconde, quel que soit le framerate
    transform.Translate(Vector3.forward * vitesse * Time.deltaTime);
}

Règle : tout ce qui bouge/évolue dans Update se multiplie par Time.deltaTime. (On détaille le mouvement en CS12.)

4. Update vs FixedUpdate

Update suit le framerate (variable) ; FixedUpdate tourne à intervalle fixe (par défaut 50×/s), synchronisé avec le moteur physique. Règle : la physique (forces, vélocité d'un Rigidbody) va dans FixedUpdate ; la lecture des entrées et la logique vont dans Update (renvoi CS12).

🔧 Démonstration pas à pas

  1. Crée un script avec un Debug.Log dans Awake, Start et Update. Attache-le, Play : observe l'ordre (Awake → Start, puis Update en boucle).
  2. Ajoute un champ public float vitesse = 90f; et dans Update() fais tourner l'objet : transform.Rotate(Vector3.up * vitesse * Time.deltaTime);.
  3. Enlève * Time.deltaTime et compare la vitesse de rotation : tu verras qu'elle dépend du framerate (incohérent).

⚠️ Pièges & erreurs courantes

🔧Exercice pratique

  1. Fais tourner un cube sur lui-même à vitesse constante, framerate-indépendante (Time.deltaTime).
  2. Ajoute un compteur de temps : dans Update, tempsEcoule += Time.deltaTime; et affiche-le toutes les secondes (avec un test).
  3. Mets des Debug.Log dans Awake/Start/Update/LateUpdate et décris l'ordre observé.
Critères de réussite
  • La rotation est identique quel que soit le framerate.
  • Tu sais expliquer ce qui va dans Awake, Start, Update, FixedUpdate, LateUpdate.
  • Tu comprends pourquoi Time.deltaTime est indispensable.
📌À retenir
  • Unity appelle ces méthodes pour toi : Awake → OnEnable → Start (une fois), puis Update/FixedUpdate/LateUpdate chaque frame.
  • Awake = init de soi ; Start = accès aux autres (déjà prêts).
  • Tout mouvement dans Update se multiplie par Time.deltaTime (par seconde).
  • Physique → FixedUpdate ; caméra qui suit → LateUpdate.

Glossaire des termes introduits

Cycle de vie (MonoBehaviour) — suite ordonnée de méthodes qu'Unity appelle automatiquement.

Time.deltaTime — temps écoulé depuis l'image précédente ; rend le mouvement indépendant du framerate.

FixedUpdate — méthode appelée à intervalle fixe, pour le code physique.

LateUpdate — méthode appelée après tous les Update (caméra de suivi).
