CS10 — Collections (List, Dictionary, tableaux)

Intermédiaire 40 min Piste 7 · C# (Unity)
PrérequisCS5 — Boucles (foreach) et CS8.
🎯Objectifs d'apprentissage

À la fin de cette leçon, tu sauras :

  • Utiliser un tableau (taille fixe) et accéder par index.
  • Utiliser une List<T> (taille dynamique) : Add, Remove, Count.
  • Utiliser un Dictionary<K,V> (recherche par clé).
  • Choisir la bonne collection selon le besoin.

Pourquoi c'est important

Un jeu manipule des groupes de choses : tous les ennemis actifs, l'inventaire, les scores du leaderboard. Une collection stocke plusieurs valeurs dans une seule variable et permet de les parcourir (foreach, CS5). Choisir la bonne — tableau, liste, dictionnaire — change la simplicité et la performance de ton code.

Théorie en profondeur

1. Le tableau (array) : taille fixe

Un tableau contient un nombre fixé à la création d'éléments du même type, accessibles par index (de 0 à Length - 1) :

int[] scores = new int[3];   // 3 cases, valent 0 par défaut
scores[0] = 50;              // accès par index (commence à 0 !)
scores[1] = 90;

string[] noms = { "Léa", "Max", "Zoé" };   // création + valeurs
Debug.Log(noms[2]);          // "Zoé"
Debug.Log(noms.Length);      // 3

On ne peut pas ajouter/retirer d'élément (taille figée). Pour une collection qui grandit, on utilise une List.

2. List<T> : taille dynamique

La List est une collection qui s'agrandit/se rétrécit à volonté. Le <T> précise le type des éléments (générique). Il faut using System.Collections.Generic; en haut du fichier :

using System.Collections.Generic;

List<string> ennemis = new List<string>();
ennemis.Add("Gobelin");        // ajouter
ennemis.Add("Orc");
ennemis.Remove("Gobelin");     // retirer par valeur
Debug.Log(ennemis.Count);      // 1   (Count, PAS Length)
Debug.Log(ennemis[0]);         // "Orc"  (accès par index aussi)

foreach (string e in ennemis)  // parcourir
{
    Debug.Log(e);
}

Note : la List utilise .Count (pas .Length), Add/Remove/Clear/Contains. C'est la collection « par défaut » pour la plupart des cas en jeu.

3. Dictionary<K,V> : recherche par clé

Un Dictionary associe une clé unique à une valeur — comme un répertoire (nom → numéro). Idéal pour un inventaire (objet → quantité) :

Dictionary<string, int> inventaire = new Dictionary<string, int>();
inventaire["potion"] = 3;      // ajouter / mettre à jour par clé
inventaire["clé"] = 1;

if (inventaire.ContainsKey("potion"))
    Debug.Log("Potions : " + inventaire["potion"]);   // accès direct par clé

foreach (var paire in inventaire)
    Debug.Log(paire.Key + " x" + paire.Value);

L'intérêt : retrouver une valeur instantanément par sa clé, sans parcourir toute la collection.

4. Quelle collection choisir ?

  • Tableau : nombre d'éléments connu et fixe, accès par position (ex. les 4 coins d'un quad).
  • List<T> : cas général — ça grandit/rétrécit (ennemis vivants, projectiles). Le réflexe par défaut.
  • Dictionary<K,V> : retrouver vite par identifiant (inventaire, table de scores par nom).

🔧 Démonstration pas à pas

  1. Crée un script, ajoute using System.Collections.Generic;.
  2. Déclare une List<string> ennemis, ajoute 3 noms, retire-en un, affiche le Count puis parcours-la en foreach.
  3. Crée un Dictionary<string, int> inventaire, ajoute « potion »→3, « bombe »→2, affiche la quantité de potions.
  4. Vérifie avec ContainsKey avant d'accéder à une clé qui pourrait ne pas exister.

⚠️ Pièges & erreurs courantes

Symptôme« The type or namespace name 'List' could not be found ».
Causeusing System.Collections.Generic; manquant.
CorrectionAjoute cette ligne en haut du fichier.
SymptômeIndexOutOfRangeException / ArgumentOutOfRange.
CauseIndex hors limites (les index vont de 0 à Count-1 / Length-1).
CorrectionVérifie l'index ; parcours avec i < Count ou un foreach.
SymptômeKeyNotFoundException sur un Dictionary.
CauseAccès à une clé absente.
CorrectionTeste avec ContainsKey ou TryGetValue avant l'accès.
Symptôme« Collection was modified » pendant un foreach.
CauseTu fais Add/Remove en plein foreach.
CorrectionParcours avec un for à l'envers, ou collecte puis modifie après la boucle.

🔧Exercice pratique

  1. Gère une liste de projectiles : ajoute-en 5, retire le premier, affiche le nombre restant.
  2. Crée un inventaire en Dictionary : ajoute des objets, incrémente la quantité d'un objet existant (inventaire["potion"]++;).
  3. Calcule la somme d'un int[] scores avec un foreach.
Critères de réussite
  • Tu utilises .Count (List) vs .Length (tableau) correctement.
  • Aucune exception d'index ou de clé.
  • Tu sais justifier le choix List vs Dictionary vs tableau.

Exercices

Exercice 1Échauffement

Crée une List<string> d'inventaire, ajoute trois objets, puis affiche le nombre d'éléments.

Voir une solution 
using System.Collections.Generic;

List<string> inventaire = new List<string>();
inventaire.Add("Potion");
inventaire.Add("Épée");
inventaire.Add("Clé");
Debug.Log("Objets : " + inventaire.Count);   // 3
Exercice 2Application

Crée un Dictionary<string, int> « nom → score », ajoute deux joueurs et affiche le score de l'un par sa clé.

Voir une solution 
using System.Collections.Generic;

Dictionary<string, int> scores = new Dictionary<string, int>();
scores["Alice"] = 100;
scores["Bob"] = 75;
Debug.Log("Score d'Alice : " + scores["Alice"]);   // 100
Exercice 3Défi

Parcours la List d'inventaire avec foreach et compte combien d'objets contiennent la lettre « é ».

Voir une solution 
int avecAccent = 0;
foreach (string objet in inventaire)
{
    if (objet.Contains("é"))
        avecAccent++;
}
Debug.Log("Objets avec « é » : " + avecAccent);
📌À retenir
  • Tableau = taille fixe, accès par index (0 → Length-1).
  • List<T> = dynamique (Add/Remove/Count) — le choix par défaut ; nécessite using System.Collections.Generic;.
  • Dictionary<K,V> = accès rapide par clé (inventaire, lookups) ; teste avec ContainsKey.
  • On parcourt avec foreach ; ne modifie pas une collection pendant son foreach.

Glossaire des termes introduits

Collection — structure stockant plusieurs valeurs (tableau, List, Dictionary…).

Index — position d'un élément (commence à 0).

List<T> — collection dynamique générique (taille variable).

Dictionary<K,V> — collection associant des clés uniques à des valeurs.
