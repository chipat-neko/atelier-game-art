BP06 — Conditions (Branch, Switch)

Débutant 35 min Piste 12 · Blueprints
PrérequisBP4 — Variables & types.
🎯Objectifs d'apprentissage

À la fin de cette leçon, tu sauras :

  • Utiliser le nœud Branch (le « if » visuel).
  • Construire une condition avec les nœuds de comparaison.
  • Combiner des booléens avec AND, OR, NOT.
  • Gérer plusieurs cas avec Switch on Int / Enum.

Pourquoi c'est important

Une condition fait réagir ton jeu différemment selon la situation : ouvrir la porte si le joueur a la clé, infliger des dégâts si la cible est vivante. C'est le branchement de la logique.

Théorie en profondeur

1. Le nœud Branch

Branch prend une Condition (Boolean) et possède deux sorties d'exécution : True et False. C'est l'équivalent visuel du if/else.

Event ─exec▶ Branch (Condition ← bAClé)
                ├─True──▶ Ouvrir la porte
                └─False─▶ Afficher "Porte verrouillée"

2. Construire la condition (comparaisons)

La Condition vient souvent d'un nœud de comparaison : >, <, >=, <=, Equal (==), Not Equal (!=). Ces nœuds renvoient un Boolean.

Get PV ─data▶ [ <= ] (B = 0) ─data▶ Branch (Condition)
            ├─True──▶ Mourir

3. AND / OR / NOT

On combine plusieurs booléens : AND (toutes vraies), OR (au moins une), NOT (inverse). Pratique pour « a la clé ET porte non bloquée ».

bAClé ──┐
        [ AND Boolean ] ─data▶ Branch
NOT bBloquée ──┘

4. Switch on Int / Enum

Pour plusieurs cas, le Switch évite d'empiler les Branch. Switch on Int route selon un entier ; Switch on Enum selon une énumération (états nommés : Menu, Jeu, Pause).

Switch on Enum (EtatJeu)
   ├─ Menu   ▶ …
   ├─ Jeu    ▶ …
   └─ Pause  ▶ …

🖐Essaie maintenant~1 min

Pose un nœud Branch avec la condition PV <= 0. Où va le fil True vs False ? Ajoute un Print « mort » sur True. C'est le if version nœuds.

🔧 Démonstration pas à pas

  1. Compare PV <= 0 et branche vers « Mourir ».
  2. Combine deux conditions avec AND.
  3. Crée une Enum d'état et route-la avec Switch on Enum.

⚠️ Pièges & erreurs courantes

SymptômeLes deux branches s'exécutent.
CauseMauvais câblage des sorties True/False.
CorrectionUne seule sortie part vers chaque chemin distinct.
SymptômeCondition toujours fausse.
CauseComparaison inversée ou mauvaise variable.
CorrectionVérifie l'opérateur et les entrées du nœud.
SymptômeCascade de Branch illisible.
CausePlusieurs cas gérés en if imbriqués.
CorrectionUtilise Switch on Enum/Int.
SymptômeÉgalité de floats capricieuse.
CauseComparaison exacte de décimaux.
CorrectionUtilise Nearly Equal avec une tolérance.

🔧Exercice pratique

  1. Affiche « vie basse » si PV < 30.
  2. Ouvre une porte si bAClé AND NOT bBloquée.
  3. Route un état de jeu avec un Switch on Enum.
Critères de réussite
  • Ton Branch a une seule sortie par chemin.
  • Tu combines correctement AND/OR/NOT.
  • Tu remplaces les if imbriqués par un Switch quand c'est pertinent.

📌À retenir
  • Branch = if visuel : sorties True / False selon un Boolean.
  • Les comparaisons (==, >, <=…) produisent la Condition.
  • AND / OR / NOT combinent les booléens.
  • Switch on Enum/Int pour plusieurs cas, plus lisible que des Branch empilés.

Glossaire des termes introduits

Branch — nœud de condition (if) à deux sorties d'exécution.

Switch on Enum/Int — aiguillage multi-cas selon une valeur.

Enum — type énuméré listant des valeurs nommées (états).

Nearly Equal — comparaison de floats avec une tolérance.
