BP08 — Fonctions, macros & graphes

Intermédiaire 40 min Piste 12 · Blueprints
PrérequisBP5 — Événements.
🎯Objectifs d'apprentissage

À la fin de cette leçon, tu sauras :

  • Créer une fonction avec entrées/sorties.
  • Distinguer fonction pure et impure.
  • Savoir quand utiliser une macro ou un collapsed graph.
  • Rendre ton graphe lisible et réutilisable.

Pourquoi c'est important

Quand un graphe grossit, il devient illisible. Regrouper la logique en fonctions nommées la rend réutilisable, testable et claire — exactement comme en code.

Théorie en profondeur

1. Créer une fonction

Dans My Blueprint, « + » à côté de Functions. Une fonction a un nœud d'entrée (ses paramètres) et un nœud Return (ses sorties). On l'appelle ensuite comme n'importe quel nœud.

Function CalculerDegats (Base : Float, Mult : Float) → (Total : Float)
   Return Total = Base * Mult

Appel : CalculerDegats (Base=10, Mult=1.5) → 15

2. Pure vs impure

  • Impure (par défaut) : a un fil d'exécution, peut modifier l'état.
  • Pure (case « Pure ») : pas de fil exec, ne fait que calculer/retourner une valeur. Réévaluée à chaque fois qu'on tire sa sortie.

⚠️ Une fonction pure appelée à plusieurs endroits est recalculée à chaque usage : à éviter si le calcul est lourd.

3. Macros & collapsed graphs

  • Macro : comme une fonction mais « dépliée » à la compilation ; peut avoir plusieurs broches d'exécution (utile pour des structures réutilisables type Gate maison). Partageable via une Macro Library.
  • Collapsed graph : replie une portion de nœuds dans un sous-graphe pour ranger visuellement, sans en faire une fonction réutilisable.

4. Fonction vs macro vs collapse

Fonction        → logique réutilisable, 1 exec d'entrée, propre, debuggable
Macro           → réutilisable, plusieurs exec possibles, dépliée à la compil
Collapsed graph → simple rangement visuel (pas de réutilisation)

🔧 Démonstration pas à pas

  1. Crée CalculerDegats(Base, Mult) qui renvoie le produit.
  2. Crée une fonction pure EstVivant() renvoyant PV > 0.
  3. Replie une portion de graphe en collapsed graph.
  4. Appelle tes fonctions depuis l'Event Graph.

🔧Exercice pratique

  1. Crée une fonction Soigner(Montant) plafonnée à 100.
  2. Crée une fonction pure PourcentVie().
  3. Replie ta logique d'init en collapsed graph nommé « Setup ».
Critères de réussite
  • Tes fonctions ont des entrées/sorties claires.
  • Tu choisis pure vs impure à bon escient.
  • Ton Event Graph reste lisible.

Exercices

Exercice 1Prise en main

Décris une fonction Doubler(x) qui renvoie 2×x.

Voir une solution 
Function Doubler (x : Float) → (Result : Float)
   Return Result = x * 2
Exercice 2Application

Fais une fonction pure EstVivant() renvoyant un Boolean.

Voir une piste 
Function EstVivant [Pure] → (bVivant : Boolean)
   Return bVivant = (PV > 0)
Exercice 3Défi

Explique pourquoi un Delay ne peut pas vivre dans une fonction.

Voir une piste

Delay est un nœud « latent » (il s'étale sur plusieurs frames). Les fonctions s'exécutent en une fois, sans état temporel : les nœuds latents y sont interdits. On les place dans l'Event Graph ou une macro.

📌À retenir
  • Fonction : entrées/sorties, réutilisable, 1 exec d'entrée.
  • Pure (sans exec, calcul) vs impure (avec exec, effet) : pure recalculée à chaque usage.
  • Macro : plusieurs exec, dépliée à la compilation ; collapsed graph : rangement visuel.
  • Factoriser = lisibilité + réutilisation + débogage.

Glossaire des termes introduits

Fonction (Blueprint) — bloc de logique réutilisable avec entrées/sorties.

Pure / Impure — sans / avec fil d'exécution (calcul vs effet de bord).

Macro — bloc réutilisable déplié à la compilation, plusieurs exec possibles.

Collapsed graph — sous-graphe pour ranger visuellement des nœuds.
