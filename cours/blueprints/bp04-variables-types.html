BP04 — Variables & types

Débutant 35 min Piste 12 · Blueprints
PrérequisBP3 — Flux d'exécution.
🎯Objectifs d'apprentissage

À la fin de cette leçon, tu sauras :

  • Créer une variable et choisir son type (Boolean, Integer, Float, String, Vector, Rotator…).
  • Utiliser les nœuds Get et Set.
  • Donner une valeur par défaut.
  • Comprendre les références d'objets et les variables exposées.

Pourquoi c'est important

Les variables sont la mémoire de ton Blueprint : PV, vitesse, état ouvert/fermé, cible visée. Bien les typer évite des erreurs et rend le graphe lisible.

Théorie en profondeur

1. Créer une variable & choisir son type

Dans My Blueprint, clique « + » à côté de Variables, nomme-la, puis choisis son type dans Details. Les types courants :

Boolean   vrai / faux            (broche rouge)
Integer   nombre entier          (broche cyan/turquoise)
Float     nombre décimal         (broche verte)
String    texte                  (broche rose/magenta)
Vector    (X, Y, Z) position     (broche jaune-or)
Rotator   (Pitch, Yaw, Roll)     (broche violette)

La couleur de la broche encode le type : on ne peut relier que des broches compatibles.

2. Get & Set

Glisse une variable dans le graphe : Unreal propose Get (lire) ou Set (écrire). Set a un fil d'exécution (il modifie l'état) ; Get est une donnée pure (pas de fil exec).

Event ─exec▶ Set PV (nouvelle valeur = 100) ─exec▶ …
Get PV ─data▶ (utilisé là où une valeur est attendue)

3. Valeur par défaut

Compile une fois, puis dans Details règle la Default Value : c'est la valeur de départ de la variable quand l'Actor apparaît. (Le champ est grisé tant que le Blueprint n'est pas compilé.)

4. Références d'objets & exposition

Une variable peut contenir une référence vers un autre objet (un Actor, un Component). On peut aussi cocher Instance Editable (œil ouvert) pour la régler depuis le niveau, par instance (vu en détail en BP12).

🔧 Démonstration pas à pas

  1. Crée une variable PV de type Float, défaut 100.
  2. Au BeginPlay, fais Set PV = 80.
  3. Affiche Get PV avec un Print String.
  4. Crée un bEstOuvert (Boolean) et change-le.

⚠️ Pièges & erreurs courantes

SymptômeImpossible de relier deux broches.
CauseTypes incompatibles (couleurs différentes).
CorrectionAjoute un nœud de conversion (cast/convert) ou change le type.
SymptômeLa Default Value est grisée.
CauseBlueprint non compilé.
CorrectionCompile une fois, puis règle la valeur.
SymptômeMa valeur ne change pas.
CauseTu as utilisé Get au lieu de Set.
CorrectionPour écrire, utilise Set (avec fil exec).
SymptômeRéférence « None » au runtime.
CauseVariable de référence non assignée.
CorrectionAssigne-la (Set) ou expose-la et règle-la dans le niveau.

🔧Exercice pratique

  1. Crée VitesseMax (Float, défaut 600) et Nom (String).
  2. Au BeginPlay, écris puis affiche ces valeurs.
  3. Crée un Vector position cible et lis ses composantes.
Critères de réussite
  • Tu choisis le bon type selon la donnée.
  • Tu distingues Get (lire) et Set (écrire).
  • Tu règles correctement les valeurs par défaut.

Exercices

Exercice 1Prise en main

Associe chaque donnée à son type : « porte ouverte ? », « munitions », « position ».

Voir une solution

Porte ouverte ? → Boolean ; munitions → Integer ; position → Vector.

Exercice 2Application

Décris le graphe pour réduire PV de 25 au démarrage.

Voir une piste 
Get PV ─data▶ (PV - 25) ─data▶ Set PV
Event BeginPlay ─exec▶ Set PV
Exercice 3Défi

Crée une référence vers un autre Actor et explique comment l'assigner.

Voir une piste

Type = « Actor Reference » (ou une classe précise). On l'assigne soit par Set dans le graphe (ex. résultat d'un Get Actor / Spawn), soit en la rendant Instance Editable et en la pointant dans le niveau.

📌À retenir
  • Une variable a un type (et une couleur de broche) à choisir avec soin.
  • Get lit (donnée) ; Set écrit (avec fil exec).
  • Compile avant de régler la Default Value.
  • Une variable peut contenir une référence vers un autre objet.

Glossaire des termes introduits

Get / Set — nœuds pour lire / écrire une variable.

Default Value — valeur initiale d'une variable (réglée après compilation).

Référence (object reference) — variable pointant vers un autre objet.

Vector / Rotator — types pour position (X,Y,Z) et rotation (Pitch,Yaw,Roll).
