BP15 — Bonnes pratiques & mini-projet

Avancé 45 min Piste 12 · Blueprints
Prérequisidéalement les leçons BP5 à BP14.
🎯Objectifs d'apprentissage

À la fin de cette leçon, tu sauras :

  • Les conventions de nommage et d'organisation Blueprint.
  • Commenter et ranger un graphe (Comment boxes, reroute).
  • Déboguer avec Print String et les Breakpoints.
  • Assembler un mini-projet : un objet à ramasser qui ajoute au score.

Pourquoi c'est important

Un Blueprint clair se relit, se corrige et se partage. Les bonnes pratiques transforment un graphe « spaghetti » en logique maintenable — indispensable en équipe.

Théorie en profondeur

1. Conventions

  • Préfixes : BP_ (Blueprint), WBP_ (Widget), BPI_ (Interface), E_ (Enum), S_ (Struct).
  • Variables booléennes préfixées b (bIsOpen).
  • Noms explicites : VitesseMax plutôt que v.
  • Une responsabilité par Blueprint, factorisée en fonctions (rappel BP8).

2. Organiser le graphe

  • Comment boxes (touche C) : encadre et titre des groupes de nœuds.
  • Reroute nodes : redresse les fils pour éviter les croisements.
  • Aligne et espace (clic droit → alignement) ; lecture gauche→droite.

3. Déboguer

Print String affiche une valeur à l'écran/au log (pense à le retirer en production). Un Breakpoint (F9 sur un nœud) met le jeu en pause pour inspecter les valeurs pas à pas ; on survole les broches pour lire les données.

4. Mini-projet : un objet à ramasser

BP_Piece (Actor)
 ├─ StaticMesh "Piece"     (apparence)
 └─ SphereCollision "Zone" (détection)

Event OnComponentBeginOverlap (Zone)
   ─exec▶ Cast To BP_Joueur (OtherActor)
            réussi ─▶ As BP_Joueur : AjouterScore(10)
                   ─▶ Play Sound "ramassage"
                   ─▶ Destroy Actor (self)

BP_Joueur
   - variable Score (Integer)
   - fonction AjouterScore(n) : Set Score = Score + n
                                ─▶ (event) MettreAJourHUD

Ce mini-projet réunit : composants & collision (BP11), communication par Cast (BP10), fonction (BP8), variables et HUD (BP12), et Destroy Actor.

🖐Essaie maintenant~1 min

Ouvre un Blueprint : remplace un long fil qui traverse le graphe par une variable ou un nœud Reroute, et commente une zone (touche C). Deux gestes vers un graphe lisible.

🔧 Démonstration pas à pas

  1. Crée BP_Piece (mesh + sphère de collision).
  2. Sur l'overlap, caste vers BP_Joueur et appelle AjouterScore(10).
  3. Joue un son, détruis la pièce.
  4. Encadre la logique dans une Comment box et nomme proprement.

⚠️ Pièges & erreurs courantes

SymptômeGraphe illisible.
CausePas de commentaires ni d'organisation.
CorrectionComment boxes, reroute, fonctions nommées.
SymptômePrint String laissés en production.
CauseOubli de nettoyage.
CorrectionRetire-les ou conditionne-les (Development only).
SymptômeLa pièce ajoute le score à un mauvais acteur.
CauseCast non vérifié (overlap d'un autre Actor).
CorrectionGère Cast Failed avant d'agir.
SymptômeScore qui se mélange entre joueurs.
CauseVariable mise au mauvais endroit (global vs joueur).
CorrectionStocke le Score dans le Blueprint joueur concerné.

🔧Exercice pratique

  1. Finalise BP_Piece et fais-en 5 instances.
  2. Affiche le score dans un HUD mis à jour à chaque ramassage.
  3. Range et commente le graphe, retire les Print de debug.
Critères de réussite
  • Nommage cohérent (préfixes, b pour les booléens).
  • Graphe commenté et lisible.
  • Le mini-projet fonctionne de bout en bout.

Exercice 3Défi

Ajoute un objectif : gagner quand toutes les pièces sont ramassées.

Voir une piste

Un compteur de pièces (ou un Event Dispatcher « OnPickup » auquel un GameMode/Manager s'abonne). À chaque ramassage, décrémente le total restant ; quand il atteint 0, déclenche l'événement de victoire (afficher un widget, charger le niveau suivant).

Glossaire des termes introduits

Comment box — cadre titré regroupant des nœuds (touche C).

Reroute node — point de passage pour redresser un fil.

Print String — nœud de débogage affichant une valeur.

Breakpoint — point d'arrêt (F9) pour inspecter l'exécution pas à pas.

Voir aussi le glossaire global.
