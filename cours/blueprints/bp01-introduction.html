BP01 — Introduction aux Blueprints

Débutant 35 min Piste 12 · Blueprints
Prérequisaucun. Utile : avoir survolé la piste Unreal Engine.
🎯Objectifs d'apprentissage

À la fin de cette leçon, tu sauras :

  • Ce qu'est un Blueprint et le principe du scripting visuel par nœuds.
  • Pourquoi Unreal propose ce système et à qui il s'adresse.
  • Les différences (et complémentarité) entre Blueprint et C++.
  • Les principaux types de Blueprints.

Pourquoi c'est important

Les Blueprints permettent de programmer la logique d'un jeu Unreal sans écrire de code texte : on relie des nœuds. C'est la porte d'entrée du gameplay sur Unreal, très utilisée par les artistes, les designers et même les programmeurs pour le prototypage.

Théorie en profondeur

1. Un Blueprint, c'est quoi ?

Un Blueprint est un asset qui contient de la logique sous forme de graphe de nœuds. Chaque nœud fait une action (déplacer, comparer, afficher) ; on les relie par des fils pour décrire quoi faire, et dans quel ordre. C'est de la vraie programmation — seulement, la syntaxe est visuelle.

2. Pourquoi le scripting visuel ?

  • Accessible : pas de syntaxe à mémoriser, pas de compilation manuelle.
  • Visuel : le flux d'exécution se lit comme un schéma.
  • Itératif : on teste immédiatement dans l'éditeur.
  • Intégré : accès direct aux composants, assets et à l'API d'Unreal.

3. Blueprint vs C++

BLUEPRINT                          C++
- visuel, par nœuds                - texte, compilé
- rapide à prototyper              - plus performant à grande échelle
- idéal gameplay & UI             - idéal systèmes lourds / réutilisables
- pas de compilation manuelle      - compilation requise

→ En pratique : on combine les deux. Les studios écrivent
  les systèmes critiques en C++ et exposent des nœuds aux
  Blueprints pour le gameplay et le design.

Ce n'est pas « l'un ou l'autre » : Blueprint et C++ sont conçus pour cohabiter (voir la piste C++ Unreal).

4. Les types de Blueprints

  • Blueprint Class (Actor) : le plus courant — un objet placé dans le monde (porte, ennemi, item).
  • Level Blueprint : logique propre à un niveau précis.
  • Widget Blueprint (UMG) : interface utilisateur.
  • Blueprint Interface / Macro Library : outils de structuration (vus plus loin).

🔧 Démonstration pas à pas

  1. Dans Unreal, crée un Blueprint Class de type Actor.
  2. Ouvre l'Event Graph : repère Event BeginPlay.
  3. Ajoute un nœud Print String et relie le fil d'exécution.
  4. Place le Blueprint dans le niveau, lance : le texte s'affiche.

⚠️ Pièges & erreurs courantes

Symptôme« Les Blueprints, ce n'est pas du vrai code ».
CauseIdée reçue sur le visuel.
CorrectionC'est de la programmation : mêmes concepts (variables, conditions, boucles).
SymptômeHésitation Blueprint ou C++.
CauseOn les croit exclusifs.
CorrectionCombine-les : prototype en BP, durcis en C++ si besoin.
SymptômeLogique mise dans le mauvais Blueprint.
CauseConfusion Level BP / Blueprint Class.
CorrectionRéutilisable → Blueprint Class ; spécifique au niveau → Level BP.
SymptômeRien ne se passe au lancement.
CauseFil d'exécution non relié, ou Blueprint pas dans le niveau.
CorrectionVérifie le fil blanc et la présence dans la scène.

📌À retenir
  • Un Blueprint = de la logique visuelle par nœuds reliés.
  • C'est de la vraie programmation (variables, conditions, boucles).

Glossaire des termes introduits

Blueprint — asset de logique visuelle par nœuds dans Unreal.

Nœud (node) — bloc qui réalise une action ou fournit une donnée.

Blueprint Class (Actor) — Blueprint réutilisable représentant un objet du monde.

Level Blueprint — logique propre à un niveau donné.
