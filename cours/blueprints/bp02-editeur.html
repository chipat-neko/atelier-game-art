BP02 — L'éditeur Blueprint

Débutant 35 min Piste 12 · Blueprints
PrérequisBP1 — Introduction.
🎯Objectifs d'apprentissage

À la fin de cette leçon, tu sauras :

  • Repérer les zones clés : Event Graph, Components, Viewport, My Blueprint, Details.
  • Naviguer dans le graphe (pan, zoom) et placer des nœuds.
  • Compiler et sauvegarder un Blueprint.
  • Utiliser le menu contextuel et la recherche de nœuds.

Pourquoi c'est important

Avant de scripter, il faut savoir où les choses se passent. L'éditeur Blueprint a une disposition standard qu'on retrouve dans tous les projets : la connaître te fait gagner un temps fou.

Théorie en profondeur

1. Les panneaux essentiels

  • Event Graph : la zone centrale où l'on relie les nœuds (le cœur du scripting).
  • Components (haut-gauche) : la liste des composants de l'Actor (Mesh, Collision, Audio…).
  • Viewport : l'aperçu 3D de l'Actor et de ses composants.
  • My Blueprint : variables, fonctions, macros, graphes — l'inventaire du Blueprint.
  • Details : les propriétés de l'élément sélectionné (nœud, variable, composant).

2. Naviguer dans le graphe

Pan (déplacer la vue) : clic droit maintenu + glisser
Zoom                  : molette
Cadrer la sélection   : touche F (ou « Home » pour tout voir)
Placer un nœud        : clic droit dans le vide → recherche
Tirer un fil          : glisser depuis une broche puis lâcher dans le vide

3. La recherche contextuelle

Un clic droit (ou tirer un fil depuis une broche) ouvre une palette contextuelle : elle ne propose que les nœuds compatibles avec le contexte. Décocher « Context Sensitive » montre tout — utile quand on cherche un nœud rare.

4. Compiler & sauvegarder

Le bouton Compile (en haut à gauche) traduit le graphe et signale les erreurs. Un Blueprint non compilé affiche un point d'interrogation. Pense à Compile puis Save après chaque changement.

🔧 Démonstration pas à pas

  1. Ouvre un Blueprint et repère les cinq panneaux.
  2. Ajoute un composant Static Mesh depuis Components.
  3. Dans l'Event Graph, clic droit → cherche « Print String ».
  4. Compile puis Save.

⚠️ Pièges & erreurs courantes

Cause« Context Sensitive » filtre trop.
CorrectionDécoche-le, ou tire le fil depuis la bonne broche.
SymptômeMes changements n'ont aucun effet.
CauseBlueprint non compilé.
CorrectionClique Compile (puis Save).
SymptômeJe perds mes nœuds de vue.
CauseZoom/pan désorienté.

SymptômeVariable invisible dans Details.
CauseRien n'est sélectionné.
CorrectionSélectionne l'élément dans My Blueprint ou le graphe.

🔧Exercice pratique

  1. Ajoute un Static Mesh et donne-lui un cube dans Details.
  2. Place 3 nœuds et relie-les, puis cadre-les avec Home.
  3. Compile, corrige une éventuelle erreur signalée.
Critères de réussite
  • Tu nommes les cinq panneaux et leur rôle.
  • Tu sais placer un nœud via la recherche contextuelle.
  • Tu compiles et sauvegardes systématiquement.

Exercices

Exercice 1Prise en main

Associe chaque panneau à son rôle (Components, Viewport, Details, My Blueprint).

Voir une solution

Components = composants de l'Actor ; Viewport = aperçu 3D ; Details = propriétés de la sélection ; My Blueprint = variables/fonctions/graphes.

Exercice 2Application

Note la suite de gestes pour placer et relier deux nœuds.

Voir une piste 
Clic droit (vide) → chercher "Event BeginPlay"
Tirer le fil exec depuis BeginPlay → lâcher → chercher "Print String"
Compile → Save
Exercice 3Défi

Explique l'intérêt d'organiser tôt ses composants dans la hiérarchie.

Voir une piste

Une hiérarchie claire (un composant racine, des enfants attachés logiquement) rend l'Actor lisible, facilite l'attachement, les transformations relatives et le débogage quand le Blueprint grossit.

Glossaire des termes introduits

Event Graph — zone centrale de scripting où l'on relie les nœuds.

Components — éléments constitutifs d'un Actor (mesh, collision, son…).

My Blueprint — panneau listant variables, fonctions, macros et graphes.

Compile — traduit le Blueprint et signale les erreurs.
