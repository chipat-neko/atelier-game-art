BP07 — Boucles & Flow Control

Intermédiaire 40 min Piste 12 · Blueprints
PrérequisBP6 — Conditions.
🎯Objectifs d'apprentissage

À la fin de cette leçon, tu sauras :

  • Utiliser ForLoop, ForEachLoop et WhileLoop.
  • Lire les broches Loop Body et Completed.
  • Maîtriser le Flow Control : DoOnce, Gate, FlipFlop, Sequence.
  • Éviter les boucles lourdes dans Tick.

Pourquoi c'est important

Répéter une action — appliquer des dégâts à tous les ennemis, parcourir un inventaire — et contrôler le flux (faire une seule fois, alterner, ouvrir/fermer un passage) sont des besoins constants du gameplay.

Théorie en profondeur

1. ForLoop

Répète entre First Index et Last Index. À chaque tour, la sortie Loop Body s'exécute ; à la fin, Completed.

ForLoop (First=0, Last=4)
   ├ Loop Body ──▶ (action, Index disponible 0..4)
   └ Completed ──▶ (après les 5 tours)

2. ForEachLoop & WhileLoop

ForEachLoop parcourt un tableau (Array Element + Array Index). WhileLoop répète tant qu'une condition reste vraie (attention aux boucles infinies : la condition doit évoluer).

ForEachLoop (Array = Ennemis)
   ├ Loop Body ──▶ Appliquer dégâts à (Array Element)
   └ Completed ──▶ …

3. Flow Control

  • DoOnce : ne laisse passer qu'une fois (jusqu'à Reset).
  • Gate : portail Open/Close qui laisse ou non passer l'exécution.
  • FlipFlop : alterne A / B à chaque passage (parfait pour un interrupteur).
  • Sequence : exécute plusieurs sorties dans l'ordre (Then 0, Then 1…).
InputAction Interact ─▶ FlipFlop
   ├ A ─▶ Ouvrir la porte
   └ B ─▶ Fermer la porte

4. Boucles & Tick

Une boucle dans Tick tourne à chaque frame : une grosse boucle × 60/s peut écrouler le framerate. Préfère un événement ponctuel, ou étale le travail.

🔧 Démonstration pas à pas

  1. ForLoop 0→4 qui affiche l'Index.
  2. ForEachLoop sur un tableau d'ennemis.
  3. FlipFlop sur une touche pour ouvrir/fermer.
  4. DoOnce pour ne déclencher un effet qu'une seule fois.

⚠️ Pièges & erreurs courantes

SymptômeLe jeu se fige.
CauseWhileLoop dont la condition ne devient jamais fausse.
CorrectionAssure-toi que la condition évolue à chaque tour.
SymptômeL'action après la boucle part trop tôt.
CauseBranché sur Loop Body au lieu de Completed.
CorrectionLa suite va sur Completed.
SymptômeEffet déclenché en boucle.
CausePas de DoOnce/Gate pour limiter.
CorrectionEncadre avec DoOnce ou un Gate.
SymptômeFramerate qui s'effondre.
CauseGrosse boucle exécutée dans Tick.
CorrectionSors la boucle de Tick (événement ponctuel).

🔧Exercice pratique

  1. Avec ForLoop, fais apparaître 5 pièces alignées.
  2. Avec ForEachLoop, soigne tous les alliés d'un tableau.
  3. Utilise FlipFlop pour un interrupteur lumière on/off.
Critères de réussite
  • Tu branches la suite sur Completed.
  • Aucune boucle infinie.
  • Tu utilises le bon nœud de Flow Control selon le besoin.

📌À retenir
  • ForLoop / ForEachLoop / WhileLoop : Loop Body à chaque tour, Completed à la fin.
  • Flow Control : DoOnce (une fois), Gate (ouvre/ferme), FlipFlop (alterne), Sequence (ordre).
  • WhileLoop : la condition doit évoluer (sinon boucle infinie).
  • Évite les grosses boucles dans Tick.

Glossaire des termes introduits

ForLoop / ForEachLoop / WhileLoop — nœuds de répétition de Blueprint.

Loop Body / Completed — sorties d'une boucle (chaque tour / fin).

Flow Control — nœuds qui contrôlent le flux (DoOnce, Gate, FlipFlop, Sequence).

Gate — portail d'exécution qu'on ouvre/ferme.
