CPP14 — Exposer au Blueprint & timers

Avancé 45 min Piste 8 · C++ (Unreal)
PrérequisCPP8 — Réflexion et CPP13.
🎯Objectifs d'apprentissage

À la fin de cette leçon, tu sauras :

  • Exposer une fonction C++ au Blueprint (BlueprintCallable, BlueprintPure).
  • Laisser un Blueprint implémenter une logique (BlueprintImplementableEvent/NativeEvent).
  • Différer ou répéter une action avec FTimerManager.
  • Organiser le partage C++ ↔ Blueprint (« C++ pour la base, BP pour l'assemblage »).

Pourquoi c'est important

La force d'Unreal est le pont C++ ↔ Blueprint (vu en CPP1) : les programmeurs écrivent des systèmes robustes en C++, et les designers les assemblent visuellement. Cette leçon montre comment ouvrir ton code au Blueprint. Et comme une boucle ne peut pas « attendre » (CPP5), les timers sont l'outil pour différer/répéter dans le temps — l'équivalent des coroutines de Unity (CS14).

Théorie en profondeur

1. Appeler du C++ depuis un Blueprint

// Appelable comme un nœud d'action dans le Blueprint
UFUNCTION(BlueprintCallable, Category = "Combat")
void Tirer();

// "Pure" : pas d'effet de bord, juste un calcul (nœud sans broche d'exécution)
UFUNCTION(BlueprintPure, Category = "Stats")
int32 GetMunitions() const;

2. Laisser le Blueprint implémenter

Inversement, le C++ peut déclencher une logique définie côté Blueprint : idéal pour le feedback (FX, sons) que les designers veulent régler.

// Implémentée UNIQUEMENT en Blueprint : pas de corps C++
UFUNCTION(BlueprintImplementableEvent, Category = "FX")
void OnTir();

// Implémentée en C++ MAIS surchargeable en Blueprint
UFUNCTION(BlueprintNativeEvent, Category = "FX")
void OnImpact();
void OnImpact_Implementation();   // la version C++ par défaut

Depuis le C++, on les appelle normalement (OnTir();) : si un Blueprint enfant les implémente, c'est sa version qui s'exécute.

SpécificateurSens
BlueprintCallableLe BP peut appeler cette fonction C++.
BlueprintPureIdem, mais fonction de calcul sans effet (pas de broche exec).
BlueprintImplementableEventLe BP implémente tout ; le C++ ne fait que l'appeler.
BlueprintNativeEventLe C++ fournit une version par défaut, surchargeable en BP.

3. Les timers : différer & répéter

FTimerManager (accessible via GetWorldTimerManager()) appelle une fonction après un délai, une fois ou en boucle :

FTimerHandle Handle;   // pour pouvoir arrêter le timer plus tard

// Appeler Recharger() une fois, dans 2 secondes
GetWorldTimerManager().SetTimer(Handle, this, &AArme::Recharger, 2.0f, false);

// Appeler Tic() toutes les 1 s, en boucle, après un délai initial de 1 s
GetWorldTimerManager().SetTimer(Handle, this, &ASpawner::Tic, 1.0f, true, 1.0f);

// Arrêter
GetWorldTimerManager().ClearTimer(Handle);

Signature : SetTimer(Handle, objet, &Classe::Fonction, Delai, bBoucle, DelaiInitial). Garde le FTimerHandle pour pouvoir l'annuler (sinon, impossible d'arrêter une boucle).

🔧 Démonstration pas à pas

  1. Marque Tirer() en BlueprintCallable et appelle-la depuis un Blueprint.
  2. Ajoute OnTir() en BlueprintImplementableEvent ; dans le BP, joue un son/FX ; appelle OnTir() depuis Tirer() en C++.
  3. Implémente une recharge différée avec SetTimer (2 s, non bouclé).
  4. Crée un spawner qui appelle Tic() toutes les 2 s via un timer bouclé ; arrête-le avec ClearTimer.

⚠️ Pièges & erreurs courantes

SymptômeMa fonction n'apparaît pas dans le Blueprint.
CausePas de UFUNCTION(BlueprintCallable) (ou Category manquante).
CorrectionAjoute le spécificateur ; recompile.
SymptômeLe timer ne se déclenche jamais.
CauseObjet détruit avant l'échéance, ou monde indisponible.
CorrectionLance le timer après BeginPlay sur un Actor vivant.
SymptômeImpossible d'arrêter un timer en boucle.
CauseTu n'as pas conservé le FTimerHandle.
CorrectionStocke le handle et passe-le à ClearTimer.
Symptôme« unresolved external » sur un NativeEvent.
CauseTu as défini OnImpact() au lieu de OnImpact_Implementation().
CorrectionImplémente la version _Implementation.

🔧Exercice pratique

  1. Une arme : Tirer() (BlueprintCallable) déclenche OnTir() (BP) et lance une recharge par timer.
  2. Un spawner qui fait apparaître un ennemi toutes les 3 s via un timer bouclé.
  3. Un bouton « pause spawn » qui appelle ClearTimer.
Critères de réussite
  • Le Blueprint appelle le C++ et implémente le feedback.
  • Les timers se déclenchent et s'arrêtent correctement.
  • Aucune boucle d'attente bloquante.
📌À retenir
  • BlueprintCallable/Pure = le BP appelle le C++ ; BlueprintImplementableEvent/NativeEvent = le BP implémente.
  • Les timers (FTimerManager::SetTimer) diffèrent ou répètent une action sans bloquer.
  • Conserve le FTimerHandle pour pouvoir ClearTimer.
  • Stratégie : systèmes en C++, assemblage et feedback en Blueprint.

Glossaire des termes introduits

BlueprintCallable — rend une fonction C++ appelable depuis un Blueprint.

BlueprintImplementableEvent — fonction implémentée uniquement côté Blueprint.

BlueprintNativeEvent — version C++ par défaut, surchargeable en Blueprint.

FTimerManager — gestionnaire de timers (différer/répéter une fonction).
