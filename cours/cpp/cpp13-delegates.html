CPP13 — Delegates & événements

Avancé 45 min Piste 8 · C++ (Unreal)
PrérequisCPP8 et CPP10.
🎯Objectifs d'apprentissage

À la fin de cette leçon, tu sauras :

  • Comprendre le couplage et pourquoi le réduire.
  • Déclarer un delegate dynamic multicast et le déclencher.
  • S'abonner avec AddDynamic et exposer au Blueprint (BlueprintAssignable).
  • Savoir quand préférer une interface.

Pourquoi c'est important

Quand le joueur meurt, l'UI, le son, le score et le spawner doivent réagir. Si la classe Sante appelle directement chacun, elle devient un nœud couplé à tout le projet. Les delegates (le patron observateur, comme les events C# de CS13) inversent la dépendance : Sante annonce, et qui veut réagir s'abonne.

Théorie en profondeur

1. Les familles de delegates

TypeAbonnésBlueprint ?
Singleun seulnon
Multicastplusieursnon
Dynamic Multicastplusieursoui (sérialisable, BlueprintAssignable)

Pour le gameplay exposé au Blueprint, on utilise presque toujours le dynamic multicast.

2. Déclarer & déclencher un événement

// Sante.h — déclare le TYPE de delegate (hors de la classe), avec 1 paramètre
DECLARE_DYNAMIC_MULTICAST_DELEGATE_OneParam(FOnVieChangee, int32, NouveauPV);

UCLASS()
class MONJEU_API USante : public UActorComponent
{
    GENERATED_BODY()
public:
    // L'ÉVÉNEMENT, exposé au Blueprint
    UPROPERTY(BlueprintAssignable, Category = "Sante")
    FOnVieChangee OnVieChangee;

    void Subir(int32 Degats);

private:
    UPROPERTY(EditAnywhere) int32 PV = 100;
};
// Sante.cpp
void USante::Subir(int32 Degats)
{
    PV = FMath::Max(0, PV - Degats);
    OnVieChangee.Broadcast(PV);   // notifie TOUS les abonnés
}

Les macros existent en variantes selon le nombre de paramètres : ..._OneParam, ..._TwoParams, etc.

3. S'abonner en C++

La fonction abonnée doit être une UFUNCTION() (exigence du dynamic delegate) :

// BarreVieUI.h
UFUNCTION()
void MajBarre(int32 NouveauPV);   // doit être une UFUNCTION

// BarreVieUI.cpp
void UBarreVieUI::Init(USante* Sante)
{
    if (Sante)
    {
        Sante->OnVieChangee.AddDynamic(this, &UBarreVieUI::MajBarre);   // s'abonner
    }
}

// se désabonner quand on n'écoute plus :
// Sante->OnVieChangee.RemoveDynamic(this, &UBarreVieUI::MajBarre);

Côté Blueprint, comme l'événement est BlueprintAssignable, on peut s'y abonner visuellement avec un nœud « Bind Event to On Vie Changee » — sans aucun code côté abonné.

4. Et les interfaces ?

Un delegate sert à notifier. Une interface sert à garantir qu'une classe sait faire quelque chose (ex. : tout ce qui est « endommageable »). Unreal les déclare en deux temps :

UINTERFACE(MinimalAPI)
class UDamageable : public UInterface { GENERATED_BODY() };

class IDamageable
{
    GENERATED_BODY()
public:
    virtual void RecevoirDegats(int32 Montant) = 0;   // à implémenter
};

// Usage polymorphe :
if (IDamageable* Cible = Cast<IDamageable>(Acteur))
{
    Cible->RecevoirDegats(10);
}

🔧 Démonstration pas à pas

  1. Crée un USante avec FOnVieChangee (BlueprintAssignable) et déclenche-le dans Subir.
  2. Une UI s'abonne en C++ via AddDynamic (fonction UFUNCTION).
  3. Dans un Blueprint, abonne-toi visuellement au même événement et joue un effet — sans toucher au C++.
  4. Ajoute un événement OnMort (sans paramètre) déclenché à 0 PV.

⚠️ Pièges & erreurs courantes

Symptôme« AddDynamic : la fonction n'est pas reconnue ».
CauseLa fonction abonnée n'est pas une UFUNCTION().
CorrectionMarque-la UFUNCTION().
SymptômeCrash quand l'événement se déclenche.
CauseUn abonné a été détruit sans se désabonner.
CorrectionAppelle RemoveDynamic dans EndPlay.
SymptômeL'événement n'apparaît pas en Blueprint.
CauseDelegate non dynamic ou pas BlueprintAssignable.
CorrectionUtilise DECLARE_DYNAMIC_MULTICAST_DELEGATE… + BlueprintAssignable.
SymptômeL'émetteur référence ses abonnés.
CauseOn a recréé du couplage au lieu d'utiliser le delegate.
CorrectionL'émetteur ne fait que Broadcast ; il ne connaît personne.

🔧Exercice pratique

  1. Un composant USante diffusant OnVieChangee(int32) et OnMort().
  2. Une UI C++ et un Blueprint qui s'abonnent tous deux à OnVieChangee.
  3. Une interface IDamageable implémentée par le joueur et les ennemis.
Critères de réussite
  • L'émetteur ne référence aucun abonné.
  • Les abonnés C++ utilisent des UFUNCTION et se désabonnent.
  • Un Blueprint peut réagir à l'événement sans modifier le C++.

📌À retenir
  • Les delegates réduisent le couplage : l'émetteur Broadcast, les abonnés réagissent.
  • Pour le gameplay exposé : DECLARE_DYNAMIC_MULTICAST_DELEGATE… + BlueprintAssignable.
  • S'abonner en C++ : AddDynamic sur une UFUNCTION ; se désabonner dans EndPlay.
  • Delegate = notifier ; interface = garantir qu'une classe sait faire.

Glossaire des termes introduits

Delegate — objet référençant des fonctions à appeler ; multicast = plusieurs abonnés.

Dynamic multicast — delegate sérialisable, exposable au Blueprint (BlueprintAssignable).

Broadcast / AddDynamic — déclencher l'événement / s'y abonner.

Interface (UINTERFACE) — contrat de fonctions qu'une classe s'engage à implémenter.
