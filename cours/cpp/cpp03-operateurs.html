CPP03 — Opérateurs & expressions

Débutant 30 min Piste 8 · C++ (Unreal)
PrérequisCPP2 — Variables & types.
🎯Objectifs d'apprentissage

À la fin de cette leçon, tu sauras :

  • Utiliser les opérateurs arithmétiques, de comparaison et logiques.
  • Employer l'affectation composée et l'incrément (+=, ++).
  • Comprendre la priorité des opérateurs et l'usage des parenthèses.
  • Convertir explicitement entre types (cast).

Pourquoi c'est important

Tout calcul de gameplay — dégâts, vitesse, score, distances, conditions de victoire — repose sur des expressions combinant des opérateurs. Bien les maîtriser évite les bugs subtils (division entière, comparaisons de flottants, priorités).

Théorie en profondeur

1. Arithmétique

int32 a = 7, b = 2;
int32 somme   = a + b;   // 9
int32 produit = a * b;   // 14
int32 quotient = a / b;  // 3  (division ENTIÈRE : la partie décimale est perdue)
int32 reste   = a % b;   // 1  (modulo : reste de la division)

float precis = 7.0f / 2.0f;  // 3.5  (au moins un flottant => division flottante)

2. Comparaison & logique

bool egal      = (a == b);   // ==  égal à
bool different = (a != b);   // !=  différent de
bool plusGrand = (a > b);    // >  >=  <  <=

bool peutTirer = aDesMunitions && !enRechargement;   // ET logique, NON
bool estBloque = murDevant || ennemiDevant;          // OU logique

Les opérateurs logiques sont court-circuités : dans A && B, si A est faux, B n'est même pas évalué. Pratique pour protéger un test (ex. : ptr != nullptr && ptr->EstActif()).

3. Affectation composée & incrément

score += 10;     // équivaut à score = score + 10
vie   -= degats; // soustraction
munitions--;     // décrémente de 1
combo++;         // incrémente de 1

4. Priorité des opérateurs

Comme en maths : * et / avant + et -. En cas de doute, mets des parenthèses — c'est plus lisible et sans ambiguïté :

int32 r1 = 2 + 3 * 4;        // 14  (multiplication d'abord)
int32 r2 = (2 + 3) * 4;      // 20  (parenthèses d'abord)
float deg = base * (1.0f + bonus) - reduction;

5. Conversions (cast)

Le C++ ne convertit pas tout automatiquement ; pour passer d'un type à un autre, on caste explicitement :

int32 pv = 73, pvMax = 100;
float ratio = (float)pv / (float)pvMax;          // 0.73  (cast style C)
float ratio2 = static_cast<float>(pv) / pvMax;    // forme C++ recommandée

int32 arrondi = FMath::RoundToInt(ratio * 100);  // utilitaires Unreal : FMath

Sans le cast, pv / pvMax ferait une division entière (73/100 = 0). C'est le piège n°1 du débutant.

🔧 Démonstration pas à pas

  1. Calcule des dégâts : float total = base * (1.0f + bonusCritique);.
  2. Teste une condition composée : peut attaquer si (munitions > 0) && !enPause.
  3. Calcule un pourcentage de vie correct avec un cast, puis arrondis-le avec FMath::RoundToInt.
  4. Utilise % pour détecter un tour pair : (tour % 2 == 0).

🔧Exercice pratique

  1. Écris une formule de dégâts avec bonus critique et réduction d'armure (en flottants).
  2. Calcule et affiche le pourcentage de vie restant (cast + arrondi).
  3. Détermine si le joueur peut tirer via une condition logique court-circuitée.
Critères de réussite
  • Aucune division entière non voulue.
  • Les conditions composées sont lisibles et correctes.
  • Tu compares les flottants avec IsNearlyEqual si besoin.

Vérifie ta compréhension

Clique sur la réponse que tu penses correcte : la bonne réponse et une explication s'affichent aussitôt.

Exercices

Exercice 1Échauffement

Calcule des dégâts en flottant : une base de 10 multipliée par (1 + bonus critique de 0,5).

Voir une solution 
float Base = 10.0f;
float Bonus = 0.5f;
float Total = Base * (1.0f + Bonus);   // 15
UE_LOG(LogTemp, Log, TEXT("%f"), Total);
Exercice 2Application

Écris une condition « peut tirer » combinant deux booléens avec && et !.

Voir une solution 
bool bAMunitions = true;
bool bEnRechargement = false;
bool bPeutTirer = bAMunitions && !bEnRechargement;   // true
Exercice 3Défi

Calcule un pourcentage de vie correct (cast + arrondi) et compare deux flottants en toute sécurité.

Voir une solution 
int32 PV = 73, PVMax = 100;
int32 Pourcent = FMath::RoundToInt((float)PV / PVMax * 100);   // 73

float A = 0.1f + 0.2f;
bool bEgaux = FMath::IsNearlyEqual(A, 0.3f);   // true (malgré l'imprécision)
📌À retenir
  • / entre entiers = division entière ; caste en float pour un ratio.
  • && et || sont court-circuités : utile pour protéger un test.
  • En cas de doute sur la priorité, parenthèse.
  • Compare les flottants avec FMath::IsNearlyEqual, jamais ==.

Glossaire des termes introduits

Modulo (%) — reste d'une division entière (utile pour la parité, les cycles).

Court-circuit — &&/|| n'évaluent pas le 2ᵉ opérande si inutile.

static_cast — conversion de type explicite et sûre, forme C++ recommandée.

FMath — bibliothèque mathématique d'Unreal (RoundToInt, IsNearlyEqual, Clamp…).
