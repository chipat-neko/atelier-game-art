CPP04 — Conditions (if, switch)

Débutant 35 min Piste 8 · C++ (Unreal)
PrérequisCPP3 — Opérateurs.
🎯Objectifs d'apprentissage

À la fin de cette leçon, tu sauras :

  • Brancher le code avec if / else if / else.
  • Utiliser l'opérateur ternaire pour une décision courte.
  • Choisir entre plusieurs cas avec switch.
  • Éviter l'erreur classique = au lieu de ==.

Pourquoi c'est important

Un jeu prend des décisions en permanence : « si la vie atteint 0, mourir », « selon l'état de l'IA, patrouiller ou attaquer ». Les conditions sont le mécanisme de base de toute logique de gameplay.

Théorie en profondeur

1. if / else if / else

if (pv <= 0)
{
    Mourir();
}
else if (pv < 30)
{
    AfficherAlerteVie();   // entre 1 et 29
}
else
{
    // tout va bien
}

Les accolades { } ne sont pas obligatoires pour une seule instruction, mais mets-les toujours : elles évitent des bugs lors de l'ajout de lignes.

2. L'opérateur ternaire

Pour choisir entre deux valeurs, une forme compacte :

FString etat = (pv > 0) ? TEXT("Vivant") : TEXT("Mort");
float vitesse = bCourt ? 600.0f : 300.0f;

À réserver aux cas simples ; pour une logique riche, un if reste plus lisible.

3. switch : choisir parmi des cas

Quand une variable peut prendre des valeurs discrètes (souvent un enum, vu en CPP8), switch est plus clair qu'une cascade de if :

switch (etatIA)
{
    case EEtatIA::Patrouille:
        Patrouiller();
        break;            // NE PAS OUBLIER : sinon on « tombe » dans le cas suivant

    case EEtatIA::Poursuite:
        Poursuivre();
        break;

    case EEtatIA::Attaque:
        Attaquer();
        break;

    default:              // tous les autres cas
        Attendre();
        break;
}

Le break est indispensable : sans lui, l'exécution continue dans le cas suivant (« fall-through »), source de bugs.

🖐Essaie maintenant~1 min

Prédis, pour int32 pv = 10 : if (pv <= 0) {…} else if (pv < 30) {…} else {…} — quelle branche pour pv = 10, 0, puis 50 ?

🔧 Démonstration pas à pas

  1. Écris une fonction EvaluerVie(int32 pv) qui logge « Mort », « Critique » ou « OK » selon des seuils.
  2. Choisis une vitesse avec un ternaire selon un booléen bCourt.
  3. Crée un enum class d'états et un switch qui appelle une action par état (avec break et default).

⚠️ Pièges & erreurs courantes

SymptômeLa condition est toujours vraie.
Causeif (pv = 0) — affectation au lieu de comparaison.
CorrectionUtilise == pour comparer. (Astuce : certains écrivent 0 == pv.)
SymptômePlusieurs cas d'un switch s'exécutent.
Causebreak oublié (fall-through).
CorrectionTermine chaque case par break.
SymptômeUne ligne ajoutée après un if sans accolades ne marche pas.
CauseSeule la 1ʳᵉ ligne dépend du if (pas d'accolades).
CorrectionMets systématiquement des accolades { }.
SymptômeCascade de if illisible.
CauseBeaucoup de cas discrets traités en if/else.
CorrectionUtilise un switch (souvent sur un enum).

🔧Exercice pratique

  1. Un système de « rang » selon le score (Bronze / Argent / Or) via if/else if.
  2. Un enum d'états d'IA et un switch qui déclenche l'action correspondante.
  3. Un ternaire qui choisit un message court selon que le joueur est vivant.
Critères de réussite
  • Aucune confusion =/==.
  • Chaque case a son break et il y a un default.
  • Les blocs sont toujours entre accolades.

Écris une fonction qui logge « Mort », « Critique » ou « OK » selon les PV, avec if / else if / else.

void EvaluerVie(int32 PV)
{
    if (PV <= 0)
        UE_LOG(LogTemp, Log, TEXT("Mort"));
    else if (PV < 30)
        UE_LOG(LogTemp, Log, TEXT("Critique"));
    else
        UE_LOG(LogTemp, Log, TEXT("OK"));
}
Exercice 2Application

Déclare un enum class EEtatIA : uint8 et écris un switch qui appelle une action par état (avec break et default).

Voir une solution 
enum class EEtatIA : uint8 { Patrouille, Poursuite, Attaque };

void Agir(EEtatIA Etat)
{
    switch (Etat)
    {
        case EEtatIA::Patrouille: Patrouiller(); break;
        case EEtatIA::Poursuite:  Poursuivre(); break;
        case EEtatIA::Attaque:    Attaquer();   break;
        default:                  Attendre();   break;
    }
}

📌À retenir
  • if/else if/else branche selon des conditions booléennes ; toujours des accolades.
  • Le ternaire cond ? a : b pour choisir une valeur courte.
  • switch pour des cas discrets ; break obligatoire, default recommandé.
  • Comparer = == ; = est une affectation (piège classique).

Glossaire des termes introduits

Ternaire — opérateur condition ? valeurSiVrai : valeurSiFaux.

switch / case — branchement multi-cas sur une valeur discrète.

Fall-through — passage involontaire d'un case au suivant faute de break.
