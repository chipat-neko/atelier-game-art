CPP02 — Variables, types & mémoire

Débutant 40 min Piste 8 · C++ (Unreal)
PrérequisCPP1 — Introduction.
🎯Objectifs d'apprentissage

À la fin de cette leçon, tu sauras :

  • Déclarer et initialiser une variable en C++.
  • Choisir le bon type Unreal (int32, float, bool, FString).
  • Utiliser const et auto à bon escient.
  • Comprendre la différence entre pile (stack) et tas (heap).

Pourquoi c'est important

Le C++ est typé statiquement et strict : chaque variable a un type fixé à la compilation, et le compilateur refuse les approximations. C'est plus exigeant qu'en C# mais cela donne contrôle et performance. Unreal ajoute par-dessus ses propres types (comme int32 ou FString) pour garantir un comportement identique sur toutes les plateformes.

Théorie en profondeur

1. Déclarer une variable

int32 score = 0;           // un entier, initialisé à 0
float vitesse = 600.0f;    // un nombre à virgule (le 'f' = littéral float)
bool bEnVie = true;        // un booléen (préfixe 'b' par convention Unreal)
FString nom = TEXT("Héros"); // une chaîne de caractères Unreal

Toujours initialiser : une variable non initialisée contient une valeur « poubelle » imprévisible (source de bugs classiques en C++).

2. Les types Unreal vs C++ standard

Unreal recommande ses propres alias pour une taille garantie sur toutes les plateformes :

Préfère int32 à int dans le code Unreal. Note : en UE5, les vecteurs (FVector) utilisent des double pour les grands mondes.

3. Les chaînes : FString, FName, FText

Unreal a trois types de texte, chacun pour un usage :

  • FString — texte modifiable, pour la manipulation (concaténer, découper).
  • FName — identifiant immuable et rapide (noms d'os, de tags), insensible à la casse.
  • FText — texte destiné à l'affichage et à la localisation (traduction).

Toute chaîne littérale Unreal passe par la macro TEXT("...") pour gérer l'Unicode :

FString message = TEXT("Score : ") + FString::FromInt(score);
UE_LOG(LogTemp, Log, TEXT("%s"), *message);   // le * convertit FString -> const TCHAR*

4. const et auto

const int32 VieMax = 100;     // const : ne changera jamais (le compilateur l'impose)
auto degats = 25;             // auto : le compilateur déduit le type (ici int32)
const float Pi = 3.14159f;

const exprime une intention (« cette valeur est fixe ») et évite des bugs. auto allège l'écriture quand le type est évident, mais n'en abuse pas : la lisibilité prime.

5. Pile (stack) vs tas (heap)

Notion clé du C++. Une variable locale ordinaire vit sur la pile : créée à l'entrée d'un bloc, détruite automatiquement à la sortie — rapide et sans souci. Les objets qui doivent survivre au-delà du bloc sont alloués sur le tas (avec new en C++ pur, ou via NewObject/SpawnActor dans Unreal), et leur durée de vie doit être gérée (on verra le ramasse-miettes d'Unreal en CPP10).

🔧 Démonstration pas à pas

  1. Dans BeginPlay(), déclare int32 score = 0; et FString nom = TEXT("Joueur");.
  2. Construis un message avec FString::FromInt et affiche-le via UE_LOG.
  3. Ajoute une constante const int32 VieMax = 100; et tente de la modifier : observe l'erreur du compilateur.
  4. Remplace une déclaration par auto et vérifie que ça compile toujours.

🔧Exercice pratique

  1. Déclare des variables pour un personnage : nom (FString), pointsDeVie (int32), vitesse (float), bEstVivant (bool).
  2. Affiche une fiche récapitulative dans l'Output Log avec un seul UE_LOG formaté.
  3. Ajoute une constante VieMax et calcule le pourcentage de vie (attention à la division entière !).
Critères de réussite
  • Toutes les variables sont initialisées et bien typées.
  • Les chaînes utilisent TEXT() et l'affichage est correct.
  • Le pourcentage est calculé en flottant, pas en entier.

📌À retenir
  • C++ est strictement typé : initialise toujours tes variables.
  • Préfère les types Unreal : int32, float, bool (préfixe b), FString.
  • FString (modifiable), FName (identifiant rapide), FText (affichage/localisation) ; TEXT("...") partout.
  • Pile = locales auto-nettoyées ; tas = objets durables (gérés par le GC, CPP10).

Glossaire des termes introduits

int32 / uint8 — entiers Unreal de taille garantie (signé 32 bits / non signé 8 bits).

FString / FName / FText — texte modifiable / identifiant rapide / texte d'affichage localisable.

TEXT() — macro entourant tout littéral texte Unreal (gestion Unicode).

Pile / Tas — mémoire auto-nettoyée (locales) vs mémoire à durée de vie gérée (objets).
