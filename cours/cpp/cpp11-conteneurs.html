CPP11 — Conteneurs Unreal (TArray, TMap…)

Intermédiaire 40 min Piste 8 · C++ (Unreal)
PrérequisCPP5 — Boucles et CPP7.
🎯Objectifs d'apprentissage

À la fin de cette leçon, tu sauras :

  • Utiliser TArray (liste dynamique) : ajouter, parcourir, retirer.
  • Utiliser TMap (clé → valeur) et TSet (ensemble unique).
  • Manipuler les chaînes FString (et savoir quand FName/FText).
  • Choisir le bon conteneur selon le besoin.

Pourquoi c'est important

Un jeu manipule des collections partout : la liste des ennemis, l'inventaire, la table des scores par joueur, l'ensemble des niveaux déjà visités. Unreal fournit ses propres conteneurs, optimisés et intégrés au moteur — l'équivalent des List/Dictionary de C# (voir la leçon C# CS10).

Théorie en profondeur

1. TArray : la liste dynamique

Le conteneur le plus utilisé : une suite ordonnée, indexée à partir de 0, de taille variable. C'est l'équivalent Unreal du tableau dynamique de la bibliothèque standard C++ (la STL), le std::vector — mais TArray est intégré au moteur (réflexion, ramasse-miettes, sérialisation), c'est pourquoi on l'utilise partout dans Unreal à la place de std::vector.

UPROPERTY()
TArray<AActor*> Ennemis;     // déclaration

Ennemis.Add(NouvelEnnemi);   // ajouter à la fin
int32 n = Ennemis.Num();     // nombre d'éléments
AActor* Premier = Ennemis[0]; // accès par index (attention aux bornes !)

if (Ennemis.Contains(Cible)) { /* ... */ }   // recherche
Ennemis.Remove(MortEnnemi);                  // retirer par valeur
Ennemis.RemoveAt(2);                         // retirer par index

for (AActor* E : Ennemis)    // parcours (range-based for, CPP5)
{
    if (IsValid(E)) { /* ... */ }
}

2. TMap : associer une clé à une valeur

UPROPERTY()
TMap<FName, int32> ScoresParJoueur;

ScoresParJoueur.Add(TEXT("Alice"), 100);
ScoresParJoueur.Add(TEXT("Bob"), 75);

if (int32* Score = ScoresParJoueur.Find(TEXT("Alice")))  // Find renvoie un pointeur
{
    UE_LOG(LogTemp, Log, TEXT("Score : %d"), *Score);     // null si absent
}

for (const TPair<FName, int32>& Paire : ScoresParJoueur)  // parcours clé/valeur
{
    UE_LOG(LogTemp, Log, TEXT("%s = %d"), *Paire.Key.ToString(), Paire.Value);
}

3. TSet : un ensemble de valeurs uniques

Quand on veut éviter les doublons et tester l'appartenance rapidement :

TSet<FName> NiveauxVisites;
NiveauxVisites.Add(TEXT("Donjon01"));
NiveauxVisites.Add(TEXT("Donjon01"));   // ignoré : déjà présent
bool bDejaVu = NiveauxVisites.Contains(TEXT("Donjon01"));  // true

4. Manipuler des FString

FString A = TEXT("Score : ");
FString B = A + FString::FromInt(42);          // concaténation
FString C = FString::Printf(TEXT("PV %d/%d"), 73, 100);  // formatage
bool bVide = C.IsEmpty();
int32 Longueur = C.Len();

Rappel (CPP2) : FString pour manipuler, FName comme clé rapide (insensible à la casse), FText pour l'affichage localisé.

5. Choisir le bon conteneur

BesoinConteneurPourquoi
Liste ordonnée, parcourue souventTArrayAccès par index rapide, ajout en fin efficace.
Retrouver par clé (id, nom)TMapRecherche rapide clé → valeur.
Valeurs uniques / test d'appartenanceTSetPas de doublon, Contains rapide.

🔧 Démonstration pas à pas

  1. Un TArray<AActor*> d'ennemis : ajoute-en, parcours-les, retire les invalides.
  2. Un TMap<FName, int32> de scores : ajoute, recherche avec Find, parcours les paires.
  3. Un TSet<FName> de niveaux visités : vérifie qu'un doublon est ignoré.
  4. Construis un message récapitulatif avec FString::Printf.

⚠️ Pièges & erreurs courantes

SymptômeCrash sur un accès par index.
CauseIndex hors bornes (≥ Num() ou tableau vide).
CorrectionVérifie Array.IsValidIndex(i) avant l'accès.
SymptômeCrash en retirant pendant un range-based for.
CauseModification du conteneur en cours d'itération.
CorrectionBoucle à l'envers par index, ou utilise RemoveAll avec un prédicat.
SymptômeFind sur une TMap plante.
CauseDéréférencement du pointeur renvoyé sans vérifier qu'il est non nul.
CorrectionTeste le pointeur (if (int32* p = Map.Find(k))).
SymptômeMes AActor* dans un TArray disparaissent.
CauseLe TArray d'UObject n'est pas une UPROPERTY().
CorrectionMarque le conteneur UPROPERTY() (le GC suit alors ses éléments).

🔧Exercice pratique

  1. Un inventaire en TArray : ajouter/retirer des objets et afficher leur nombre.
  2. Une table TMap « nom d'objet → quantité » avec mise à jour et recherche.
  3. Un TSet des succès débloqués (sans doublon).
Critères de réussite
  • Aucun accès hors bornes.
  • Les conteneurs d'UObject sont des UPROPERTY().
  • Le bon conteneur est choisi pour chaque besoin.

📌À retenir
  • TArray (liste indexée), TMap (clé→valeur), TSet (unique) — équivalents Unreal des collections.
  • Parcours avec le range-based for ; vérifie les bornes (IsValidIndex).
  • FString pour manipuler ; FString::Printf pour formater.
  • Un conteneur d'UObject doit être UPROPERTY() (sinon GC).

Glossaire des termes introduits

TArray — liste dynamique indexée d'Unreal (équivalent de List).

TMap — table associant des clés à des valeurs (équivalent de Dictionary).

TSet — ensemble de valeurs uniques, test d'appartenance rapide.

FString::Printf — construit une chaîne formatée (style printf).
