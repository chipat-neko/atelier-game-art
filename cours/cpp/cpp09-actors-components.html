CPP09 — Actors, Components & cycle de vie

Intermédiaire 45 min Piste 8 · C++ (Unreal)
PrérequisCPP8 — Réflexion.
🎯Objectifs d'apprentissage

À la fin de cette leçon, tu sauras :

  • Distinguer un AActor de ses Components.
  • Créer des components en C++ avec CreateDefaultSubobject et les attacher.
  • Comprendre le cycle de vie : BeginPlay, Tick, EndPlay.
  • Faire apparaître/détruire un Actor avec SpawnActor et Destroy.

Pourquoi c'est important

Dans Unreal, tout ce qui existe dans un niveau est un Actor (un coffre, un personnage, une lumière). Un Actor est composé de Components qui lui donnent ses capacités : un mesh pour la forme, une collision, un son. C'est le pendant Unreal du modèle GameObject/Component d'Unity — un même socle pensé différemment.

Théorie en profondeur

1. Créer des Components dans le constructeur

Les components « par défaut » se créent dans le constructeur avec CreateDefaultSubobject, puis s'organisent en hiérarchie :

// Coffre.h
UPROPERTY(VisibleAnywhere) USceneComponent* Racine;
UPROPERTY(VisibleAnywhere) UStaticMeshComponent* Mesh;
// Coffre.cpp
ACoffre::ACoffre()
{
    PrimaryActorTick.bCanEverTick = true;

    Racine = CreateDefaultSubobject<USceneComponent>(TEXT("Racine"));
    RootComponent = Racine;                 // le component « racine » de l'Actor

    Mesh = CreateDefaultSubobject<UStaticMeshComponent>(TEXT("Mesh"));
    Mesh->SetupAttachment(RootComponent);   // attache le mesh sous la racine
}

Le RootComponent définit la position de l'Actor ; les autres components s'y attachent et suivent ses déplacements. Note la UPROPERTY() sur chaque pointeur de component (sinon le GC les détruit — CPP10).

2. Le cycle de vie d'un Actor

FonctionQuandUsage typique
ConstructeurÀ la création de l'objet (et dans l'éditeur).Créer les components, valeurs par défaut.
BeginPlay()Au démarrage du jeu / au spawn.Initialisation runtime, récupérer des références.
Tick(DeltaTime)Chaque frame (si activé).Logique continue (multiplie par DeltaTime).
EndPlay()À la destruction / fin du jeu.Nettoyage, désabonnements.
 
void ACoffre::BeginPlay()
{
    Super::BeginPlay();          // TOUJOURS appeler la version parente
    UE_LOG(LogTemp, Log, TEXT("Coffre prêt"));
}

void ACoffre::Tick(float DeltaTime)
{
    Super::Tick(DeltaTime);
    AddActorLocalRotation(FRotator(0, 90 * DeltaTime, 0));  // tourne sur lui-même
}
💡Constructeur ≠ BeginPlay

Le constructeur s'exécute aussi dans l'éditeur (pour afficher l'Actor) — n'y mets pas de logique de jeu (chercher d'autres Actors, lancer des timers…). Réserve ça à BeginPlay, qui ne tourne qu'en jeu.

3. Spawn & Destroy

// Faire apparaître un Actor à l'exécution
FVector Position(0, 0, 200);
FRotator Rotation = FRotator::ZeroRotator;
AProjectile* Tir = GetWorld()->SpawnActor<AProjectile>(ClasseProjectile, Position, Rotation);

// Le détruire
Tir->Destroy();                // détruit maintenant
SetLifeSpan(3.0f);            // ou : auto-destruction de CET Actor dans 3 s

GetWorld() donne accès au monde courant ; SpawnActor y crée un Actor d'une classe donnée.

🖐Essaie maintenant~1 min

Dans BeginPlay() tu écrirais UE_LOG(LogTemp, Warning, TEXT("Nom : %s"), *GetName());. Quel événement Unreal correspond au Start() d'Unity ? (BeginPlay ; Tick = Update.)

🔧 Démonstration pas à pas

  1. Crée ACoffre avec une racine et un UStaticMeshComponent attaché ; assigne un mesh dans l'éditeur.
  2. Active le Tick et fais tourner le coffre sur lui-même (multiplié par DeltaTime).
  3. Logge dans BeginPlay et EndPlay pour visualiser le cycle.
  4. Depuis un autre Actor, fais apparaître un coffre avec SpawnActor puis détruis-le après 3 s.

⚠️ Pièges & erreurs courantes

SymptômeCrash au lancement, pas dans l'éditeur.

🔧Exercice pratique

  1. Un ARamassable avec mesh attaché qui tourne lentement (Tick + DeltaTime).
  2. Un ASpawner qui fait apparaître un ramassable au BeginPlay via SpawnActor.
  3. Auto-destruction du ramassable après quelques secondes (SetLifeSpan).
Critères de réussite
  • Hiérarchie de components correcte (racine + mesh attaché).
  • Aucune logique de jeu dans le constructeur.
  • Spawn et destruction fonctionnent sans warning GC.

📌À retenir
  • Un Actor est composé de Components créés dans le constructeur (CreateDefaultSubobject).
  • RootComponent = base ; les autres s'attachent via SetupAttachment.
  • Cycle : Constructeur → BeginPlay → Tick (chaque frame) → EndPlay ; toujours Super::.
  • SpawnActor crée, Destroy/SetLifeSpan détruit ; la logique de jeu va dans BeginPlay, pas le constructeur.

Glossaire des termes introduits

Actor — tout objet plaçable dans un niveau Unreal (préfixe A).

Component — brique attachée à un Actor lui donnant une capacité (mesh, collision, son).

CreateDefaultSubobject — crée un component par défaut dans le constructeur.

RootComponent — component racine définissant la transformation de l'Actor.

BeginPlay / Tick / EndPlay — étapes du cycle de vie d'un Actor.
