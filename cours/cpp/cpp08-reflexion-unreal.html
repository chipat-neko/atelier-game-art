CPP08 — Le système de réflexion Unreal

Intermédiaire 45 min Piste 8 · C++ (Unreal)
PrérequisCPP7 — Classes & POO.
🎯Objectifs d'apprentissage

À la fin de cette leçon, tu sauras :

  • Ce qu'est la réflexion et ce qu'elle apporte à Unreal.
  • Utiliser UCLASS, GENERATED_BODY, UPROPERTY, UFUNCTION.
  • Exposer une propriété à l'éditeur et au Blueprint avec des spécificateurs.
  • Déclarer un UENUM et un USTRUCT.

Pourquoi c'est important

C'est le concept qui distingue le C++ Unreal du C++ ordinaire. La réflexion permet au moteur de « connaître » tes classes et leurs membres à l'exécution : il peut alors les afficher dans l'éditeur, les sérialiser (sauvegarde), les exposer au Blueprint, et les gérer dans le ramasse-miettes. Sans les macros U*, ton code reste invisible pour Unreal.

Comment ça marche

Avant la compilation, un outil nommé UHT (Unreal Header Tool) lit tes .h, repère les macros (UCLASS, UPROPERTY…) et génère du code (le fameux .generated.h) qui décrit tes types au moteur. C'est pour ça que #include "X.generated.h" doit être le dernier include et que GENERATED_BODY() est obligatoire.

Théorie en profondeur

1. UCLASS & GENERATED_BODY

UCLASS()                                   // « cette classe est connue d'Unreal »
class MONJEU_API AObjetRamassable : public AActor
{
    GENERATED_BODY()                       // injecte le code de réflexion généré
public:
    AObjetRamassable();
};

2. UPROPERTY : exposer une variable

Une UPROPERTY est une variable connue d'Unreal. Selon ses spécificateurs, elle apparaît dans l'éditeur, dans le Blueprint, et/ou est suivie par le GC :

// Réglable dans l'éditeur (Details) ET lisible/écrivable en Blueprint
UPROPERTY(EditAnywhere, BlueprintReadWrite, Category = "Stats")
int32 Munitions = 30;

// Réglable par instance dans l'éditeur, lecture seule en Blueprint
UPROPERTY(EditAnywhere, BlueprintReadOnly, Category = "Stats")
float Cadence = 0.2f;

// Référence d'objet suivie par le GC (indispensable, voir CPP10)
UPROPERTY()
UStaticMeshComponent* Mesh;
SpécificateurEffet
EditAnywhereModifiable dans le panneau Details (archétype et instances).
VisibleAnywhereVisible mais non modifiable dans l'éditeur.
BlueprintReadWriteLisible et modifiable depuis un Blueprint.
BlueprintReadOnlyLisible seulement depuis un Blueprint.
Category = "…"Regroupe les propriétés dans l'éditeur.

3. UFUNCTION : exposer une fonction

// Appelable depuis un Blueprint
UFUNCTION(BlueprintCallable, Category = "Combat")
void Tirer();

// Apparaît comme un nœud d'événement déclenchable (voir CPP14)
UFUNCTION(BlueprintImplementableEvent)
void OnRamasse();

On approfondira l'exposition Blueprint (BlueprintCallable, événements) en CPP14.

4. UENUM & USTRUCT

UENUM(BlueprintType)                  // un enum exposé au Blueprint
enum class EEtatIA : uint8
{
    Patrouille,
    Poursuite,
    Attaque
};

USTRUCT(BlueprintType)                // un regroupement de données exposé
struct FStatsArme
{
    GENERATED_BODY()

    UPROPERTY(EditAnywhere) int32 Degats = 10;
    UPROPERTY(EditAnywhere) float Portee = 1000.0f;
};

🔧 Démonstration pas à pas

  1. Sur une classe Actor, ajoute UPROPERTY(EditAnywhere, Category="Stats") int32 PV = 100; et compile.
  2. Place l'Actor dans la scène et modifie PV dans le panneau Details : la réflexion en action.
  3. Ajoute UFUNCTION(BlueprintCallable) void Soigner(int32 Montant); et appelle-la depuis un Blueprint enfant.
  4. Déclare un UENUM(BlueprintType) d'états et expose une UPROPERTY de ce type.

⚠️ Pièges & erreurs courantes

SymptômeMa variable n'apparaît pas dans l'éditeur.
CausePas de UPROPERTY(EditAnywhere) (variable C++ « nue »).
CorrectionAjoute la macro avec le bon spécificateur.
SymptômeMon objet référencé disparaît (GC).
CausePointeur d'UObject sans UPROPERTY() : invisible au GC.
CorrectionMarque toute référence d'UObject avec UPROPERTY() (voir CPP10).
Symptôme« GENERATED_BODY must be the first thing… ».
CauseGENERATED_BODY() mal placé ou manquant.
CorrectionPlace-le en première ligne du corps de la classe.
SymptômeUHT échoue / erreurs cryptiques.
Cause.generated.h pas en dernier, ou enum sans : uint8.
Correction.generated.h en dernier ; enum class … : uint8 pour UENUM(BlueprintType).

🔧Exercice pratique

  1. Une classe AArme avec des UPROPERTY exposées (munitions, cadence, dégâts) classées par Category.
  2. Un UENUM(BlueprintType) de types de munitions et une UPROPERTY de ce type.
  3. Une UFUNCTION(BlueprintCallable) Recharger() testable depuis l'éditeur.
Critères de réussite
  • Les propriétés apparaissent et sont éditables dans Details.
  • La fonction est appelable depuis un Blueprint.
  • Aucune référence d'UObject n'est sans UPROPERTY().

📌À retenir
  • La réflexion rend tes types visibles au moteur (éditeur, GC, Blueprint, sérialisation).
  • UCLASS + GENERATED_BODY() ; UPROPERTY pour les variables, UFUNCTION pour les fonctions.
  • Les spécificateurs (EditAnywhere, BlueprintReadWrite, Category) règlent l'exposition.
  • Toute référence d'UObject doit être une UPROPERTY() (sinon le GC la détruit).

Glossaire des termes introduits

Réflexion — capacité du moteur à connaître tes types et membres à l'exécution.

UHT (Unreal Header Tool) — outil qui génère le code de réflexion avant la compilation.

UPROPERTY / UFUNCTION — macros exposant une variable / une fonction au moteur.

UENUM / USTRUCT — énumération / structure de données exposées à Unreal.
