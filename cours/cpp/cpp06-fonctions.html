CPP06 — Fonctions & en-têtes (.h / .cpp)

Intermédiaire 45 min Piste 8 · C++ (Unreal)
PrérequisCPP2 et CPP5.
🎯Objectifs d'apprentissage

À la fin de cette leçon, tu sauras :

  • Déclarer une fonction dans le .h et la définir dans le .cpp.
  • Choisir le passage de paramètre : valeur, référence, const référence.
  • Renvoyer une valeur et utiliser des paramètres par défaut.
  • Comprendre le rôle des includes et de #pragma once.

Pourquoi c'est important

Les fonctions découpent un programme en morceaux nommés, réutilisables et testables. En C++, une particularité majeure : une fonction se déclare (sa signature, dans le .h) puis se définit (son code, dans le .cpp). Comprendre cette séparation est la clé pour s'y retrouver dans un projet Unreal.

Théorie en profondeur

1. Déclaration (.h) vs définition (.cpp)

// Arme.h — on DÉCLARE : « cette fonction existe, voici sa signature »
UCLASS()
class MONJEU_API AArme : public AActor
{
    GENERATED_BODY()

public:
    void Tirer();                          // ne renvoie rien
    int32 CalculerDegats(int32 Base) const; // renvoie un int32, ne modifie pas l'objet
};
// Arme.cpp — on DÉFINIT : le code réel
#include "Arme.h"

void AArme::Tirer()                        // noter le AArme:: (la classe propriétaire)
{
    UE_LOG(LogTemp, Log, TEXT("Pan !"));
}

int32 AArme::CalculerDegats(int32 Base) const
{
    return Base * 2;                       // renvoie une valeur
}

Le AArme:: indique à quelle classe la fonction appartient. Le const en fin de signature promet que la fonction ne modifie pas l'objet (bonne pratique pour les accesseurs).

2. Passage de paramètres : le point crucial

C'est la notion C++ à bien saisir. Trois façons de passer un argument :

FormeSyntaxeEffetQuand
Par valeurvoid F(int32 x)Copie l'argument ; l'original est intact.Petits types (int, float, bool).
Par référencevoid F(int32& x)Travaille sur l'original : le modifie.Renvoyer plusieurs résultats.
Par const référencevoid F(const FString& s)Pas de copie, lecture seule.Gros objets (FString, structs).
 
// Modifier l'original via une référence
void AppliquerDegats(int32& PV, int32 Degats)
{
    PV -= Degats;        // modifie la variable du code appelant
}

// Lire un gros objet sans le copier
void Afficher(const FString& Message)
{
    UE_LOG(LogTemp, Log, TEXT("%s"), *Message);
}

Règle simple : petits types par valeur, gros objets en const référence (pour éviter une copie coûteuse), et référence non-const seulement quand on veut modifier l'argument.

3. Paramètres par défaut & surcharge

// Valeur par défaut : l'appelant peut omettre l'argument
void Soigner(int32 Montant = 10);
Soigner();      // soigne 10
Soigner(25);    // soigne 25

// Surcharge : même nom, signatures différentes
void Log(int32 Valeur);
void Log(const FString& Texte);   // le compilateur choisit selon le type passé

4. Includes & #pragma once

Pour utiliser une classe d'un autre fichier, on l'#include. Le #pragma once en tête de chaque .h empêche qu'il soit inclus plusieurs fois (ce qui provoquerait des erreurs de redéfinition) :

#pragma once               // garde d'inclusion : une seule fois
#include "CoreMinimal.h"
#include "Arme.h"          // pour utiliser AArme ici

On verra en CPP15 comment limiter les includes lourds avec les forward declarations pour accélérer la compilation.

🔧 Démonstration pas à pas

  1. Déclare dans un .h une fonction int32 CalculerScore(int32 Base, int32 Multi = 1) const;.
  2. Définis-la dans le .cpp avec le préfixe MaClasse::.
  3. Écris une fonction void AppliquerDegats(int32& PV, int32 Degats) qui modifie le PV de l'appelant.
  4. Écris void Afficher(const FString& Msg) et vérifie qu'aucune copie n'est faite.

🔧Exercice pratique

  1. Une fonction const CalculerDegats qui renvoie une valeur sans modifier l'objet.
  2. Une fonction qui échange deux entiers via références (int32&).
  3. Une surcharge Afficher pour un int32 et pour un FString.
Critères de réussite
  • Déclaration dans le .h, définition dans le .cpp.
  • Les gros objets passent en const référence.
  • Le projet compile et l'éditeur de liens ne se plaint pas.

📌À retenir
  • Déclaration (.h) = signature ; définition (.cpp) = code, préfixée Classe::.
  • Petits types par valeur, gros objets en const référence, référence pour modifier.
  • const en fin de signature : la fonction ne modifie pas l'objet.
  • #pragma once en tête de chaque .h.

Glossaire des termes introduits

Déclaration / définition — annoncer la signature / fournir le code d'une fonction.

Passage par référence — Type& : la fonction agit sur l'original, sans copie.

const référence — const Type& : lecture sans copie d'un gros objet.

#pragma once — garde empêchant la double inclusion d'un en-tête.
