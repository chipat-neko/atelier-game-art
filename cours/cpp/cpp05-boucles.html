CPP05 — Boucles (for, while, range-based)

Débutant 35 min Piste 8 · C++ (Unreal)
PrérequisCPP4 — Conditions.
🎯Objectifs d'apprentissage

À la fin de cette leçon, tu sauras :

  • Répéter du code avec for, while et do-while.
  • Parcourir une collection avec le range-based for.
  • Contrôler une boucle avec break et continue.
  • Comprendre pourquoi une boucle ne doit pas bloquer une frame.

Pourquoi c'est important

Dès qu'il faut traiter « tous les ennemis », « chaque case d'une grille » ou répéter une action N fois (spawner 10 objets), on utilise une boucle. C'est l'outil de la répétition.

Théorie en profondeur

1. La boucle for

Quand le nombre de répétitions est connu :

for (int32 i = 0; i < 10; i++)   // init ; condition ; incrément
{
    UE_LOG(LogTemp, Log, TEXT("Spawn n°%d"), i);
}

2. while & do-while

while (munitions > 0)     // teste AVANT chaque tour
{
    Tirer();
    munitions--;
}

do
{
    Action();             // exécute AU MOINS une fois, puis teste
} while (bContinuer);

3. Le range-based for (le plus utile en jeu)

Pour parcourir chaque élément d'un conteneur (un TArray, vu en CPP11) sans gérer d'index :

for (AActor* Ennemi : Ennemis)        // "pour chaque Ennemi dans Ennemis"
{
    if (Ennemi)                        // ignore les éléments nuls
    {
        Ennemi->Destroy();
    }
}

// Avec référence pour modifier sur place, ou const pour lecture seule :
for (const FString& Nom : Noms)
{
    UE_LOG(LogTemp, Log, TEXT("%s"), *Nom);
}

Le & (référence) évite de copier chaque élément ; const indique qu'on ne les modifie pas (références détaillées en CPP6).

4. break & continue

for (AActor* A : Acteurs)
{
    if (!A) continue;            // passe au tour suivant (saute le reste)
    if (A->ActorHasTag("Boss"))
    {
        Cible = A;
        break;                  // sort immédiatement de la boucle
    }
}
🛑Ne bloque jamais une frame

Une boucle très longue ou while(true) sans sortie s'exécute entièrement dans la même frame et gèle le jeu. Pour étaler un traitement dans le temps, on utilise les timers (CPP14) ou le Tick (CPP9), pas une boucle qui attend.

🔧 Démonstration pas à pas

  1. Avec un for, logge les nombres de 1 à 5.
  2. Avec un while, vide un stock de munitions en tirant.
  3. Parcours un TArray<AActor*> avec un range-based for et détruis chaque élément valide.
  4. Cherche le premier acteur taggé « Boss » avec break, en ignorant les nuls par continue.

⚠️ Pièges & erreurs courantes

CorrectionGarantis la progression vers la sortie ; pour le temps, utilise un timer.

🔧Exercice pratique

  1. Boucle for qui calcule la somme des entiers de 1 à 100.
  2. Parcours un TArray de PV et compte combien sont « morts » (≤ 0).
  3. Trouve le premier ennemi à portée avec break et saute les nuls avec continue.
Critères de réussite
  • Aucune boucle ne bloque la frame.
  • Le parcours du tableau utilise des références const quand c'est de la lecture.
  • Pas d'erreur de borne ni de modification du conteneur pendant l'itération.

📌À retenir
  • for = nombre connu ; while = tant que vrai ; do-while = au moins une fois.
  • Le range-based for (for (auto& x : conteneur)) est l'outil idiomatique du parcours.
  • break sort, continue passe au tour suivant.
  • Ne bloque jamais une frame : pour le temps, timers/Tick, pas une boucle d'attente.

Glossaire des termes introduits

Range-based for — boucle for (T x : conteneur) qui visite chaque élément.

break / continue — sortir de la boucle / passer immédiatement au tour suivant.

Borne — limite d'index d'un parcours (0 à N-1 pour un tableau de taille N).
