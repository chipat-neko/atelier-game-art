CPP01 — Introduction au C++ & Unreal

Débutant 40 min Piste 8 · C++ (Unreal)
Prérequisaucune expérience du C++ requise. Avoir déjà ouvert Unreal aide (voir la piste Unreal).
🎯Objectifs d'apprentissage

À la fin de cette leçon, tu sauras :

  • Pourquoi Unreal utilise le C++ et comment il se marie au Blueprint.
  • Ce qu'est un projet C++ Unreal (module, .h/.cpp, .Build.cs).
  • Créer ta première classe C++ depuis l'éditeur.
  • Compiler et utiliser le Live Coding pour itérer vite.

Pourquoi c'est important

Unreal Engine est lui-même écrit en C++. Tout ce que tu fais en Blueprint repose sur du code C++ sous-jacent. Apprendre le C++ te donne accès à tout le moteur : performances maximales, systèmes que le Blueprint ne peut pas faire, et la capacité de lire/comprendre le code source d'Unreal. Ce n'est pas « Blueprint ou C++ » — les deux coexistent, et les studios professionnels combinent presque toujours les deux.

C++ vs Blueprint : complémentaires, pas rivaux

AxeC++Blueprint
PerformanceCode natif compilé, maximal.Interprété (machine virtuelle) : plus lent à grande échelle.
Idéal pourSystèmes de base, logique lourde, accès complet au moteur.Prototypage, UI, level design, câblage gameplay.
ItérationNécessite une compilation (Live Coding accélère).Immédiate, visuelle, sans compilation longue.
VersionnageTexte : diffs et revues de code propres.Binaire : diffs difficiles, conflits délicats.
AccessibilitéExigeant : syntaxe stricte, mémoire.Accessible aux non-programmeurs.

L'approche classique : écrire les fondations et les systèmes coûteux en C++, puis exposer au Blueprint (variables et fonctions) pour que les designers les assemblent (on verra l'exposition en CPP14).

Théorie en profondeur

1. Anatomie d'un projet C++ Unreal

Quand tu crées un projet C++, Unreal génère un module de code :

  • Source/MonProjet/ — tes fichiers .h (en-têtes, déclarations) et .cpp (définitions, code).
  • MonProjet.Build.cs — décrit les dépendances du module (un fichier C#, pas C++).
  • MonProjet.uproject — le projet, ouvrable dans l'éditeur.

Chaque classe C++ vit généralement dans deux fichiers : un en-tête (.h) qui déclare ce qui existe, et une source (.cpp) qui définit ce que ça fait (on détaille en CPP6).

2. Ta première classe C++

Dans l'éditeur : Tools ▸ New C++ Class ▸ choisis une classe parente (souvent Actor) ▸ nomme-la. Unreal génère le squelette :

// MonActor.h — l'EN-TÊTE : on déclare la classe
#pragma once

#include "CoreMinimal.h"
#include "GameFramework/Actor.h"
#include "MonActor.generated.h"   // généré par Unreal — toujours en DERNIER include

UCLASS()
class MONPROJET_API AMonActor : public AActor
{
    GENERATED_BODY()

public:
    AMonActor();                  // constructeur

protected:
    virtual void BeginPlay() override;   // appelé au lancement du jeu

public:
    virtual void Tick(float DeltaTime) override;  // appelé chaque frame
};
// MonActor.cpp — la SOURCE : on définit le comportement
#include "MonActor.h"

AMonActor::AMonActor()
{
    PrimaryActorTick.bCanEverTick = true;   // active le Tick
}

void AMonActor::BeginPlay()
{
    Super::BeginPlay();
    UE_LOG(LogTemp, Warning, TEXT("Bonjour depuis le C++ !"));
}

void AMonActor::Tick(float DeltaTime)
{
    Super::Tick(DeltaTime);
}

Remarque le préfixe A de AMonActor : Unreal impose des préfixes par convention (A pour Actor, U pour objet, F pour structure…). On y reviendra en CPP8.

3. Compiler & Live Coding

Le C++ doit être compilé (traduit en code machine) avant de s'exécuter — contrairement au Blueprint. Trois façons :

  • Live Coding (Ctrl+Alt+F11 depuis l'éditeur) : recompile à chaud sans fermer l'éditeur — idéal pour itérer.
  • Bouton Compile de la barre d'outils de l'éditeur.
  • Build complet depuis l'IDE (Visual Studio / Rider) — nécessaire après certains changements de structure.

4. UE_LOG : ton premier outil de débogage

L'équivalent du print : UE_LOG écrit dans la fenêtre Output Log. Note le TEXT("...") autour des chaînes (obligatoire pour le texte Unreal, vu en CPP2) :

UE_LOG(LogTemp, Warning, TEXT("Valeur : %d"), 42);     // %d pour un entier
UE_LOG(LogTemp, Log, TEXT("Texte simple"));

🔧 Démonstration pas à pas

  1. Crée un projet C++ (modèle « Blank », C++) ou ajoute du C++ à un projet existant.
  2. Tools ▸ New C++ Class ▸ Actor, nomme-la MonActor. Attends la compilation.
  3. Dans BeginPlay(), ajoute un UE_LOG(...) qui affiche un message.
  4. Lance le Live Coding (Ctrl+Alt+F11), glisse l'Actor dans la scène, joue, et observe l'Output Log.

⚠️ Pièges & erreurs courantes

CorrectionPlace toujours #include "X.generated.h" en dernier.
SymptômeMes modifications C++ n'ont aucun effet.
CauseTu n'as pas recompilé.
CorrectionLance le Live Coding ou clique sur Compile.
SymptômeLive Coding refuse certains changements.
CauseAjout/suppression de membres UPROPERTY/UFUNCTION ou de classes.

Symptôme« cannot open include file ».
CauseModule non déclaré comme dépendance.

🔧Exercice pratique

  1. Crée une classe C++ Salutation héritant d'Actor.
  2. Affiche un message différent dans le constructeur et dans BeginPlay().
  3. Compile via Live Coding, place l'Actor, joue, et repère les deux messages dans l'Output Log.
Critères de réussite
  • La classe compile sans erreur.
  • Les messages apparaissent dans l'Output Log au bon moment.
  • Tu sais relancer une compilation à chaud.

📌À retenir
  • Unreal est en C++ ; C++ et Blueprint sont complémentaires, pas concurrents.
  • Une classe = un .h (déclaration) + un .cpp (définition) ; le C++ se compile.
  • Les préfixes (A, U, F) sont imposés ; .generated.h toujours en dernier include.
  • Live Coding pour itérer ; UE_LOG(..., TEXT("...")) pour déboguer.

Glossaire des termes introduits

Module — unité de code Unreal compilée ensemble, décrite par un fichier .Build.cs.

Live Coding — recompilation à chaud du C++ sans fermer l'éditeur.

UE_LOG — macro d'affichage de messages dans l'Output Log (débogage).
