B7 — Shading & matériaux (nodes)

Intermédiaire 50 min Piste 2 · Blender
PrérequisF7 — PBR, F1 (sRGB/linéaire) et B6 — UV.
🎯Objectifs d'apprentissage

À la fin de cette leçon, tu sauras :

  • Utiliser le Shader Editor et le nœud Principled BSDF.
  • Brancher tes maps PBR (Base Color, Metallic, Roughness, Normal) sur les bonnes entrées.
  • Régler l'espace colorimétrique Non-Color sur les maps de données — l'erreur n°1.
  • Distinguer EEVEE (temps réel) et Cycles (path tracer), et créer des node groups réutilisables.

Pourquoi c'est important

Le Principled BSDF de Blender est un shader PBR (renvoi F7) compatible avec ce que comprennent Unity et Unreal : ses entrées (Base Color, Metallic, Roughness, Normal) correspondent aux maps standard. Bien le brancher dans Blender te donne un aperçu fidèle de ton matériau avant l'export — à condition de ne pas commettre l'erreur classique de l'espace colorimétrique, qui ruine normales et roughness.

Théorie en profondeur

1. Le Shader Editor et le Principled BSDF

Dans le Workspace Shading, le bas de l'écran affiche le Shader Editor (un éditeur de nœuds). Un nouveau matériau contient déjà un nœud Principled BSDF relié à Material Output. Le Principled regroupe toutes les propriétés PBR en un seul nœud : Base Color, Metallic, Roughness, Specular, Normal, Emission, Alpha, etc. On branche une texture sur une entrée via un nœud Image Texture.

2. Brancher ses maps PBR (et l'espace colorimétrique)

Le branchement type (cohérent avec F7) :

  • Base Color ← Image Texture de l'albedo, en Color Space = sRGB (c'est une couleur).
  • Metallic ← Image Texture metallic, en Non-Color.
  • Roughness ← Image Texture roughness, en Non-Color.
  • Normal ← Image Texture normal (Non-Color) → nœud Normal Map → entrée Normal du Principled. Le nœud intermédiaire est obligatoire : il convertit le RGB en vecteur.
📌La règle qui sauve des heures

Les maps de données (normal, roughness, metallic, AO, height) doivent être en Color Space = Non-Color dans le nœud Image Texture. Seules les maps de couleur (Base Color, Emission) restent en sRGB. Oublier le Non-Color → Blender « décode » des nombres comme des couleurs → rendu plastique/faux (renvoi F1).

3. EEVEE vs Cycles

Blender a deux moteurs de rendu (choisis dans Render Properties▸Render Engine) :

  • EEVEE : moteur temps réel (rasterizer), rapide et interactif — proche de la façon dont un moteur de jeu calcule l'image. Idéal pour un aperçu fidèle de tes matériaux destinés au jeu.
  • Cycles : moteur par lancer de rayons (path tracer), physiquement réaliste (reflets, GI, caustiques), mais lent et bruité (le bruit diminue avec les échantillons). Pour les rendus de présentation et le baking de qualité (renvoi B8).

Un même matériau Principled rend de façon cohérente dans les deux, mais certains effets ne s'affichent qu'en Cycles. Pour du game art, EEVEE est le compagnon naturel.

4. Node groups réutilisables

Quand un ensemble de nœuds se répète (un « détail tilé », un mélange de deux matériaux par masque), on le sélectionne et on fait Ctrl+G pour créer un node group : un nœud unique réutilisable avec ses entrées/sorties exposées. C'est l'équivalent des Material Functions d'Unreal (renvoi E5). L'add-on Node Wrangler (activé en B1) accélère tout : Ctrl+T ajoute automatiquement les nœuds d'une texture.

🔧 Démonstration pas à pas

  1. Sélectionne ton objet déplié (B6), va dans le Workspace Shading, crée un matériau (New).
  2. Avec Node Wrangler : sélectionne le Principled BSDF, Ctrl+T n'ajoute qu'une texture ; sinon ajoute un Image Texture (Maj+A ▸ Texture) et charge ta Base Color → branche sur Base Color (laisse sRGB).
  3. Ajoute la Roughness en Image Texture → passe son Color Space en Non-Color → branche sur Roughness.
  4. Ajoute la Normal en Image Texture (Non-Color) → nœud Normal Map → entrée Normal.
  5. Bascule en Material Preview ou Rendered (EEVEE) pour voir le résultat ; ajuste.
  6. Sélectionne le bloc « détail » réutilisable, Ctrl+G pour en faire un node group nommé.

⚠️ Pièges & erreurs courantes

SymptômeMa surface paraît « plastique », délavée ou éclaire bizarrement.
CauseRoughness/normal laissées en sRGB au lieu de Non-Color.
CorrectionPasse toutes les maps de données en Color Space = Non-Color.
SymptômeMa normal map n'a aucun effet de relief.
CauseTexture branchée directement sur Normal sans le nœud Normal Map.
CorrectionInsère un nœud Normal Map entre l'Image Texture et l'entrée Normal.
SymptômeLe rendu Cycles est très lent / très bruité.
CausePath tracer + GPU non activé + peu d'échantillons.
CorrectionActive le GPU (B1), monte les samples ou active le Denoise ; ou passe en EEVEE pour le travail courant.
SymptômeMon matériau Blender ne ressemble pas à celui obtenu dans le moteur.
CauseDifférences de pipeline ; les matériaux ne se transfèrent pas tels quels.
CorrectionExporte les textures et reconstruis le matériau dans le moteur (renvoi C2, U6, E5).

🔧Exercice pratique

  1. Crée un matériau PBR complet (Base Color, Roughness, Metallic, Normal) sur ton objet déplié, espaces colorimétriques corrects.
  2. Compare le rendu EEVEE et Cycles du même matériau.
  3. Crée un node group « détail tilé » et réutilise-le sur un autre objet.
Critères de réussite
  • Les maps de données sont en Non-Color, la normal passe par un nœud Normal Map.
  • Le matériau réagit correctement à la lumière (pas d'aspect plastique).
  • Tu sais expliquer la différence EEVEE/Cycles et quand utiliser chacun.

Vérifie ta compréhension

Clique sur la réponse que tu penses correcte : la bonne réponse et une explication s'affichent aussitôt.

Exercices

Exercice 1Prise en main

Crée un matériau avec le Principled BSDF, règle sa couleur de base et sa rugosité, et observe en aperçu rendu.

Voir une piste

Dans le Shading workspace, le Principled BSDF est déjà là. Change Base Color et Roughness, passe le viewport en Rendered (Eevee/Cycles) pour voir l'effet en direct.

Exercice 2Application

Branche une texture Base Color et une Roughness map via des nœuds Image Texture (avec le bon espace colorimétrique).

Voir une piste

Base Color en sRGB ; Roughness en Non-Color. Ajoute Shift+A ▸ Texture ▸ Image Texture pour chaque, relie-les aux bonnes entrées. Le Node Wrangler (Ctrl+T) accélère le branchement coordonnées/mapping.

Exercice 3Défi

Ajoute une normal map correctement (Non-Color + nœud Normal Map) et vérifie le relief sous un éclairage mobile.

Voir une piste

Image Texture (Non-Color) → nœud Normal Map → entrée Normal du Principled. Déplace une lampe : le relief doit réagir de façon cohérente. Si le creux/bosse est inversé, vérifie la convention (OpenGL/DirectX) de la map.

📌À retenir
  • Le Principled BSDF est un shader PBR : branche tes maps sur ses entrées.
  • Maps de données en Non-Color ; couleur en sRGB ; normale via le nœud Normal Map.
  • EEVEE temps réel (aperçu jeu) vs Cycles path tracer (réaliste, bake).
  • Les node groups factorisent ; Node Wrangler accélère le travail.

Glossaire des termes introduits

Shader Editor — éditeur de nœuds des matériaux dans Blender.

Principled BSDF — shader PBR tout-en-un de Blender, compatible avec les moteurs.

Non-Color — espace colorimétrique des maps de données (normal, roughness…), à ne pas décoder comme une couleur.

Node group — ensemble de nœuds regroupé en un nœud réutilisable.
