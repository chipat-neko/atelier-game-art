Guide complet de l'interface

Débutant 45 min Piste 2 · Blender
PrérequisB1 — Installation & configuration. Cette leçon est une référence : garde-la sous la main pour retrouver n'importe quelle commande.
🎯Objectif de ce guide

Ce guide te montre, capture à l'appui, où se trouve chaque chose dans Blender et à quoi elle sert : chaque menu est ouvert et chaque item est expliqué dans un tableau. L'idée est simple : quand tu cherches une commande, tu la trouves ici, sans aller sur une vidéo ou un autre site.

💡Comment lire ce guide

Chaque capture est cliquable : clique dessus pour l'agrandir en plein écran (Échap ou un clic pour fermer). Les raccourcis sont indiqués entre parenthèses (ex. Ctrl S). Certains items ouvrent un sous-menu (petite flèche ▸ à droite).

La barre supérieure

Tout en haut de la fenêtre, la barre supérieure (Topbar) contient les menus qui agissent sur tout le fichier et l'application : ouvrir/enregistrer, préférences, rendu, fenêtres, aide. À droite se trouvent aussi les onglets d'espaces de travail (Layout, Modeling, Sculpting…).

Le Viewport 3D — Object Mode

Dans le grand viewport 3D, une seconde barre de menus agit sur la scène et les objets. En Object Mode (le mode par défaut), on y trouve View, Select, Add et Object.

Le Viewport 3D — Edit Mode

Quand tu entres en Edit Mode (touche Tab) sur un mesh, la barre de menus change : Add et Object laissent place à Mesh, Vertex, Edge, Face et UV. Ce sont les outils de modélisation pure (voir B3).

Les panneaux autour du viewport

Autour du viewport, quatre zones reviennent toujours. On les repère sur la vue d'ensemble ci-dessous (revois B2 pour la navigation).

PanneauRôle
Barre d'outils (T)À gauche du viewport : outils actifs (sélection, déplacer, tourner, échelle, annoter, mesurer…).
OutlinerEn haut à droite : l'arborescence de la scène (objets, collections, données). On y sélectionne, range, masque.
PropertiesÀ droite : une colonne d'onglets (Render, Output, View Layer, Scene, World, Object, Modifiers 🔧, Particles, Physics, Constraints, Data, Material). Chaque onglet regroupe les réglages d'un domaine.
TimelineEn bas : la lecture, les images-clés et le curseur de temps pour l'animation.

Le panneau Properties, onglet par onglet

Le panneau Properties (à droite) est organisé en une colonne d'onglets (des icônes) qui filtrent les réglages par domaine. Cliquer sur une icône change tout le contenu affiché en dessous. Voici, de haut en bas, à quoi sert chaque onglet.

OngletÀ quoi il sert
Active ToolRéglages de l'outil actif de la barre d'outils (rayon d'une brosse, options de l'outil de découpe…).
RenderLe moteur de rendu (EEVEE ou Cycles), le nombre d'échantillons (samples), l'anti-aliasing, la qualité globale.
OutputLa sortie : résolution, fréquence d'images, plage (Start/End), format et dossier des fichiers rendus.
View LayerLes calques de rendu et les passes (isoler diffuse, ombres… pour le compositing).
SceneLa caméra active, les unités de mesure, la gravité, et la gestion des couleurs (color management).
WorldL'environnement : couleur ou image de ciel (HDRI), éclairage d'ambiance global.
ObjectLe Transform (Location/Rotation/Scale), la visibilité, les relations (parent, collections).
Modifiers (clé à molette)La pile de modifiers non destructifs (Mirror, Bevel, Subdivision… — voir B4).
ParticlesLes systèmes de particules : cheveux/fourrure, émissions, dispersion.
PhysicsLes simulations : corps rigides, tissu, fluide, collisions, force fields.
Object ConstraintsLes contraintes (Copy Location, Track To, Child Of…) qui pilotent l'objet automatiquement.
Object Data (triangle vert)Les données du maillage : groupes de sommets, UV Maps, shape keys, attributs, normales.
Material (sphère à damier)Les matériaux de l'objet et leurs emplacements (slots) — le point d'entrée du shading (voir B7).

Les principaux éditeurs

Blender n'a pas de « fenêtres » fixes : chaque zone est un éditeur qu'on peut changer via le petit menu déroulant (une icône) en haut à gauche de la zone. C'est ainsi qu'on passe le viewport en Shader Editor, en UV Editor, etc. Les espaces de travail (Layout, Shading, UV Editing…) ne sont que des dispositions d'éditeurs préconfigurées. Voici les plus utiles.

ÉditeurÀ quoi il sert
3D ViewportLa vue 3D : modéliser, sculpter, animer, poser lumières et caméras.
OutlinerL'arborescence de la scène (objets, collections, données liées).
PropertiesLes réglages par domaine (voir la section ci-dessus).
Shader EditorConstruire les matériaux par nœuds (voir B7).
UV EditorVoir et éditer le dépliage UV (voir B6).
Image EditorVoir, charger ou peindre une image / texture.
Geometry Node EditorGéométrie procédurale par nœuds (voir B9).
Timeline · Dope Sheet · Graph EditorL'animation : lecture, images-clés, et courbes d'interpolation.
CompositorLe post-traitement du rendu par nœuds (glow, corrections, passes).
Video SequencerMontage vidéo (assembler des plans, du son).
Text Editor · Python ConsoleÉcrire des scripts Python (automatiser, créer des add-ons).
File BrowserParcourir les fichiers (s'ouvre lors d'un import/export/enregistrement).
📌À retenir
  • La barre supérieure agit sur le fichier et l'appli : File (ouvrir/enregistrer, import/export), Edit (annuler, F3 = chercher une commande, Préférences), Render (F12), Window, Help (le manuel officiel).
  • Le viewport 3D a ses propres menus : en Object Mode View / Select / Add / Object ; en Edit Mode ils deviennent Mesh / Vertex / Edge / Face.
  • Deux réflexes qui sauvent : F3 pour chercher n'importe quelle commande, et Object ▸ Apply (Ctrl A) pour remettre Scale à 1 avant d'exporter.
  • Le panneau Properties range les réglages par onglet (Render, Object, Modifiers, Material…) ; et chaque zone est un éditeur qu'on change via le menu en haut à gauche (viewport, Shader Editor, UV Editor…).
  • Chaque capture de ce guide est cliquable pour l'agrandir : reviens ici dès que tu cherches « où est cette commande ? ».

Vérifie ta compréhension

Clique sur la réponse que tu penses correcte : la bonne réponse et une explication s'affichent aussitôt.

Critères de réussite
  • Je sais ce que couvre chaque menu de la barre supérieure (File, Edit, Render, Window, Help).
  • Je connais deux réflexes clés : F3 (chercher une commande) et Object ▸ Apply (Ctrl A) avant d'exporter.
  • Je distingue les menus du viewport en Object Mode (View / Select / Add / Object) et en Edit Mode (Mesh / Vertex / Edge / Face).
  • Je situe les panneaux autour du viewport : barre d'outils (T), Outliner, Properties, Timeline.
  • Je sais que chaque zone est un éditeur interchangeable (viewport, Shader Editor, UV Editor…) et je repère les onglets du panneau Properties.
